    வெளிப்புற ஒருங்கிணைப்புகள்¶

    மூன்றாம் தரப்பு சேவைகள்¶

    சேவை நோக்கம் குறிப்புகள்
    LaunchDarkly அம்சக் கொடிகள் மற்றும் கட்டுப்படுத்தப்பட்ட வெளியீடுகள் அனைத்து 9 பின்னணி வேலைகளையும் கட்டுப்படுத்துகிறது; கட்டமைப்பு இல்லாமல் கொடி மாறுதல்
    OpenAI (gpt-5.4-mini) AI-உருவாக்கப்பட்ட அறிக்கை சுருக்கங்கள் விருப்பமானது; உண்மையான வெளியீட்டிற்கு temperature 0.3; அம்சக் கொடியால் கட்டுப்படுத்தப்பட்டது
    MailerSend பரிவர்த்தனை மின்னஞ்சல் வழங்கல் அறிக்கைகள் மற்றும் வாழ்க்கைச் சுழற்சி அறிவிப்பு மின்னஞ்சல்களுக்கு பயன்படுத்தப்படுகிறது; 3-முயற்சி மீண்டும் முயற்சி
    Umami தனியுரிமை-நட்பு பகுப்பாய்வு விருப்பமானது; 13 தனிப்பயன் நிகழ்வு வகைகள் கண்காணிக்கப்படுகின்றன
    OpenRouteService வழிசெலுத்தல்/திசைகள் API வாகனம், நடைபாதை மற்றும் சைக்கிள் வழிகள்
    LocationIQ புவியியல் குறியீடு API முகவரி ஆய கண்டுபிடிப்பு

    தரவு கட்டமைப்பு¶

    நிறுவன உறவு மாதிரி¶

    கணினி சபை-அளவிலான தனிமைப்படுத்தலுடன் படிநிலையாக தரவை ஒழுங்கமைக்கிறது:

    Congregation (Organization boundary)
    ├── Territory (Geographic region)
    │   ├── Map (Building/Location)
    │   │   └── Address (Individual unit)
    │   └── Option (Address type classification)
    ├── User (System user)
    │   └── Role (Permission assignment)
    ├── Assignment (Publisher access)
    └── Message (Notification)

    முக்கிய தொகுப்புகள்¶

    1. Congregations - நிறுவன அமைப்புகள் மற்றும் எல்லைகள் - பல-குத்தகை தனிமைப்படுத்தலை கட்டுப்படுத்துகிறது - பணிப்பு கால அளவு மற்றும் மீண்டும் முயற்சி வரம்புகளை வரையறுக்கிறது - பிராந்திய அமைப்புகள் (நேர மண்டலம், நாடு)

    2. Territories - புவியியல் பிரிவுகள் - பல வரைபடங்களை உள்ளடக்கியது - ஒருங்கிணைந்த நிறைவு சதவீதத்தை கண்காணிக்கிறது - வயல் சேவை பகுதிகளை ஒழுங்கமைக்கிறது

    3. Maps - குறிப்பிட்ட இடங்கள் (கட்டிடங்கள்) - தனி அல்லது பல-தள கட்டமைப்புகள் - புவியியல் ஆயங்கள் - முன்னேற்ற கண்காணிப்பு மற்றும் புள்ளிவிவரங்கள்

    4. Addresses - தனிப்பட்ட அலகுகள்/வீடுகள் - நிலை கண்காணிப்பு (முடிக்கப்படவில்லை, முடிந்தது, வீட்டில் இல்லை, DNC) - வருகை குறிப்புகள் மற்றும் வரலாறு - தள மற்றும் அலகு தகவல்

    5. Users - கணினி கணக்குகள் - மின்னஞ்சல்/கடவுச்சொல் அல்லது OAuth2 அங்கீகாரம் - பல-காரணி அங்கீகார ஆதரவு - சபை-குறிப்பிட்ட பாத்திரங்கள்

    6. Roles - அனுமதி ஒதுக்கீடுகள் - மூன்று நிலைகள்: read_only, conductor, administrator - சபை-குறிப்பிட்ட அனுமதிகள் - பாத்திர-அடிப்படையிலான அணுகல் கட்டுப்பாடு

    7. Assignments - வெளியீட்டாளர் அணுகல் டோக்கன்கள் - நேர-வரையறுக்கப்பட்ட அணுகல் இணைப்புகள் - அநாமதேய வயல் தொழிலாளர் அணுகல் - தானியங்கி காலாவதி

    8. Options - முகவரி வகைப்படுத்தல்கள் - சபைக்கு ஏற்ப தனிப்பயனாக்கக்கூடியது - எண்ணக்கூடிய மற்றும் எண்ண முடியாத வகைகள் - முன்னேற்ற கணக்கீடுகளை பாதிக்கிறது

    9. Messages - தகவல்தொடர்புகள் - வெளியீட்டாளர் கருத்துக்கள் - நிர்வாகி அறிவுறுத்தல்கள் - மின்னஞ்சல் அறிவிப்புகள்

    நிலை நிர்வாக உத்தி¶

    நான்கு-அடுக்கு அணுகுமுறை:

    1. Global Context - கருப்பொருள், மொழி, பயன்பாட்டு முறை
    2. Custom Hooks - டொமைன்-குறிப்பிட்ட தர்க்கம் மற்றும் தரவு
    3. Local Component State - UI மாற்றிகள் மற்றும் படிவங்கள்
    4. Persistent State - விருப்பத்தேர்வுகளுக்கு LocalStorage

    நன்மைகள்: - Redux மேல்பட்டை இல்லை - எளிமையான தரவு ஓட்டம் - சிறந்த குறியீடு பிரித்தல் - எளிதான சோதனை

    Middleware Wrapper Sentry ஒருங்கிணைப்பு மையப்படுத்தப்பட்ட பிழை கண்காணிப்பு
    Event Hooks PocketBase வாழ்க்கைச் சுழற்சி தானியங்கி புதுப்பிப்புகள்
    Job Scheduler அம்சக் கொடிகளுடன் Cron திட்டமிடப்பட்ட செயல்பாடுகள்
    Transaction Pattern தரவுத்தள செயல்பாடுகள் தரவு நிலைத்தன்மை

    Aggregate Caching தொகுத்த cron (10 நிமிடம் / 11 நிமிடம் lookback) முன்-கணக்கிடப்பட்ட பகுதி முன்னேற்ற புள்ளிவிவரங்கள்
    Custom Hooks வணிக தர்க்கம் மீண்டும் பயன்படுத்தக்கூடிய தர்க்கம்
    Provider Pattern Context வழங்குநர்கள் உலகளாவிய நிலை
    Lazy Loading Route பிரித்தல் செயல்திறன்
    Policy Pattern அங்கீகாரம் பாதுகாப்பு விதிகள்

    4. Multi-Factor Authentication (MFA) - நிர்வாகிகளுக்கு மேம்பட்ட பாதுகாப்பு

    PocketBase: சுய-ஹோஸ்ட், விற்பனையாளர் lock-in இல்லை, பராமரிக்க எளிதானது

    Wouter: இலகுரக (React Router க்கு 40KB க்கு எதிராக 3KB)

    Bootstrap: போர்-சோதிக்கப்பட்ட, விரிவான கூறுகள், நல்ல அணுகல்தன்மை

    Leaflet: இலவச வரைபடங்கள், API வரம்புகள் இல்லை, இலகுரக

    SQLite: சுயேச்சையானது, தனி தரவுத்தள சர்வர் தேவையில்லை, சுய-ஹோஸ்டிங்கிற்கு சரியானது
