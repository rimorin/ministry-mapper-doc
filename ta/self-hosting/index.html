    Ministry Mapper சொந்தமாக ஹோஸ்ட் செய்தல்¶

    சொந்தமாக ஹோஸ்ட் செய்தல் பரிந்துரைக்கப்படவில்லை

    Ministry Mapper-ஐ சொந்தமாக ஹோஸ்ட் செய்ய கணிசமான தொழில்நுட்ப நிபுணத்துவமும் தொடர்ந்த பராமரிப்பும் தேவை. அதற்கு பதிலாக ministry-mapper.com இல் உள்ள எங்கள் ஹோஸ்ட் செய்யப்பட்ட சேவையை பயன்படுத்த பரிந்துரைக்கிறோம். பின்வருபவை உங்களிடம் இருந்தால் மட்டுமே சொந்தமாக ஹோஸ்ட் செய்வதை தொடரவும்:

    • அனுபவமிக்க சிஸ்டம் நிர்வாகிகள் அல்லது டெவலப்பர்கள்
    • தொடர்ந்த பராமரிப்பு மற்றும் புதுப்பிப்புகளுக்கான வளங்கள்
    • பாதுகாப்பு சிறந்த நடைமுறைகளின் புரிதல்
    • தனியுரிமை விதிமுறைகளை கடைபிடிக்கும் திறன்
    • ஹோஸ்ட் செய்யப்பட்ட சேவையால் பூர்த்தி செய்ய முடியாத குறிப்பிட்ட தேவைகள்

    சொந்தமாக ஹோஸ்ட் செய்வதை நாங்கள் ஊக்குவிக்காத காரணம்¶

    Ministry Mapper திறந்த மூலமாக இருந்தாலும், சொந்தமாக ஹோஸ்ட் செய்வதில் சவால்கள் உள்ளன:

    • தொழில்நுட்ப சிக்கலானது: Docker, தரவுத்தளங்கள், வலை சேவையகங்கள் மற்றும் cloud உள்கட்டமைப்பு பற்றிய அறிவு தேவை
    • பாதுகாப்பு பொறுப்பு: அனைத்தையும் பாதுகாப்பாகவும் புதுப்பித்தும் வைத்திருப்பதற்கு நீங்களே பொறுப்பு
    • பராமரிப்பு சுமை: வழக்கமான புதுப்பிப்புகள், காப்புப்பிரதிகள் மற்றும் கண்காணிப்பு அவசியம்
    • தனியுரிமை இணக்கம்: GDPR, CCPA மற்றும் பிற தனியுரிமை சட்ட இணக்கத்தை நீங்களே உறுதி செய்ய வேண்டும்
    • செலவு: சேவையக செலவுகள், API செலவுகள் மற்றும் நேர முதலீடு பெரும்பாலும் ஹோஸ்ட் செய்யப்பட்ட சேவை கட்டணங்களை மீறும்
    • ஆதரவு: சொந்தமாக ஹோஸ்ட் செய்யும் சிக்கல்களுக்கு சீமித சமூக ஆதரவு

    சொந்தமாக ஹோஸ்ட் செய்வதற்கான முன்நிபந்தனைகள்¶

    எங்கள் பரிந்துரைகளை மீறி சொந்தமாக ஹோஸ்ட் செய்ய முடிவு செய்திருந்தால், பின்வருபவை உங்களிடம் உள்ளதா என உறுதிப்படுத்தவும்:

    தொழில்நுட்ப தேவைகள்¶

    • Linux சேவையக நிர்வாகத்தில் அனுபவம்
    • Docker மற்றும் containerization-ன் புரிதல்
    • சூழல் மாறிகள் மற்றும் கட்டமைப்பு பற்றிய நடைமுறை அறிவு
    • வலை சேவையக கட்டமைப்பின் அறிவு (Nginx/Apache)
    • SSL/TLS சான்றிதழ்களில் அனுபவம்
    • DNS மற்றும் டொமைன் கட்டமைப்பின் புரிதல்

    உள்கட்டமைப்பு தேவைகள்¶

    • Cloud ஹோஸ்டிங் சேவை (Railway, Render, DigitalOcean, AWS போன்றவை)
    • உங்கள் நிகழ்விற்கான டொமைன் பெயர்
    • அறிவிப்புகளுக்கான மின்னஞ்சல் சேவை (விருப்பத்தேர்வு)
    • விருப்பத்தேர்வு API செலவுகளுக்கான பட்ஜெட் (Sentry போன்றவை)

    நேர அர்ப்பணிப்பு¶

    • ஆரம்ப அமைவு: 2-4 மணி நேரம்
    • தொடர்ந்த பராமரிப்பு: மாதம் 2-4 மணி நேரம்
    • சிக்கல்கள் எழும்போது அவசர ஆதரவு

    கட்டமைப்பு கண்ணோட்டம்¶

    Ministry Mapper இரண்டு தனி கூறுகளை கொண்டுள்ளது:

    1. Backend (ministry-mapper-be): Go நீட்டிப்புகளுடன் PocketBase சேவையகம்
    2. இணைப்பி: github.com/rimorin/ministry-mapper-be
    3. தொழில்நுட்பம்: Go, PocketBase, SQLite

    4. போர்ட் 8080 (Docker) அல்லது 8090 (உள்ளூர்) இல் இயங்குகிறது

    5. Frontend (ministry-mapper-v2): React வலை பயன்பாடு

    6. இணைப்பி: github.com/rimorin/ministry-mapper-v2
    7. தொழில்நுட்பம்: React, TypeScript, Vite
    8. ஹோஸ்டிங் சேவையில் பயன்படுத்தப்படும் நிலையான கோப்புகள்

    இரண்டும் பயன்படுத்தப்பட்டு ஒன்றாக வேலை செய்ய கட்டமைக்கப்பட வேண்டும்.

    Backend சொந்த ஹோஸ்டிங் வழிகாட்டி¶

    சூழல் கட்டமைப்பு¶

    இந்த அமைவுகளுடன் .env கோப்பை உருவாக்கவும்:

    # Application Settings
PB_APP_NAME=Ministry Mapper
PB_APP_URL=https://your-frontend-domain.com

# Initial Admin Account (change password after first login!)
PB_ADMIN_EMAIL=admin@your-domain.com
PB_ADMIN_PASSWORD=change_this_secure_password

# Email Configuration (SMTP)
PB_SMTP_HOST=smtp.gmail.com
PB_SMTP_PASSWORD=your_app_password
PB_SMTP_USERNAME=your_email@gmail.com
PB_SMTP_PORT=587
PB_SMTP_SENDER_ADDRESS=noreply@your-domain.com
PB_SMTP_SENDER_NAME=Ministry Mapper

# Security Settings
PB_ALLOW_ORIGINS=https://your-frontend-domain.com
PB_HIDE_CONTROLS=true
PB_ENABLE_RATE_LIMITING=true

# Authentication (optional)
PB_OTP_ENABLED=false
PB_MFA_ENABLED=false

# Error Tracking (optional but recommended)
SENTRY_DSN=your_sentry_dsn
SENTRY_ENV=production
SOURCE_COMMIT=

# Service Integration (optional)
MAILERSEND_API_KEY=
MAILERSEND_FROM_EMAIL=
LAUNCHDARKLY_SDK_KEY=
LAUNCHDARKLY_CONTEXT_KEY=

    Docker பயன்படுத்தல்¶

    1. இணைப்பியை குளோன் செய்யவும் 

      git clone https://github.com/rimorin/ministry-mapper-be.git
cd ministry-mapper-be

    2. Docker படம் உருவாக்கவும் 

      docker build -t ministry-mapper-backend .

    3. Container இயக்கவும் 

      docker run -d \
  -p 8080:8080 \
  -v /path/to/pb_data:/app/pb_data \
  --env-file .env \
  --name ministry-mapper \
  ministry-mapper-backend

    நிர்வாக பலகத்தை அணுக முடியவில்லை: - URL https://your-domain.com/_/ ஆக இருக்க வேண்டும் (அடிக்கோடை கவனிக்கவும்) - PB_HIDE_CONTROLS அமைப்பை சரிபார்க்கவும் - Backend இயங்குகிறதா என சரிபார்க்கவும் - உலாவி cache ஐ அழிக்கவும்

    சொந்தமாக ஹோஸ்ட் செய்பவராக உங்கள் பொறுப்புகள்¶

    சொந்தமாக ஹோஸ்ட் செய்யும்போது, நீங்கள் தரவு கட்டுப்படுத்தியாக இருக்கிறீர்கள் மற்றும் பின்வருவனவற்றை செய்ய வேண்டும்:

    • தனியுரிமை சட்டங்களை கடைபிடிக்கவும்: GDPR (EU), CCPA (California) போன்றவை
    • தனியுரிமை கொள்கை உருவாக்கவும்: தரவு சேகரிப்பு மற்றும் பயன்பாட்டை விளக்கவும்
    • சேவை விதிமுறைகள் உருவாக்கவும்: பயனர் பொறுப்புகளை வரையறுக்கவும்
    • தரவு பாதுகாப்பை செயல்படுத்தவும்: குறியாக்கம், அணுகல் கட்டுப்பாடுகள், காப்புப்பிரதிகள்
    • தரவு கோரிக்கைகளை கையாளவும்: பயனர் அணுகல், நீக்கம், பெயர்வு உரிமைகள்
    • தரவு மீறல்களை தெரிவிக்கவும்: சட்டப்பூர்வ காலக்கெடுவிற்குள்
    • பதிவுகளை பராமரிக்கவும்: தரவு செயலாக்க செயல்பாடுகள்
    • DPO நியமிக்கவும்: விதிமுறைகளால் தேவைப்பட்டால்

    10. பொறுப்பு வரம்புகள்

    11. நிறுத்தல் கொள்கைகள்

    12. Cookie கொள்கை - நீங்கள் cookies/கண்காணிப்பு பயன்படுத்தினால்

    தரவு பாதுகாப்பு நடவடிக்கைகள்¶

    • எல்லா இடத்திலும் HTTPS பயன்படுத்தவும்
    • ஓய்வில் உள்ள தரவை குறியாக்கம் செய்யவும்
    • அணுகல் கட்டுப்பாடுகளை செயல்படுத்தவும்
    • வழக்கமான பாதுகாப்பு தணிக்கைகள்
    • காப்புப்பிரதி குறியாக்கம்
    • பாதுகாப்பான கடவுச்சொல் கொள்கைகள்
    • பல்-காரணி அங்கீகரணம்
    • செயல்பாடு பதிவு
    • வழக்கமான பாதுகாப்பு புதுப்பிப்புகள்

    செலவு பரிசீலனைகள்¶

    உள்கட்டமைப்பு செலவுகள்¶

    Backend ஹோஸ்டிங்: - VPS: $10-50/மாதம் (DigitalOcean, Linode) - PaaS: $10-30/மாதம் (Railway, Render) - Cloud: மாறுபடும் (AWS, GCP)

    Frontend ஹோஸ்டிங்: - நிலையான ஹோஸ்டிங்: $0-10/மாதம் (Vercel, Netlify, Cloudflare) - CDN செலவுகள்: பொதுவாக உள்ளடங்கியது

    டொமைன்: - $10-20/ஆண்டு

    SSL சான்றிதழ்: - இலவசம் (Let's Encrypt) அல்லது $0-100/ஆண்டு

    சேவை செலவுகள்¶

    மின்னஞ்சல் சேவை: - Gmail SMTP: இலவசம் (வரம்பிட்டது) - MailerSend: இலவச அடுக்கு கிடைக்கும் - SendGrid: இலவச அடுக்கு கிடைக்கும்

    பிழை கண்காணிப்பு: - Sentry: இலவச அடுக்கு கிடைக்கும் - கட்டணம்: $26-80/மாதம்

    காப்புப்பிரதி சேமிப்பு: - அளவைப் பொறுத்து $5-20/மாதம்

    நேர முதலீடு¶

    ஆரம்ப அமைவு: - Backend: 2-3 மணி நேரம் - Frontend: 1-2 மணி நேரம் - கட்டமைப்பு: 1-2 மணி நேரம் - சோதனை: 2-3 மணி நேரம் - மொத்தம்: 6-10 மணி நேரம்

    மாதாந்திர பராமரிப்பு: - கண்காணிப்பு: 1-2 மணி நேரம் - புதுப்பிப்புகள்: 1-2 மணி நேரம் - ஆதரவு: 1-3 மணி நேரம் - மொத்தம்: 3-7 மணி நேரம்

    மொத்த உரிமைச் செலவு (வருடாந்திரம்)¶

    சிறிய சபை (< 100 பயனர்கள்): - உள்கட்டமைப்பு: $240-720 - API-கள்: $0-100 (பொதுவாக இலவச அடுக்கு) - சேவைகள்: $0-300 - டொமைன்/SSL: $10-20 - மொத்தம்: $250-1,140 + 48-96 மணி நேரம்/ஆண்டு

    அர்ப்பணிப்பதற்கு முன் இதை ஒரு ஹோஸ்ட் செய்யப்பட்ட சேவை சந்தாவுடன் ஒப்பிடவும்.

    உதவி பெறுவது எங்கே¶

    சமூக ஆதரவு¶

    • GitHub Issues: ministry-mapper-v2 மற்றும் ministry-mapper-be
    • புதிய issues உருவாக்குவதற்கு முன் ஏற்கனவே உள்ள issues படிக்கவும்
    • சிக்கல்களை தெரிவிக்கும்போது விரிவான தகவலை வழங்கவும்

    ஆதரவு கோரிக்கைகளில் சேர்க்க வேண்டியவை¶

    1. பதிப்பு எண்கள் (backend மற்றும் frontend)
    2. பயன்படுத்தல் சூழல் (Docker, உள்ளூர், ஹோஸ்டிங் சேவை)
    3. பிழை செய்திகள் (முழு உரை)
    4. மீண்டும் நிகழ்த்தும் படிகள்
    5. எதிர்பார்க்கப்பட்ட vs உண்மையான நடத்தை
    6. தொடர்புடைய பதிவுகள்
    7. கட்டமைப்பு (சுத்திகரிக்கப்பட்டது, ரகசியங்கள் இல்லாமல்)

    இதற்கு பதிலாக ஹோஸ்ட் செய்யப்பட்ட சேவையை பயன்படுத்தவும்¶

    ministry-mapper.com ஐ பார்வையிடவும்: - உடனடி அமைவு - தொழில்நுட்ப அறிவு தேவையில்லை - தேவைப்படும்போது தொழில்முறை ஆதரவு - தானியங்கி காப்புப்பிரதிகள் மற்றும் பாதுகாப்பு புதுப்பிப்புகள் - சட்ட இணக்க உதவி - சிறந்த நம்பகத்தன்மை மற்றும் இயக்க நேரம் - குறைந்த மொத்த உரிமைச் செலவு
