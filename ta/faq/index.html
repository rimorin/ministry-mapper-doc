    அடிக்கடி கேட்கப்படும் கேள்விகள் (FAQ)¶

    பொதுவான கேள்விகள்¶

    Ministry Mapper என்றால் என்ன?¶

    Ministry Mapper என்பது ஒரு நவீன இணைய பயன்பாடாகும், இது சபைகள் மண்டல ஊழியப் பகுதிகளை டிஜிட்டல் முறையில் நிர்வகிக்க உதவுகிறது. காகித பகுதி சீட்டுகளுக்கு பதிலாக, Ministry Mapper இணையதளத்தில் இணைப்பு உள்ள எந்த சாதனத்திலிருந்தும் எல்லாவற்றையும் டிஜிட்டல் முறையில் ஒழுங்கமைக்கவும் கண்காணிக்கவும் உங்களுக்கு உதவுகிறது. இது முன்-முனைக்கு React, பின்-முனைக்கு PocketBase மற்றும் ஊடாடும் வரைப்படத்திற்கு Leaflet பயன்படுத்துகிறது.

    Ministry Mapper ஐ எவ்வாறு அணுகுவது?¶

    பரிந்துரைக்கப்பட்டது: ministry-mapper.com இல் எங்கள் நேர சேவையை பயன்படுத்தவும்

    எளிமையாக: 1. ministry-mapper.com ஐ பார்வையிடவும் 2. பதிவு பக்கத்தைப் பயன்படுத்தி ஒரு கணக்கை உருவாக்கவும் 3. உங்கள் மின்னஞ்சல் முகவரியை சரிபார்க்கவும் 4. சரியான அனுமதிகளுக்கு உங்கள் சபையின் நிர்வாகியை தொடர்புகொள்ளவும்

    மாற்று: சுய-நேர்விடல் (பரிந்துரைக்கப்படவில்லை) - விவரங்களுக்கு Self-Hosting Guide ஐ பாருங்கள். சுய-நேர்விடலுக்கு குறிப்பிடத்தக்க தொழில்நுட்ப நிபுணத்துவம் மற்றும் தொடர்ந்த பராமரிப்பு தேவைப்படுகிறது.

    Ministry Mapper பயன்படுத்த இலவசமா?¶

    நேர சேவை: விலை நிர்ணயம் மற்றும் திட்டங்களுக்கு ministry-mapper.com ஐ பார்வையிடவும்.

    சுய-நேர்விடல்: மூல குறியீடு இலவசமாகவும் திறந்த மூலமாகவும் உள்ளது, ஆனால் நீங்கள் உங்களுடையது வழங்க வேண்டும்: - முன்-முனைக்கு நேர்விடல் (எ.கா., Vercel, Netlify, AWS) - PocketBase பின்-முனை வரிசைப்படுத்தல் - விருப்பமானது: பிழை கண்காணிப்பிற்கான Sentry கணக்கு

    குறிப்பு: சுய-நேர்விடல் செலவுகள் (சேவையகம், API கட்டணங்கள், பராமரிப்பு நேரம்) பெரும்பாலும் நேர சேவை கட்டணங்களை விட அதிகமாக இருக்கும்.

    Ministry Mapper என்ன மொழிகளை ஆதரிக்கிறது?¶

    தற்போது ஆதரிக்கப்படும் மொழிகள்:

    • English (ஆங்கிலம்)
    • Japanese (日本語)
    • Korean (한국語)
    • Chinese (中文)
    • Indonesian (Bahasa Indonesia)
    • Malay (Bahasa Melayu)
    • Spanish (Español)
    • தமிழ் (Tamil)

    இடைமுகம் தானாகவே உங்கள் உலாவியின் மொழியை கண்டறியும். பயன்பாட்டில் கைமுறையாக மொழிகளை மாற்றவும் முடியும்.

    Ministry Mapper பயன்படுத்த தொழில்நுட்ப அறிவு தேவையா?¶

    பதிப்பாளராக/பயனராக: தொழில்நுட்ப அறிவு தேவையில்லை! ஸ்மார்ட்போன் அல்லது கணினியை பயன்படுத்த முடிந்தால், ministry-mapper.com இல் நேர சேவை மூலம் Ministry Mapper ஐ பயன்படுத்தலாம்.

    சுய-நேர்விடல் நிர்வாகியாக: ஆம், குறிப்பிடத்தக்க தொழில்நுட்ப அறிவு தேவைப்படுகிறது:

    • Linux சேவையக நிர்வாகத்தில் அனுபவம்
    • Docker மற்றும் கண்டெய்னரைசேஷன் புரிந்துகொள்ளல்
    • சுற்றுச்சூழல் மாறிகள் மற்றும் கட்டமைப்பு பற்றிய அறிவு
    • SSL/TLS சான்றிதழ்களுடன் பரிச்சயம்
    • இணைய சேவையக கட்டமைப்பு (Nginx/Apache)
    • DNS மற்றும் டொமைன் கட்டமைப்பு

    சுய-நேர்விடலுக்கு பதிலாக நேர சேவையை பயன்படுத்துமாறு நாங்கள் கடுமையாக பரிந்துரைக்கிறோம். நீங்கள் இன்னும் சுய-நேர்விட விரும்பினால் முழு விவரங்களுக்கு Self-Hosting Guide ஐ பாருங்கள்.

    Ministry Mapper ஆஃப்லைனில் வேலை செய்யுமா?¶

    இல்லை, பயன்பாடு சரியாக செயல்பட இணைய இணைப்பு தேவைப்படுகிறது. ஏனெனில்:

    • PocketBase பின்-முனையுடன் இணைக்க வேண்டும்
    • பயனர்களிடையே நேரடி புதுப்பிப்புகளுக்கு இணைப்பு தேவைப்படுகிறது

    WiFi கிடைக்காத நிலையில் மொபைல் தரவில் பயன்பாடு நன்றாக செயல்படுகிறது.

    தனியுரிமை மற்றும் சட்டம்¶

    நான் என்ன தனியுரிமை சட்டங்களை கருத்தில் கொள்ள வேண்டும்?¶

    முக்கியம்: Ministry Mapper குடியிருப்பு முகவரிகளை கண்காணிக்கிறது, இது தரவு தனியுரிமை சட்டங்களுக்கு உட்பட்டிருக்கலாம். இந்த சட்டங்கள் நாடுகள் மற்றும் பகுதிகளிடையே கணிசமாக வேறுபடுகின்றன:

    • ஐரோப்பா: GDPR (பொது தரவு பாதுகாப்பு ஒழுங்காற்று)
    • கலிபோர்னியா: CCPA (கலிபோர்னியா நுகர்வோர் தனியுரிமை சட்டம்)
    • பிரேசில்: LGPD (Lei Geral de Proteção de Dados)
    • பல மற்றவை: பல்வேறு உள்ளூர் ஒழுங்குமுறைகள்

    Ministry Mapper ஐ பயன்படுத்துவதற்கு முன்பு உங்கள் உள்ளூர் ஒழுங்குமுறைகளை முழுமையாக மதிப்பாய்வு செய்து இணக்கத்தை உறுதிசெய்யவும். உங்கள் பகுதிக்கு குறிப்பிட்ட சரியான சட்ட ஆலோசனைக்கு ஒரு சட்ட நிபுணரை அணுகவும்.

    நேர சேவை: ministry-mapper.com இல் நேர சேவை இணக்க உதவியை வழங்கலாம், ஆனால் உங்கள் உள்ளூர் சட்டங்களை பின்பற்றுவதற்கு நீங்கள் பொறுப்பாளி.

    சுய-நேர்விடல்: அனைத்து தனியுரிமை சட்ட இணக்கத்திற்கும் நீங்கள் முழு பொறுப்பாளி.

    Ministry Mapper என்ன தரவை சேமிக்கிறது?¶

    பயனர் தரவு (PocketBase நிர்வகிக்கப்படுகிறது):

    • மின்னஞ்சல் முகவரி
    • பெயர்
    • கடவுச்சொல் (மறைகுறியாக்கப்பட்டது)
    • பயனர் பங்குகள் மற்றும் அனுமதிகள்

    பகுதி தரவு:

    • பகுதி எல்லைகள் மற்றும் பெயர்கள்
    • முகவரிகள்
    • நிலை தகவல்
    • குடும்பத்தினர் பற்றிய குறிப்புகள்
    • பணி வரலாறு
    • புதுப்பிப்பு நேரமுத்திரைகள்

    • பிழை பதிவுகள்
    • செயல்திறன் அளவீடுகள்

    அனைத்து தரவு சேமிப்பும் பாதுகாப்பு சிறந்த நடைமுறைகளுக்கு இணங்கி இருக்கிறது. நேர சேவை தரவு கொள்கைகளுக்கு, ministry-mapper.com ஐ சரிபார்க்கவும்.

    தொழில்நுட்ப அமைவு (சுய-நேர்விடல்)¶

    பின்வரும் பிரிவுகள் Ministry Mapper ஐ சுய-நேர்விட விரும்பும் மேம்பட்ட பயனர்களுக்கானவை. ministry-mapper.com இல் நேர சேவையை பயன்படுத்துமாறு நாங்கள் கடுமையாக பரிந்துரைக்கிறோம். சுய-நேர்விடலுக்கு குறிப்பிடத்தக்க தொழில்நுட்ப நிபுணத்துவம், தொடர்ந்த பராமரிப்பு மற்றும் பாதுகாப்பு பொறுப்புகள் தேவைப்படுகின்றன.

    முழுமையான சுய-நேர்விடல் வழிமுறைகளுக்கு, Self-Hosting Guide ஐ பாருங்கள்.

    கணினி தேவைகள் என்ன?¶

    நேர சேவையை பயன்படுத்துவதற்கு: - நவீன இணைய உலாவி (Chrome, Firefox, Safari, Edge) - இணைய இணைப்பு - மின்னஞ்சல் முகவரி - மொபைல் சாதனம் அல்லது கணினி

    சுய-நேர்விடல் மேம்பாட்டிற்கு:

    • Node.js >= 22.0.0
    • npm தொகுப்பு நிர்வாகி
    • Git
    • Docker (பின்-முனைக்கு)

    சுய-நேர்விடல் உற்பத்திக்கு:

    • கிளவுட் நேர்விடல் சேவை (Railway, Render, DigitalOcean, AWS, முதலியன)
    • டொமைன் பெயர்
    • SSL/TLS சான்றிதழ்
    • PocketBase பின்-முனை (தனி வரிசைப்படுத்தல்)
    • (விருப்பமானது) கண்காணிப்பிற்கான Sentry கணக்கு
    • (விருப்பமானது) அறிவிப்புகளுக்கான மின்னஞ்சல் சேவை

    என்ன சுற்றுச்சூழல் மாறிகள் தேவைப்படுகின்றன?¶

    சுய-நேர்விடலுக்கு மட்டும் - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இந்த பிரிவு பொருந்தாது. நேர சேவை அனைத்து கட்டமைப்பையும் தானியங்கியாக கையாளுகிறது.

    முன்-முனை (.env கோப்பு):

    VITE_SYSTEM_ENVIRONMENT=local
VITE_VERSION=$npm_package_version
VITE_POCKETBASE_URL=your_pocketbase_backend_url
VITE_OPENROUTE_API_KEY=your_openrouteservice_key
VITE_LOCATIONIQ_API_KEY=your_locationiq_key
VITE_SENTRY_DSN=your_sentry_dsn (optional)
VITE_PRIVACY_URL=your_privacy_policy_url
VITE_TERMS_URL=your_terms_url
VITE_ABOUT_URL=your_about_page_url

    உற்பத்திக்கு: மேலே உள்ளது போன்ற அதே மாறிகள், ஆனால் VITE_SYSTEM_ENVIRONMENT=production உடன்

    முழு விவரங்களுக்கு Frontend Setup Guide ஐ பாருங்கள்.

    முன்-முனையை எவ்வாறு வரிசைப்படுத்துவது?¶

    சுய-நேர்விடலுக்கு மட்டும் - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இதை தவிர்க்கவும்.

    1. பயன்பாட்டை உருவாக்கவும்:
    npm run build

    1. மேலே குறிப்பிட்டபடி சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும்

    2. உங்கள் நேர்விடல் வழங்குனருக்கு build/ கோப்புறையை வரிசைப்படுத்தவும்:

    3. Vercel: உங்கள் GitHub repo ஐ இணைத்து சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும்

    4. Netlify: build கோப்புறையை இழுத்து விடவும் அல்லது Git வழியாக இணைக்கவும்

    5. AWS S3: build கோப்புறையை பதிவேற்றி நிலையான வலைத்தள நேர்விடலை கட்டமைக்கவும்

    6. PocketBase பின்-முனை வரிசைப்படுத்தப்பட்டுள்ளது மற்றும் VITE_POCKETBASE_URL இல் குறிப்பிட்ட URL இல் அணுகக்கூடியதாக இருப்பதை உறுதிசெய்யவும்

    விரிவான வழிமுறைகளுக்கு Frontend Setup Guide மற்றும் Self-Hosting Guide ஐ பாருங்கள்.

    சுய-நேர்விடலுக்கு மட்டும் - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இதை தவிர்க்கவும்.

    2. சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும் (Backend Setup Guide ஐ பாருங்கள்)

    4. திட்டத்துடன் தரவுத்தளத்தை துவக்கவும்
    5. மின்னஞ்சல் அறிவிப்புகளுக்கு SMTP ஐ கட்டமைக்கவும் (விருப்பமானது)

    விரிவான வழிமுறைகள்: Backend Setup Guide மற்றும் Self-Hosting Guide ஐ பாருங்கள்.

    1. Sentry கணக்கை உருவாக்கவும்
    2. ஒரு React திட்டத்தை உருவாக்கவும்
    3. அமைப்புகளுக்கு சென்று DSN key ஐ பெறவும்
    4. பின்வரும் சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும்:
    5. VITE_SENTRY_DSN: உங்கள் Sentry திட்ட DSN
    6. VITE_SYSTEM_ENVIRONMENT: உற்பத்திக்கு "production" என அமைக்கவும் (டிரேசிங் மாதிரி விகிதத்தை பாதிக்கிறது)
    7. VITE_VERSION: Sentry இல் வெளியீட்டு கண்காணிப்பிற்கு பயன்படுத்தப்படுகிறது

    குறிப்பு: Sentry விருப்பமானது ஆனால் உற்பத்தி வரிசைப்படுத்தல்களுக்கு பரிந்துரைக்கப்படுகிறது.

cd ministry-mapper-v2

    பயன்பாடு "Cannot connect to backend" காட்டுகிறது¶

    நேர சேவை பயனர்களுக்கு: இந்த பிழையை நீங்கள் பார்த்தால், சேவை சிக்கல்களை எதிர்கொள்ளலாம். நிலை பக்கத்தை சரிபார்க்கவும் அல்லது ஆதரவை தொடர்புகொள்ளவும்.

    சுய-நேர்விடல் பயனர்களுக்கு:

    சாத்தியமான காரணங்கள்:

    1. PocketBase பின்-முனை இயங்கவில்லை அல்லது அணுகக்கூடியதாக இல்லை
    2. VITE_POCKETBASE_URL சுற்றுச்சூழல் மாறி தவறானது
    3. CORS சிக்கல்கள் (பின்-முனை முன்-முனை டொமைனை அனுமதிக்கவில்லை)

    தீர்வுகள்:

    • பின்-முனை இயங்குகிறது மற்றும் அணுகக்கூடியதாக இருப்பதை சரிபார்க்கவும்
    • உங்கள் சுற்றுச்சூழல் மாறிகளில் URL ஐ சரிபார்க்கவும்
    • உங்கள் முன்-முனை டொமைனை அனுமதிக்க PocketBase இல் CORS அமைப்புகளை கட்டமைக்கவும்

    மீண்டும் உருவாக்கிய பிறகு மாற்றங்கள் பிரதிபலிக்கவில்லை¶

    சுய-நேர்விடலுக்கு மட்டும்:

    சாத்தியமான காரணங்கள்:

    1. உலாவி தற்காலிக சேமிப்பு
    2. CDN தற்காலிக சேமிப்பு (பயன்படுத்துகிறீர்கள் என்றால்)
    3. Build artifacts சரியாக வரிசைப்படுத்தப்படவில்லை

    தீர்வுகள்:

    • உலாவி தற்காலிக சேமிப்பை அழிக்கவும் அல்லது incognito பயன்முறையை பயன்படுத்தவும்
    • பொருந்தும் பட்சத்தில் CDN தற்காலிக சேமிப்பை செல்லாது செய்யவும்
    • உங்கள் நேர்விடல் சேவையில் build கோப்புறை முழுமையாக மாற்றப்பட்டது என்பதை சரிபார்க்கவும்
    • பிழைகளுக்கு உலாவி console ஐ சரிபார்க்கவும்

    பயனர் அங்கீகாரம் வேலை செய்யவில்லை¶

    நேர சேவை பயனர்களுக்கு: அங்கீகார சிக்கல்கள் இருந்தால் ஆதரவை தொடர்புகொள்ளவும்.

    சுய-நேர்விடல் பயனர்களுக்கு:

    சாத்தியமான காரணங்கள்:

    1. PocketBase பின்-முனை பயனர் நிர்வாகம் கட்டமைக்கப்படவில்லை
    2. நெட்வொர்க் இணைப்பு சிக்கல்கள்
    3. தவறான PocketBase URL
    4. CORS கட்டமைப்பு சிக்கல்கள்

    தீர்வுகள்:

    • PocketBase பின்-முனை சரியாக கட்டமைக்கப்பட்டுள்ளது என்பதை சரிபார்க்கவும்
    • பிழை செய்திகளுக்கு உலாவி console ஐ சரிபார்க்கவும்
    • பின்-முனை API endpoints ஐ நேரடியாக சோதிக்கவும்
    • PocketBase இல் CORS அமைப்புகளை சரிபார்க்கவும்

    பயன்பாடு மெதுவாக அல்லது பதிலளிக்காமல் உள்ளது¶

    சாத்தியமான காரணங்கள்:

    1. பெரிய பகுதி தரவுத்தொகுப்புகள்
    2. நெட்வொர்க் தாமதம்
    3. போதுமான சேவையக வளங்கள் இல்லை (சுய-நேர்விடல்)
    4. உலாவி செயல்திறன் சிக்கல்கள்

    தீர்வுகள்:

    • உங்கள் இணைய இணைப்பை சரிபார்க்கவும்
    • வேறு உலாவியை பயன்படுத்த முயற்சிக்கவும்
    • சுய-நேர்விடலுக்கு: பகுதி தரவை மேம்படுத்துங்கள் (தேவையற்ற புலங்களை குறைக்கவும்)
    • சுய-நேர்விடலுக்கு: அருகிலுள்ள நேர்விடல் பகுதியை பயன்படுத்துங்கள்
    • சுய-நேர்விடலுக்கு: தேவைப்பட்டால் சேவையக வளங்களை மேம்படுத்துங்கள்
    • உலாவி தற்காலிக சேமிப்பு மற்றும் தரவை அழிக்கவும்
    • செயல்திறன் எச்சரிக்கைகளுக்கு உலாவி console ஐ சரிபார்க்கவும்

    பங்களிப்பு¶

    பிழையை எவ்வாறு புகாரளிப்பது?¶

    1. GitHub Issues இல் பிழை ஏற்கனவே புகாரளிக்கப்பட்டுள்ளதா என சரிபார்க்கவும்
    2. இல்லையென்றால், பின்வருவனவற்றுடன் புதிய issue ஐ உருவாக்கவும்:
    3. சிக்கலின் தெளிவான விளக்கம்
    4. மறு உருவாக்கும் படிகள்
    5. எதிர்பார்க்கப்பட்ட vs உண்மையான நடத்தை
    6. உலாவி மற்றும் சாதன தகவல்
    7. திரைப்பிடிப்புகள் அல்லது பிழை செய்திகள்
    8. உங்கள் சுற்றுச்சூழல் கட்டமைப்பு (உணர்திறன் தரவு இல்லாமல்)

    அம்சங்களை கோரலாமா?¶

    ஆம்! GitHub இல் அம்சக் கோரிக்கையை உருவாக்குங்கள்:

    • அம்சத்தின் தெளிவான விளக்கம்
    • பயன்பாட்டு வழக்கு மற்றும் நன்மைகள்
    • அது எவ்வாறு வேலை செய்யும்
    • யார் பயனடைவார்கள்

    குறிப்பு: இது தன்னார்வலர்கள் பராமரிக்கும் திட்டம், எனவே செயலாக்கம் கிடைக்கும் நேரம் மற்றும் வளங்களை பொறுத்தது.

    குறியீட்டை பங்களிக்கலாமா?¶

    நிச்சயமாக! பங்களிப்புகள் வரவேற்கப்படுகின்றன:

    எவ்வாறு பங்களிப்பது:

    1. களஞ்சியத்தை fork செய்யவும்
    2. ஒரு அம்சக் கிளையை உருவாக்கவும்
    3. ஏற்கனவே உள்ள குறியீடு நடை பின்பற்றி மாற்றங்களை செய்யவும்
    4. புதிய செயல்பாட்டிற்கு சோதனைகளை எழுதவும்
    5. தரத்தை உறுதிசெய்ய npm test மற்றும் npm run prettier இயக்கவும்
    6. தெளிவான விளக்கத்துடன் pull request சமர்பிக்கவும்

    பங்களிப்பு வழிகாட்டுதல்கள்:

    • TypeScript மற்றும் React சிறந்த நடைமுறைகளை பின்பற்றவும்
    • குறியீட்டுத் தளத்தில் ஏற்கனவே உள்ள முறைகளை பயன்படுத்தவும்
    • தேவைப்பட்டால் ஆவணங்களை புதுப்பிக்கவும்
    • மாற்றங்களை கவனமாகவும் குறைந்தபட்சமாகவும் வைத்திருங்கள்
    • சமர்பிப்பதற்கு முன்பு முழுமையாக சோதிக்கவும்

    கட்டமைப்பு விவரங்கள் மற்றும் குறியீட்டு மரபுகளுக்கு CLAUDE.md ஐ பாருங்கள்.

    பயன்பாட்டை புதிய மொழியில் மொழிபெயர்க்கலாமா?¶

    ஆம்! பயன்பாடு மொழிபெயர்ப்புகளுக்கு சர்வதேசமயமாக்கல் (i18n) பயன்படுத்துகிறது:

    1. ஏற்கனவே உள்ள மொழிபெயர்ப்புகளுக்கு முன்-முனை களஞ்சியத்தில் src/i18n/ கோப்பகத்தை சரிபார்க்கவும்
    2. ஏற்கனவே உள்ள கட்டமைப்பை பின்பற்றி புதிய மொழி கோப்பை உருவாக்கவும்
    3. உங்கள் மொழிபெயர்ப்புகளை சேர்க்கவும்
    4. மொழி தேர்ந்தெடுப்பு கூறை புதுப்பிக்கவும்
    5. pull request சமர்பிக்கவும்

    புதிய பதிப்பிற்கு எவ்வாறு மேம்படுத்துவது?¶

    நேர சேவை பயனர்களுக்கு: புதுப்பிப்புகள் தானியங்கி. நீங்கள் எதுவும் செய்ய தேவையில்லை!

    3. பயனர் வழிகாட்டி
    4. சுய-நேர்விடல் வழிகாட்டி (சுய-நேர்விட்டிருந்தால்)
    5. பின்-முனை அமைவு வழிகாட்டி (சுய-நேர்விட்டிருந்தால்)
    6. முன்-முனை அமைவு வழிகாட்டி (சுய-நேர்விட்டிருந்தால்)
    7. GitHub Issues இல் தேடுங்கள்
    8. GitHub Discussions இல் கேளுங்கள்
    9. ஆதரவை தொடர்புகொள்ளுங்கள் (நேர சேவை பயனர்களுக்கு)
    10. உங்கள் கேள்வியுடன் புதிய issue ஐ திறக்கவும்

    நினைவில் கொள்ளுங்கள்: Ministry Mapper திறந்த மூலமாகும் மற்றும் தன்னார்வலர்களால் பராமரிக்கப்படுகிறது. உதவி கேட்கும்போது பொறுமையாகவும் மரியாதையாகவும் இருங்கள்!
