    பின்னணி அமைவு வழிகாட்டி¶

    சுய-தொகுப்பு பரிந்துரைக்கப்படவில்லை

    இந்த வழிகாட்டி Ministry Mapper-ஐ சுய-தொகுக்க விரும்பும் மேம்பட்ட பயனர்களுக்கானது. தொழில்நுட்ப சிக்கலான தன்மை, பராமரிப்பு சுமை மற்றும் பாதுகாப்பு பொறுப்புகள் காரணமாக சுய-தொகுப்பை நாங்கள் ஊக்குவிக்கவில்லை.

    பெரும்பாலான பயனர்கள்: ministry-mapper.com இல் எங்கள் நேரடி சேவையை பயன்படுத்தவும்.

    பயனர் ஆவணங்களை தேடுகிறீர்களா?

    Ministry Mapper-ஐ எவ்வாறு பயன்படுத்துவது என்று கற்றுக்கொள்ள விரும்பும் சாதாரண பயனராக இருந்தால், பதிலாக தொடங்குதல் அல்லது பயனர் வழிகாட்டி பார்க்கவும்.

    கண்ணோட்டம்¶

    Ministry Mapper பின்னணி (ministry-mapper-be) Go மற்றும் PocketBase மூலம் கட்டமைக்கப்பட்டுள்ளது. இது வழங்குகிறது:

    • தரவு சேமிப்பு மற்றும் மேலாண்மை (SQLite database)
    • பயனர் அங்கீகாரம் மற்றும் அங்கீகாரம்
    • நிகழ்நேர தரவு ஒத்திசைவு
    • பகுதி மேலாண்மைக்கான தனிப்பயன் API endpoints
    • தானியங்கி பணிகளுக்கான பின்னணி jobs
    • SMTP மூலம் மின்னஞ்சல் அறிவிப்புகள்

    இந்த வழிகாட்டி இவற்றை அறிந்திருக்கும் என்று கருதுகிறது: - சேவையக நிர்வாக அனுபவம் - Docker மற்றும் containerization புரிதல் - சூழல் மாறிகள் பற்றிய நெறிமுறை - பாதுகாப்பு சிறந்த நடைமுறைகள் அறிவு

    முழுமையான சுய-தொகுப்பு வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    விரைவு குறிப்பு¶

    இந்தப் பக்கம் பின்னணி உள்ளமைவுக்கான விரைவு குறிப்பை வழங்குகிறது. முழுமையான அமைவு வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    தொழில்நுட்ப அடுக்கு¶

    • Go: வேகமான, திறமையான நிரலாக்க மொழி
    • PocketBase: உள்ளமைக்கப்பட்ட admin panel உடன் Backend-as-a-service
    • SQLite: pb_data கோப்பகத்தில் இலகுரக database
    • Docker: Alpine Linux அடிப்படையிலான container

    முக்கிய அம்சங்கள்¶

    • தரவு சேமிப்பு மற்றும் மேலாண்மை
    • பயனர் அங்கீகாரம்
    • நிகழ்நேர subscriptions (Server-Sent Events)
    • தனிப்பயன் API endpoints
    • திட்டமிடப்பட்ட பின்னணி jobs
    • மின்னஞ்சல் அறிவிப்புகள் (SMTP)
    • பிழை கண்காணிப்பு (Sentry)
    • அம்சக் கொடிகள் (LaunchDarkly)

    முழுமையான அமைவு வழிமுறைகள்

    கீழ்கண்ட பகுதிகள் பின்னணி உள்ளமைவுக்கான தொழில்நுட்ப குறிப்பை வழங்குகின்றன. சூழல் மற்றும் விளக்கங்களுடன் வரிசைமுறை அமைவு வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    சூழல் மாறிகள் விளக்கம்¶

    சேவை ஒருங்கிணைப்பு விசைகள் (விருப்பமானது)¶

    # MailerSend API key - MailerSend சேவை மூலம் மின்னஞ்சல் அனுப்புவதற்கு
# SMTP பயன்படுத்தினால் காலியாக விடவும்
MAILERSEND_API_KEY=

# MailerSend அனுப்புநரின் மின்னஞ்சல் முகவரி
MAILERSEND_FROM_EMAIL=

# LaunchDarkly SDK key - பின்னணி jobs-ஐ கட்டுப்படுத்தும் அம்சக் கொடிகளுக்கு
# உள்ளமைக்காவிட்டால் jobs இயல்பாக இயங்கும்
LAUNCHDARKLY_SDK_KEY=
LAUNCHDARKLY_CONTEXT_KEY=

# OpenAI API key - மாதாந்திர அறிக்கைகள் மற்றும் செய்தி சுருக்கங்களில் AI-உருவாக்கிய சுருக்கங்களுக்கு
# விருப்பமானது: இல்லாமலும் அறிக்கைகள் மற்றும் சுருக்கங்கள் நன்றாக வேலை செய்யும்; AI பகுதிகள் மட்டும் தவிர்க்கப்படும்
OPENAI_API_KEY=

    மின்னஞ்சல் (MailerSend)¶

    Variable Required Description
    MAILERSEND_API_KEY Optional பரிவர்த்தனை மின்னஞ்சல் வழங்கலுக்கான MailerSend API key (அறிக்கைகள் மற்றும் lifecycle மின்னஞ்சல்களுக்கு பயன்படுகிறது). அமைக்காவிட்டால் SMTP-க்கு திரும்பும்.
    MAILERSEND_FROM_EMAIL Optional MailerSend மின்னஞ்சல்களுக்கான அனுப்புநர் மின்னஞ்சல் முகவரி.

    அம்சக் கொடிகள் (LaunchDarkly)¶

    Variable Required Description
    LAUNCHDARKLY_SDK_KEY Optional LaunchDarkly server-side SDK key. அம்சக் கொடியால் கட்டுப்படுத்தப்படும் பின்னணி jobs பயன்படுத்தினால் தேவை.

    நிறுவல் வழிமுறைகள்¶

    விருப்பம் 1: Docker பயன்படுத்தல் (பரிந்துரைக்கப்படுகிறது)¶

    Docker பயன்படுத்தலை எளிதாகவும் தளங்களில் ஒரேசீராகவும் செய்கிறது.

    1. Docker படத்தை உருவாக்கவும்¶

    docker build -t ministry-mapper-backend .

    2. Container-ஐ இயக்கவும்¶

    docker run -d \
  -p 8080:8080 \
  -v /path/to/pb_data:/app/pb_data \
  --env-file .env \
  --name ministry-mapper \
  ministry-mapper-backend

    முக்கிய குறிப்புகள்:

    • Container 8080 port இல் இயங்குகிறது (8090 அல்ல)
    • -p 8080:8080 container-இன் port 8080-ஐ உங்கள் host port 8080-க்கு இணைக்கிறது
    • -v /path/to/pb_data:/app/pb_data உங்கள் database-ஐ நிரந்தரமாக சேமிக்கிறது (/path/to/pb_data-ஐ உங்கள் சிஸ்டத்தில் உள்ள உண்மையான பாதையுடன் மாற்றவும்)
    • --env-file .env உங்கள் சூழல் மாறிகளை .env கோப்பிலிருந்து ஏற்றுகிறது
    • -d container-ஐ பின்னணியில் (detached mode) இயக்குகிறது

    விருப்பம் 2: உள்ளூர் மேம்பாடு¶

    உங்கள் கணினியில் மேம்பாடு மற்றும் சோதனைக்கு:

    1. Go சார்புகளை நிறுவவும்¶

    ./scripts/install.sh

    இந்த script அனைத்து சார்புகளையும் நிறுவ go get ./... மற்றும் go mod tidy இயக்குகிறது.

    2. சூழல் மாறிகளை அமைக்கவும்¶

    cp .env.sample .env
# உங்கள் அமைப்புகளுடன் .env திருத்தவும்

    3. பயன்பாட்டை இயக்கவும்¶

    ./scripts/start.sh

    இந்த script .env இலிருந்து சூழல் மாறிகளை export செய்து go run main.go serve இயக்குகிறது.

    பின்னணி இயல்பாக http://localhost:8090 இல் தொடங்கும் (PocketBase இயல்புநிலை port).

    Admin Panel அணுகுவது¶

    உங்கள் பின்னணி இயங்கியதும்:

    1. http://your-backend-url/_/ க்கு செல்லவும் (underscore மற்றும் இறுதி சாய்வை கவனிக்கவும்)
    2. PB_ADMIN_EMAIL மற்றும் PB_ADMIN_PASSWORD இலிருந்து உங்கள் admin credentials மூலம் உள்நுழையவும்
    3. PocketBase admin dashboard காண்பீர்கள்

    Admin Panel இல் என்ன செய்யலாம்¶

    • Collections: அனைத்து database அட்டவணைகளையும் (users, congregations, territories, maps, addresses, முதலியன) பார்க்கவும் மற்றும் திருத்தவும்
    • Logs: பயன்பாட்டு மற்றும் கோரிக்கை பதிவுகளை பார்க்கவும்
    • Settings: அங்கீகாரம், மின்னஞ்சல், backups மற்றும் பலவற்றை உள்ளமைக்கவும்
    • Files: பதிவேற்றப்பட்ட கோப்புகள் மற்றும் சேமிப்பை நிர்வகிக்கவும்
    • API Rules: ஒவ்வொரு collection-க்கும் அனுமதிகளை அமைக்கவும்

    பாதுகாப்பு குறிப்பு: பொது அணுகலிலிருந்து admin UI-ஐ மறைக்க உற்பத்தியில் PB_HIDE_CONTROLS=true அமைக்கவும்.

    Database மேலாண்மை¶

    உங்கள் தரவை காப்புப்பிரதி எடுப்பது¶

    உங்கள் தரவு pb_data கோப்பகத்தில் உள்ளது. வழக்கமான காப்புப்பிரதிகள் மிகவும் முக்கியம்.

    கையேட்டில் காப்புப்பிரதி¶

    # முழு pb_data கோப்பகத்தின் காப்புப்பிரதி உருவாக்கவும்
tar -czf ministry-mapper-backup-$(date +%Y%m%d).tar.gz /path/to/pb_data/

    தானியங்கி தினசரி காப்புப்பிரதி¶

    தானியங்கியாக காப்புப்பிரதி எடுக்க cron job அமைக்கவும்:

    # crontab-க்கு சேர்க்கவும் (இயக்கவும்: crontab -e)
0 2 * * * tar -czf /backups/ministry-mapper-backup-$(date +\%Y\%m\%d).tar.gz /path/to/pb_data/

    இது தினமும் இரவு 2 மணிக்கு இயங்கி நேர-முத்திரை வாய்ந்த காப்புப்பிரதிகளை உருவாக்குகிறது.

    காப்புப்பிரதியிலிருந்து மீட்டெடுப்பது¶

    # முதலில் பயன்பாட்டை நிறுத்தவும்
# பின்னர் காப்புப்பிரதியை பிரிக்கவும்
tar -xzf ministry-mapper-backup-20240101.tar.gz -C /path/to/

    # சமீபத்திய குறியீட்டை இழுக்கவும்
git pull origin main

# சார்புகளை புதுப்பிக்கவும்
./scripts/update.sh

# பயன்பாட்டை மறுதொடக்கம் செய்யவும்
./scripts/start.sh

    குறிப்பு: புதுப்பிப்புகளின்போது pb_data இல் உள்ள உங்கள் தரவு பாதுகாக்கப்படுகிறது.

    Database Migrations¶

    மேம்பாட்டு mode இல் (go run பயன்படுத்தும்போது) தொடங்கும்போது பின்னணி தானாக database migrations இயக்குகிறது. பதிவுகளில் இப்படி செய்திகள் காண்பீர்கள்:

    Running migrations...
Migration 1759244334_collections_snapshot completed

    எவ்வாறு வேலை செய்கிறது:

    • Migrations migrations/ கோப்பகத்தில் சேமிக்கப்படுகின்றன
    • go run மூலம் இயக்குகிறீர்கள் என்று PocketBase கண்டறிந்தால் தானாக migrate செய்கிறது
    • உற்பத்தியில் (compiled binary), migrations-ஐ கையேட்டில் இயக்க வேண்டும்: ./main migrate

    முக்கியம்: migration கோப்புகளை கையேட்டில் நீக்கவோ மாற்றவோ கூடாது. இது உங்கள் database-ஐ சிதைக்கலாம்.

    கண்காணிப்பு மற்றும் பதிவுகள்¶

    பதிவுகளை பார்ப்பது¶

    Docker பதிவுகள்¶

    # சமீபத்திய பதிவுகளை பார்க்கவும்
docker logs ministry-mapper

# நிகழ்நேரத்தில் பதிவுகளை பின்தொடரவும் (பிழைத்திருத்தத்திற்கு பயனுள்ளது)
docker logs -f ministry-mapper

# கடைசி 100 வரிகளை பார்க்கவும்
docker logs --tail 100 ministry-mapper

    பயன்பாட்டு பதிவுகள்¶

    PocketBase பதிவுகளை pb_data/logs/ கோப்பகத்தில் சேமிக்கிறது. இவை உள்ளடக்குகின்றன:

    • கோரிக்கை பதிவுகள்
    • பிழை பதிவுகள்
    • தனிப்பயன் பயன்பாட்டு பதிவுகள்

    பதிவுகளில் எதை தேட வேண்டும்¶

    • தொடக்க செய்திகள்: வெற்றிகரமான துவக்கத்தை உறுதிப்படுத்துகிறது (எ.கா., "Starting Ministry Mapper build...")
    • Migration செய்திகள்: பயன்படுத்தப்படும் database migrations
    • Sentry துவக்கம்: பிழை கண்காணிப்பு அமைவு
    • Cron Jobs: பின்னணி பணி செயல்பாடு (territory aggregates, message processing, முதலியன)
    • பிழை செய்திகள்: database, மின்னஞ்சல், API calls அல்லது LaunchDarkly-உடன் சிக்கல்கள்
    • அங்கீகார நிகழ்வுகள்: உள்நுழைவு முயற்சிகள் மற்றும் பயனர் செயல்பாடு
    • செயல்திறன் சிக்கல்கள்: மெதுவான queries அல்லது timeout

    சிக்கல் தீர்வு¶

    Port ஏற்கனவே பயன்பாட்டில் உள்ளது¶

    சிக்கல்: "Port already in use" பிழை

    தீர்வு:

    # Docker-க்கு (port 8080)
lsof -i :8080

# உள்ளூர் மேம்பாட்டிற்கு (port 8090)
lsof -i :8090

# Port பயன்படுத்தும் process-ஐ நிறுத்தவும்
kill -9 <process_id>

# அல்லது வேறு port-க்கு மாப் செய்யவும்
docker run -p 8081:8080 ...  # host port 8081-ஐ container port 8080-க்கு மாப் செய்கிறது

    ```

    Database பூட்டப்பட்டுள்ளது¶

    சிக்கல்: "Database is locked" பிழை

    தீர்வு: - Database அணுகும் அனைத்து containers/processes-ஐ நிறுத்தவும் - காப்புப்பிரதி process இயங்குகிறதா என சரிபார்க்கவும் - பயன்பாட்டின் ஒரே நிகழ்வு மட்டுமே இயங்குகிறதா என உறுதிப்படுத்தவும் - SQLite பல processes இலிருந்து ஒரே நேரத்தில் எழுதுவதை ஆதரிக்கவில்லை - பயன்பாட்டை மறுதொடக்கம் செய்யவும்

    மின்னஞ்சல் அனுப்பப்படவில்லை¶

    சிக்கல்: பயனர்கள் மின்னஞ்சல்களை பெறவில்லை

    தீர்வுகள்: 1. SMTP அமைப்புகளை சரிபார்க்கவும்: .env இல் அனைத்து PB_SMTP_* மாறிகளையும் சரிபார்க்கவும் 2. Gmail பயனர்கள்: வழக்கமான கடவுச்சொல்லுக்கு பதிலாக App Password பயன்படுத்தவும் 3. Spam கோப்பை சரிபார்க்கவும்: மின்னஞ்சல்கள் வடிகட்டப்பட்டிருக்கலாம் 4. SMTP இணைப்பை சோதிக்கவும்: SMTP credentials-ஐ சரிபார்க்க telnet அல்லது மின்னஞ்சல் client போன்ற கருவி பயன்படுத்தவும் 5. பதிவுகளை மதிப்பாய்வு செய்யவும்: pb_data/logs/ அல்லது Docker பதிவுகளில் பிழை செய்திகளை தேடவும் 6. Port சிக்கல்கள்: Port 587 (TLS) அல்லது 465 (SSL) firewall மூலம் தடுக்கப்படவில்லை என உறுதிப்படுத்தவும்

    நினைவகம் தீர்ந்தது¶

    சிக்கல்: நினைவக பிழைகளுடன் பயன்பாடு crash ஆகிறது

    தீர்வுகள்: - Docker நினைவக வரம்புகளை அதிகரிக்கவும்: docker run -m 1g ... - நினைவக கசிவுகளுக்கு பதிவுகளை சரிபார்க்கவும் - பெரிய database queries-ஐ மேம்படுத்தவும் - உங்கள் hosting திட்டத்தின் RAM ஒதுக்கீட்டை மேம்படுத்தவும் - பெரிய data sets-க்கு SQLite PRAGMA மேம்படுத்தல்களை பரிசீலிக்கவும்

    Admin Panel அணுக முடியவில்லை¶

    சிக்கல்: Admin URL அணுகும்போது 404 பிழை

    தீர்வுகள்: - URL வடிவம் உறுதிப்படுத்தவும்: http://your-url/_/ (underscore மற்றும் இறுதி சாய்வுடன்) - PB_HIDE_CONTROLS=true இருக்கிறதா சரிபார்க்கவும் (admin UI அணுகலை தடுக்கிறது) - பயன்பாடு இயங்குகிறதா சரிபார்க்கவும்: docker ps அல்லது process சரிபார்க்கவும் - உலாவி cache மற்றும் cookies அழிக்கவும் - தொடக்க பிழைகளுக்கு பதிவுகளை சரிபார்க்கவும்

    செயல்திறன் குறிப்புகள்¶

    சிறிய சபைகள் (< 100 பயனர்கள்)¶

    • இயல்புநிலை அமைப்புகள் நன்றாக வேலை செய்கின்றன
    • 512MB RAM போதுமானது
    • SQLite இந்த சுமையை எளிதாக கையாளும்
    • சிறப்பு உள்ளமைவு தேவையில்லை

    நடுத்தர சபைகள் (100-500 பயனர்கள்)¶

    • பரிந்துரை: 1GB RAM
    • வீத வரம்பை இயக்கவும்: PB_ENABLE_RATE_LIMITING=true
    • தினசரி தானியங்கி காப்புப்பிரதிகளை அமைக்கவும்
    • pb_data/data.db இல் database அளவை கண்காணிக்கவும்

    பெரிய சபைகள் (> 500 பயனர்கள்)¶

    • பரிந்துரை: 2GB+ RAM
    • வீத வரம்பை இயக்கவும்
    • அர்ப்பணிக்கப்பட்ட hosting பரிசீலிக்கவும் (shared அல்ல)
    • மெதுவான queries-க்கு பதிவுகளை கண்காணிக்கவும்
    • காப்புப்பிரதி தக்கவைப்பு கொள்கை அமைக்கவும் (கடைசி 30 நாட்கள் வைத்திருக்கவும்)
    • பின்னணி job அதிர்வெண்ணை கட்டுப்படுத்த LaunchDarkly அம்சக் கொடிகளை பரிசீலிக்கவும்

    API Endpoints¶

    பின்னணி இந்த தனிப்பயன் அங்கீகரிக்கப்பட்ட API endpoints வழங்குகிறது (அனைத்திற்கும் பயனர் அங்கீகாரம் தேவை):

    வரைபட மேலாண்மை¶

    • POST /map/codes - குறிப்பிட்ட வரைபடத்திற்கான அனைத்து வரைபட codes-ஐ பெறவும்
    • POST /map/code/add - வரைபடத்தில் புதிய முகவரி code சேர்க்கவும்
    • POST /map/codes/update - வரைபட codes-இன் வரிசை/ஒழுங்கை புதுப்பிக்கவும்
    • POST /map/code/delete - வரைபடத்திலிருந்து முகவரி code நீக்கவும்
    • POST /map/add - புதிய வரைபடம் உருவாக்கவும்
    • POST /map/territory/update - வரைபடம் எந்த பகுதியில் சேர்ந்தது என்பதை புதுப்பிக்கவும்
    • POST /map/reset - வரைபடத்தை மீட்டமைக்கவும் (அனைத்து ஒப்படைப்புகள் மற்றும் தரவை அழிக்கவும்)

    தள மேலாண்மை¶

    • POST /map/floor/add - வரைபடத்தில் தள நிலை சேர்க்கவும்
    • POST /map/floor/remove - வரைபடத்திலிருந்து தளத்தை நீக்கவும்

    பகுதி மேலாண்மை¶

    • POST /territory/reset - பகுதியை மீட்டமைக்கவும் (ஒப்படைப்புகளை அழிக்கவும்)
    • POST /territory/link - பகுதிக்கான விரைவு அணுகல் இணைப்பை உருவாக்கவும்

    உள்ளமைவு¶

    • POST /options/update - சபை options/settings-ஐ புதுப்பிக்கவும்

    அங்கீகாரம்: அனைத்து endpoints-க்கும் request header இல் செல்லுபடியாகும் PocketBase அங்கீகார token தேவை.

    PocketBase உள்ளமைக்கப்பட்ட APIs: தனிப்பயன் endpoints-க்கு கூடுதலாக, PocketBase அனைத்து collections-க்கும் /api/collections/{collection}/records இல் தானியங்கி REST APIs வழங்குகிறது (CRUD செயல்பாடுகள்).

    பின்னணி Jobs (Cron Scheduler)¶

    பின்னணி cron scheduler பயன்படுத்தி தானியங்கி பின்னணி பணிகளை இயக்குகிறது. இந்த jobs LaunchDarkly அம்சக் கொடிகளால் கட்டுப்படுத்தப்படுகின்றன:

    திட்டமிடப்பட்ட Jobs¶

    1. பகுதி Aggregates (*/10 * * * * - ஒவ்வொரு 10 நிமிடமும்)
    2. பகுதி புள்ளிவிவரங்கள் மற்றும் தொகுக்கப்பட்ட தரவை புதுப்பிக்கிறது

    3. அம்சக் கொடி: enable-territory-aggregations

    4. ஒப்படைப்பு சுத்தப்படுத்தல் (*/5 * * * * - ஒவ்வொரு 5 நிமிடமும்)

    5. காலாவதியான அல்லது செல்லாத பகுதி ஒப்படைப்புகளை நீக்குகிறது

    6. அம்சக் கொடி: enable-assignments-cleanup

    7. செய்தி செயலாக்கம் (*/30 * * * * - ஒவ்வொரு 30 நிமிடமும்)

    8. queue-இல் உள்ள நிலுவை செய்திகளை செயலாக்குகிறது
    9. OPENAI_API_KEY அமைக்கப்பட்டிருந்தால் விருப்பமாக AI-உருவாக்கிய கண்ணோட்டத்தை இணைக்கிறது

    10. அம்சக் கொடி: enable-message-processing

    11. வழிமுறை செயலாக்கம் (*/30 * * * * - ஒவ்வொரு 30 நிமிடமும்)

    12. நிலுவை பகுதி ஒப்படைப்பு வழிமுறைகளை கையாளுகிறது

    13. அம்சக் கொடி: enable-instruction-processing

    14. குறிப்பு செயலாக்கம் (0 * * * * - மணிதோறும்)

    15. சபைகளுக்கான புதுப்பிக்கப்பட்ட குறிப்புகளை செயலாக்குகிறது
    16. OPENAI_API_KEY அமைக்கப்பட்டிருந்தால் விருப்பமாக AI-உருவாக்கிய சுருக்கத்தை இணைக்கிறது

    17. அம்சக் கொடி: enable-note-processing

    18. மாதாந்திர அறிக்கை (0 0 1 * * - ஒவ்வொரு மாதத்தின் 1ம் தேதி நள்ளிரவில்)

    19. சபை தரவின் Excel அறிக்கையை உருவாக்கி நிர்வாகிகளுக்கு மின்னஞ்சல் அனுப்புகிறது
    20. அம்சக் கொடி: enable-monthly-report

    21. AI சுருக்கங்கள் கொடி: enable-report-ai-summary (OPENAI_API_KEY தேவை)

    22. ஒதுக்கப்படாத பயனர் செயலாக்கம் (0 1 * * * - தினமும் 01:00 UTC)

    23. பங்கு ஒப்படைப்பு இல்லாத கணக்குகளுக்கான பயனர் lifecycle-ஐ நடைமுறைப்படுத்துகிறது: 3ம் நாள் → முதல் எச்சரிக்கை மின்னஞ்சல், 6ம் நாள் → இறுதி எச்சரிக்கை மின்னஞ்சல், 7ம் நாள் → கணக்கு முடக்கப்படும், 37+ நாட்கள் → நிரந்தர நீக்கம்

    24. அம்சக் கொடி: enable-unprovisioned-user-processing

    25. செயலற்ற பயனர் செயலாக்கம் (30 1 * * * - தினமும் 01:30 UTC)

    26. உள்ளமைக்கப்பட்ட வரம்பை தாண்டி செயலற்றிருக்கும் கணக்குகளை எச்சரிக்கை செய்து இறுதியில் முடக்குகிறது
    27. அம்சக் கொடி: enable-inactive-user-processing

    குறிப்பு: LaunchDarkly உள்ளமைக்கப்படவில்லை என்றால், அனைத்து jobs-ம் இயல்பாக இயங்கும் (இயக்கப்பட்டது: true).

    AI அறிக்கை சுருக்கங்கள்¶

    OPENAI_API_KEY உள்ளமைக்கப்பட்டு மற்றும் enable-report-ai-summary LaunchDarkly கொடி இயக்கப்பட்டிருக்கும்போது, மாதாந்திர Excel அறிக்கை ஒவ்வொரு பகுதி sheet-க்கும் AI-உருவாக்கிய சுருக்கத்தை உள்ளடக்குகிறது. AI செயல்பாட்டு தரவை பகுப்பாய்வு செய்து 2–3 வாக்கிய கண்ணோட்டம் மற்றும் முக்கிய கவனிப்புகளின் பட்டியலை உருவாக்குகிறது.

    இந்த அம்சம் முற்றிலும் விருப்பமானது — அறிக்கைகள் இதன்றியும் முழுமையாக வேலை செய்யும். இதை இயக்க:

    1. உங்கள் .env கோப்பில் OPENAI_API_KEY அமைக்கவும்.
    2. உங்கள் LaunchDarkly திட்டத்தில் enable-report-ai-summary கொடியை இயக்கவும் (அல்லது LaunchDarkly-ஐ தவிர்க்கவும், கொடி இயல்பாக இயக்கப்பட்டதாக அமையும்).

    Database Hooks¶

    பயன்பாடு சில நிகழ்வுகளை தானாக கையாளுகிறது:

    • முகவரி புதுப்பிப்புகள்: முகவரி குறிப்புகள் மாறும்போது, timestamps மற்றும் பயனர் தகவல் பதிவு செய்யப்படுகின்றன
    • முகவரி புதுப்பிப்புகள்: வரைபட aggregate மீண்டும் கணக்கிடலை தூண்டுகிறது
    • பயனர் அங்கீகாரம்: கடைசி உள்நுழைவு timestamp புதுப்பிக்கிறது
    • பயனர் உருவாக்கம்: மின்னஞ்சல் முகவரிகளை சீராக்கி சிறிய எழுத்தாக்குகிறது
