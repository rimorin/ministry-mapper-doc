    முன்னோடி அமைவு வழிகாட்டி¶

    சுய-தொகுப்பு பரிந்துரைக்கப்படவில்லை

    இந்த வழிகாட்டி Ministry Mapper-ஐ சுய-தொகுக்க விரும்பும் மேம்பட்ட பயனர்களுக்கானது. தொழில்நுட்ப சிக்கலான தன்மை, பராமரிப்பு சுமை மற்றும் பாதுகாப்பு பொறுப்புகள் காரணமாக சுய-தொகுப்பை நாங்கள் ஊக்குவிக்கவில்லை.

    பெரும்பாலான பயனர்கள்: ministry-mapper.com இல் எங்கள் நேரடி சேவையை பயன்படுத்தவும்.

    பயனர் ஆவணங்களை தேடுகிறீர்களா?

    Ministry Mapper-ஐ எவ்வாறு பயன்படுத்துவது என்று கற்றுக்கொள்ள விரும்பும் சாதாரண பயனராக இருந்தால், பதிலாக தொடங்குதல் அல்லது பயனர் வழிகாட்டி பார்க்கவும்.

    கண்ணோட்டம்¶

    Ministry Mapper முன்னோடி (ministry-mapper-v2) என்பது React அடிப்படையிலான வலை பயன்பாடு, இது வழங்குகிறது:

    • பயனர் அங்கீகாரம் மற்றும் கணக்கு மேலாண்மை
    • ஊடாடும் பகுதி மேலாண்மை இடைமுகம்
    • ஊடாடும் வரைபட செயல்பாடு
    • நிகழ்நேர தரவு ஒத்திசைவு
    • மொபைல்-பதிலளிக்கக்கூடிய வடிவமைப்பு
    • பல மொழி ஆதரவு (7 மொழிகள்)
    • முற்போக்கான வலை பயன்பாட்டு திறன்கள்

    இந்த வழிகாட்டி இவற்றை அறிந்திருக்கும் என்று கருதுகிறது: - Node.js மற்றும் npm அனுபவம் - சூழல் மாறிகள் பற்றிய புரிதல் - நிலையான தளம் பயன்படுத்துவதில் நெறிமுறை - வலை பயன்பாட்டு பாதுகாப்பு அறிவு

    முழுமையான சுய-தொகுப்பு வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    விரைவு குறிப்பு¶

    இந்தப் பக்கம் முன்னோடி உள்ளமைவுக்கான விரைவு குறிப்பை வழங்குகிறது. முழுமையான அமைவு வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    விரைவு குறிப்பு¶

    இந்தப் பக்கம் முன்னோடி உள்ளமைவுக்கான விரைவு குறிப்பை வழங்குகிறது. முழுமையான அமைவு வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    தொழில்நுட்ப அடுக்கு¶

    • React 19: நவீன UI நூலகம்
    • TypeScript: வகை-பாதுகாப்பான JavaScript
    • Vite 7: வேகமான உருவாக்க கருவி மற்றும் dev சேவையகம்
    • Bootstrap 5: பதிலளிக்கக்கூடிய CSS கட்டமைப்பு
    • Leaflet: ஊடாடும் வரைபடம்
    • PocketBase SDK: பின்னணி இணைப்பு
    • Sentry: பிழை கண்காணிப்பு
    • i18next: பல மொழி ஆதரவு
    • Vite PWA: முற்போக்கான வலை பயன்பாட்டு அம்சங்கள்

    முழுமையான அமைவு வழிமுறைகள்

    கீழ்கண்ட பகுதிகள் முன்னோடி உள்ளமைவுக்கான தொழில்நுட்ப குறிப்பை வழங்குகின்றன. வரிசைமுறை அமைவு வழிமுறைகள், பயன்படுத்தல் விருப்பங்கள் மற்றும் சிக்கல் தீர்வுடன் கூடிய முழுமையான வழிமுறைகளுக்கு, சுய-தொகுப்பு வழிகாட்டி பார்க்கவும்.

    சூழல் மாறிகள்¶

    இந்த அமைப்புகளுடன் root கோப்பகத்தில் .env கோப்பை உருவாக்கவும்:

    தேவையான மாறிகள்¶

    # System Environment - பயன்படுத்தல் சூழலை குறிப்பிடுகிறது
VITE_SYSTEM_ENVIRONMENT=production

# Version - package.json பதிப்பை தானாக பயன்படுத்துகிறது
VITE_VERSION=$npm_package_version

# PocketBase Backend URL - இறுதி சாய்வு இல்லாமல்
VITE_POCKETBASE_URL=https://your-backend-url.com

# Legal Pages - உற்பத்திக்கு தேவை
VITE_PRIVACY_URL=https://your-site.com/privacy
VITE_TERMS_URL=https://your-site.com/terms
VITE_ABOUT_URL=https://your-site.com/about

# Map & Routing
VITE_OPENROUTE_API_KEY=your_openrouteservice_api_key
VITE_LOCATIONIQ_API_KEY=your_locationiq_api_key

    விருப்பமான மாறிகள்¶

    # Error Tracking (Sentry) - உற்பத்திக்கு பரிந்துரைக்கப்படுகிறது
VITE_SENTRY_DSN=https://your_sentry_dsn@sentry.io/123456
# Build-time Sentry tokens for source map upload
SENTRY_AUTH_TOKEN=your_sentry_auth_token
SENTRY_ORG=your_sentry_org_slug
SENTRY_PROJECT=your_sentry_project_slug

# Maintenance Mode - பயனர்களுக்கு பராமரிப்பு பதாகையை காட்டு
VITE_MAINTENANCE_MODE=false

    • மதிப்புகள்:
    • local - உங்கள் உள்ளூர் கணினியில் மேம்பாடு
    • production - நேரடி பயன்படுத்தல்
    • விளைவு: Sentry பிழை கண்காணிப்பு உள்ளமைவு மற்றும் பதிவுகளை பாதிக்கிறது

    VITE_VERSION¶

    • நோக்கம்: பிழை அறிக்கைகளில் பயன்பாட்டு பதிப்பை கண்காணிக்கிறது
    • மதிப்பு: $npm_package_version (package.json இலிருந்து தானாக படிக்கிறது)
    • தற்போதைய பதிப்பு: 1.9.1 (package.json இன்படி)
    • விளைவு: Sentry அறிக்கைகளில் காட்டப்படுகிறது மற்றும் எந்த பதிப்பில் சிக்கல்கள் உள்ளன என்பதை கண்காணிக்க உதவுகிறது

    VITE_POCKETBASE_URL¶

    • நோக்கம்: உங்கள் PocketBase பின்னணி சேவையகத்தின் URL
    • வடிவம்: இறுதி சாய்வு இல்லாத முழு URL
    • எடுத்துக்காட்டு: https://api.ministry-mapper.com அல்லது உள்ளூருக்கு http://localhost:8090
    • முக்கியம்: முன்னோடி பயன்படுத்தப்படும் இடத்திலிருந்து அணுகக்கூடியதாக இருக்க வேண்டும்

    VITE_PRIVACY_URL¶

    • நோக்கம்: உங்கள் தனியுரிமை கொள்கை பக்கத்திற்கான இணைப்பு
    • தேவை: ஆம், சட்ட இணக்கத்திற்கு (GDPR, CCPA, முதலியன)
    • எடுத்துக்காட்டு: https://ministry-mapper.com/privacy

    VITE_TERMS_URL¶

    • நோக்கம்: உங்கள் சேவை விதிமுறைகள் பக்கத்திற்கான இணைப்பு
    • தேவை: ஆம், சட்ட இணக்கத்திற்கு
    • எடுத்துக்காட்டு: https://ministry-mapper.com/terms

    VITE_ABOUT_URL¶

    • நோக்கம்: உங்கள் Ministry Mapper நிகழ்விற்பான தகவலுக்கான இணைப்பு
    • தேவை: இல்லை, ஆனால் பரிந்துரைக்கப்படுகிறது
    • எடுத்துக்காட்டு: https://ministry-mapper.com/about

    VITE_SENTRY_DSN¶

    • நோக்கம்: கண்காணிப்பிற்காக JavaScript பிழைகளை Sentry-க்கு அனுப்புகிறது
    • எங்கிருந்து பெறுவது: sentry.io (இலவச அடுக்கு கிடைக்கிறது)
    • தேவை: இல்லை, ஆனால் உற்பத்திக்கு மிகவும் பரிந்துரைக்கப்படுகிறது
    • நன்மைகள்: பிழைகளை கண்காணிக்கவும், செயல்திறனை கண்காணிக்கவும், சிக்கல்கள் தெரிவிக்கப்படவும்
    • குறிப்பு: VITE_SYSTEM_ENVIRONMENT "production" ஆக இருக்கும்போது மட்டுமே செயல்படும்

    SENTRY_AUTH_TOKEN / SENTRY_ORG / SENTRY_PROJECT¶

    • நோக்கம்: சிறுமைப்படுத்தப்பட்ட stack traces-ஐ உங்கள் மூல குறியீட்டிற்கு திரும்பி தீர்க்க, npm run build இன்போது source maps-ஐ Sentry-க்கு பதிவேற்றுவதற்கு பயன்படுத்தப்படுகிறது
    • தேவை: இல்லை — இவை இல்லாவிட்டால் source maps உள்ளூரில் உருவாக்கப்படும், ஆனால் அவற்றை Sentry-க்கு பதிவேற்றுவது உற்பத்தியில் பிழை பிழைத்திருத்தத்தை மிகவும் மேம்படுத்துகிறது
    • எவ்வாறு பெறுவது: Settings → Auth Tokens இல் உங்கள் Sentry நிறுவன அமைப்புகளில் ஒரு auth token உருவாக்கவும்

    VITE_OPENROUTE_API_KEY¶

    • நோக்கம்: ஊடாடும் வரைபடத்தில் திருப்பம்-திருப்பம் வழித்தடம் மற்றும் வழிகாட்டுதல்களை (ஓட்டுதல், நடத்தல், சைக்கிள் ஓட்டுதல்) இயக்குகிறது
    • எங்கிருந்து பெறுவது: openrouteservice.org (இலவச அடுக்கு கிடைக்கிறது)
    • தேவை: ஆம்

    VITE_LOCATIONIQ_API_KEY¶

    • நோக்கம்: முகவரிகளை கோரடினேட்களுக்கு மாற்றுதல் மற்றும் தலைகீழ் மாற்றுதல் (கோரடினேட்களை முகவரிகளுக்கு)
    • எங்கிருந்து பெறுவது: locationiq.com (இலவச அடுக்கு கிடைக்கிறது)
    • தேவை: ஆம்

    VITE_MAINTENANCE_MODE¶

    • நோக்கம்: true என அமைக்கும்போது, பயனர்களுக்கு பராமரிப்பு பதாகையை காட்டுகிறது
    • மதிப்புகள்: true அல்லது false
    • இயல்புநிலை: false
    • பயன்பாட்டு நிகழ்வு: பின்னணி மேம்படுத்தல்கள் அல்லது திட்டமிடப்பட்ட வேலையில்லா நேரத்தின்போது தற்காலிகமாக true என அமைக்கவும்

    உள்ளூர் மேம்பாட்டு அமைவு¶

    முன்நிபந்தனைகள்¶

    Node.js >=24.0.0 தேவை:

    # உங்கள் Node.js பதிப்பை சரிபார்க்கவும்
node --version

    Node.js >=24.0.0 இல்லாவிட்டால், இங்கிருந்து பதிவிறக்கவும்: nodejs.org

    அமைவு வழிமுறைகள்¶

    1. களஞ்சியத்தை குளோன் செய்யவும்¶

    git clone https://github.com/rimorin/ministry-mapper-v2.git
cd ministry-mapper-v2

    2. சார்புகளை நிறுவவும்¶

    npm install

    இது அனைத்து தேவையான தொகுப்புகளை பதிவிறக்குகிறது. உங்கள் இணைய இணைப்பைப் பொறுத்து சில நிமிடங்கள் ஆகலாம்.

    3. சூழல் கோப்பை உருவாக்கவும்¶

    # எடுத்துக்காட்டு சூழல் கோப்பை நகலெடுக்கவும்
cp .env.example .env

    உங்கள் அமைப்புகளுடன் .env திருத்தவும்:

    VITE_SYSTEM_ENVIRONMENT=local
VITE_VERSION=$npm_package_version
VITE_POCKETBASE_URL=http://localhost:8090
VITE_PRIVACY_URL=http://localhost:3000/privacy
VITE_TERMS_URL=http://localhost:3000/terms
VITE_ABOUT_URL=http://localhost:3000/about
VITE_SENTRY_DSN=
VITE_OPENROUTE_API_KEY=
VITE_LOCATIONIQ_API_KEY=
VITE_MAINTENANCE_MODE=false

    குறிப்பு: Port 8090 இல் இயங்கும் PocketBase பின்னணி தேவை. பின்னணி அமைவிற்கு ministry-mapper-be களஞ்சியம் பார்க்கவும்.

    4. மேம்பாட்டு சேவையகத்தை தொடங்கவும்¶

    npm start

    பயன்பாடு http://localhost:3000 இல் திறக்கும் (port 3000 vite.config.js இல் உள்ளமைக்கப்பட்டுள்ளது).

    மேம்பாட்டு அம்சங்கள்¶

    • Hot Module Replacement (HMR): முழு பக்க ஏற்றம் இல்லாமல் மாற்றங்கள் உடனடியாக தோன்றும்
    • TypeScript சரிபார்ப்பு: முனையம் மற்றும் உலாவி console இல் பிழைகள் காட்டப்படும்
    • Source Maps: அசல் மூல குறியீடு குறிப்புகளுடன் பிழைத்திருத்தம் எளிதாகும்
    • Fast Refresh: React கூறுகள் நிலையை இழக்காமல் புதுப்பிக்கும்

    மேம்பாட்டு கட்டளைகள்¶

    # மேம்பாட்டு சேவையகத்தை தொடங்கவும் (port 3000)
npm start

# Vitest மூலம் சோதனைகளை இயக்கவும்
npm test

# குறியீடு வடிவமைப்பை சரிபார்க்கவும்
npm run prettier

# வடிவமைப்பு சிக்கல்களை தானாக சரிசெய்யவும்
npm run prettier:fix

# உற்பத்திக்கு உருவாக்கவும்
npm run build

# உற்பத்தி உருவாக்கத்தை உள்ளூரில் முன்னோட்டமிடவும்
npm run serve

    குறியீடு தரக் கருவிகள்¶

    Prettier (குறியீடு வடிவமைப்பு):

    • .prettierrc இல் உள்ளமைக்கப்பட்டுள்ளது
    • Husky மூலம் commit இன்போது தானாக வடிவமைக்கிறது
    • npm run prettier:fix மூலம் கையேட்டில் இயக்கவும்

    ESLint (குறியீடு lint):

    • eslint.config.mjs இல் உள்ளமைக்கப்பட்டுள்ளது
    • React மற்றும் TypeScript விதிகள் இயக்கப்பட்டுள்ளன
    • பொதுவான பிழைகள் மற்றும் குறியீடு தர சிக்கல்களை சரிபார்க்கிறது

    Husky (git hooks):

    • commit செய்வதற்கு முன் staged கோப்புகளில் Prettier இயக்குகிறது
    • ஒரேசீரான குறியீடு வடிவமைப்பை உறுதி செய்கிறது
    • commitlint மூலம் commit செய்திகளை சரிப்படுத்துகிறது

    உற்பத்தி பயன்படுத்தல்¶

    உற்பத்திக்கு உருவாக்குதல்¶

    # மேம்படுத்தப்பட்ட உற்பத்தி கோப்புகளை உருவாக்கவும்
npm run build

    உருவாக்கத்தின்போது என்ன நடக்கும்:

    • TypeScript JavaScript ஆக தொகுக்கப்படுகிறது
    • React குறியீடு மேம்படுத்தப்பட்டு சிறுமைப்படுத்தப்படுகிறது
    • CSS செயலாக்கப்பட்டு சிறுமைப்படுத்தப்படுகிறது
    • Source maps உருவாக்கப்படுகின்றன
    • வேகமான ஏற்றத்திற்கு குறியீடு தொகுதிகளாக பிரிக்கப்படுகிறது
    • வெளியீடு build/ கோப்பகத்திற்கு செல்கிறது
    • PWA ஆதரவிற்காக service worker உருவாக்கப்படுகிறது

    உருவாக்க வெளியீடு:

    • build/index.html - முக்கிய HTML கோப்பு
    • build/assets/ - JavaScript, CSS மற்றும் பிற சொத்துக்கள்
    • கோப்பு பெயர்கள் கேச்சிங்கிற்காக content hashes உள்ளன
    • மொத்த அளவு பொதுவாக 1MB-க்கும் குறைவு (வெளிப்புற சார்புகளை தவிர)

    விருப்பம் 1: Vercel (பரிந்துரைக்கப்படுகிறது)¶

    வேகமானது, இலவச அடுக்கு, தானியங்கி பயன்படுத்தல்கள்.

    அமைவு:

    1. vercel.com இல் கணக்கை உருவாக்கவும்
    2. "New Project" கிளிக் செய்யவும்
    3. உங்கள் GitHub களஞ்சியத்தை இறக்குமதி செய்யவும்
    4. உள்ளமைக்கவும்:
    5. Framework Preset: Vite
    6. Build Command: npm run build
    7. Output Directory: build
    8. Install Command: npm install
    9. Vercel dashboard இல் சூழல் மாறிகளை சேர்க்கவும்
    10. பயன்படுத்தவும்

    Vercel அம்சங்கள்:

    • git push இல் தானியங்கி பயன்படுத்தல்கள்
    • pull requests-க்கு முன்னோட்ட பயன்படுத்தல்கள்
    • தனிப்பயன் டொமைன்கள்
    • இயல்பிலேயே HTTPS
    • உலகளாவிய CDN
    • தனிப்பட்ட திட்டங்களுக்கு இலவசம்

    விருப்பம் 2: Netlify¶

    சிறந்த அம்சங்களுடன் எளிய பயன்படுத்தல்.

    5. Build Command: npm run build
    6. Publish Directory: build
    7. சூழல் மாறிகளை சேர்க்கவும்
    8. பயன்படுத்தவும்

    Netlify அம்சங்கள்:

    • தானியங்கி பயன்படுத்தல்கள்
    • படிவம் கையாளுதல்
    • Serverless functions (தேவைப்பட்டால்)
    • தனிப்பயன் டொமைன்கள்
    • இயல்பிலேயே HTTPS

    5. Framework preset: Vite

    7. Build output directory: build
    8. சூழல் மாறிகளை சேர்க்கவும்
    9. பயன்படுத்தவும்

    Cloudflare அம்சங்கள்:

    • உலகளாவிய CDN நெட்வொர்க்
    • DDoS பாதுகாப்பு
    • Analytics
    • கிளையன்ட்-பக்க கண்காணிப்பு இல்லாமல் Web Analytics

    விருப்பம் 4: AWS S3 + CloudFront¶

    அதிக கட்டுப்பாடு, பெரிய பயன்படுத்தல்களுக்கு நல்லது.

    அமைவு:

    1. S3 bucket உருவாக்கவும்
    2. நிலையான இணையதள hosting-ஐ இயக்கவும்
    3. build/ உள்ளடக்கங்களை பதிவேற்றவும்
    4. CloudFront distribution உருவாக்கவும்
    5. தனிப்பயன் டொமைனை உள்ளமைக்கவும்
    6. SSL சான்றிதழை அமைக்கவும்

    AWS பரிசீலனைகள்:

    • மிகவும் சிக்கலான அமைவு
    • பயன்பாட்டிற்கு ஏற்ப கட்டணம் (பொதுவாக மிகவும் மலிவான)
    • அதிக உள்ளமைவு விருப்பங்கள்
    • நிறுவன பயன்படுத்தல்களுக்கு நல்லது

    விருப்பம் 5: சுய-தொகுப்பு¶

    உங்கள் சொந்த சேவையகத்தில் host செய்யவும்.

    Nginx பயன்படுத்தி:

    server {
    listen 80;
    server_name your-domain.com;
    root /var/www/ministry-mapper/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Enable gzip compression
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;

    # Cache static assets
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}

    Apache பயன்படுத்தி:

    <VirtualHost *:80>
    ServerName your-domain.com
    DocumentRoot /var/www/ministry-mapper/build

    <Directory /var/www/ministry-mapper/build>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted

        # Enable React Router
        RewriteEngine On
        RewriteBase /
        RewriteRule ^index\.html$ - [L]
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule . /index.html [L]
    </Directory>
</VirtualHost>

    சுய-தொகுப்பிற்கு முக்கியம்:

    • HTTPS மூலம் வழங்கவும் (PWA அம்சங்களுக்கு தேவை)
    • பின்னணி வேறு டொமைனில் இருந்தால் சரியான CORS headers உள்ளமைக்கவும்
    • நிலையான சொத்துக்களுக்கு சரியான caching headers அமைக்கவும்
    • SPA routing சரியாக வேலை செய்கிறதா என்பதை உறுதிப்படுத்தவும் (அனைத்து routes-ம் index.html-ஐ வழங்கும்)

    பயன்படுத்தலில் சூழல் மாறிகள்¶

    முக்கியம்: Vite உருவாக்க நேரத்தில் அவற்றை செலுத்துவதால், சூழல் மாறிகளை runtime இல் அல்ல, உருவாக்க நேரத்தில் அமைக்க வேண்டும்.

    Vercel/Netlify/Cloudflare-க்கு:

    • தளத்தின் dashboard இல் சூழல் மாறிகளை சேர்க்கவும்
    • உருவாக்க செயல்முறையின்போது அவை கிடைக்கும்

    சுய-தொகுப்பிற்கு:

    • npm run build இயக்குவதற்கு முன் சூழல் மாறிகளை அமைக்கவும்
    • அல்லது உங்கள் அமைப்புகளுடன் .env.production கோப்பை உருவாக்கவும்

    Sentry அமைப்பு (விருப்பமானது ஆனால் பரிந்துரைக்கப்படுகிறது)¶

    Sentry JavaScript பிழைகள் மற்றும் செயல்திறனை கண்காணிக்கிறது.

    1. Sentry கணக்கை உருவாக்கவும்¶

    1. sentry.io க்கு செல்லவும்
    2. பதிவு செய்யவும் (வள்ளல் இலவச அடுக்கு கிடைக்கிறது)
    3. புதிய திட்டத்தை உருவாக்கவும்
    4. தளத்தை தேர்வுசெய்யவும்: "React"

    2. உங்கள் DSN-ஐ பெறவும்¶

    1. திட்டத்தை உருவாக்கிய பின், DSN காண்பீர்கள்
    2. அல்லது Settings → Projects → [Your Project] → Client Keys (DSN) க்கு செல்லவும்
    3. DSN URL-ஐ நகலெடுக்கவும்

    3. சூழலில் சேர்க்கவும்¶

    VITE_SENTRY_DSN=https://your_key@your_org.sentry.io/your_project_id

    குறிப்பு: VITE_SYSTEM_ENVIRONMENT production என அமைக்கும்போது மட்டுமே Sentry செயல்படும்.

    4. Sentry-ஐ உள்ளமைக்கவும் (விருப்பமானது)¶

    Source Maps: உருவாக்க செயல்முறை தானாக source maps உருவாக்குகிறது, இது சிறுமைப்படுத்தப்பட்ட உற்பத்தி குறியீட்டை பிழைத்திருத்தம் செய்ய உதவுகிறது.

    விழிப்பூட்டல்கள்: Sentry dashboard இல் அமைக்கவும்:

    • புதிய சிக்கல்களுக்கு மின்னஞ்சல் அறிவிப்புகள்
    • Slack/Discord ஒருங்கிணைப்பு
    • முக்கியமான பிழைகளுக்கு தனிப்பயன் விழிப்பூட்டல் விதிகள்
    • வெளியீடு கண்காணிப்பு

    ஆதரிக்கப்படும் மொழிகள்¶

    • ஸ்பானிஷ் (es - Español)
    • ஜப்பானிய (ja - 日本語)

    மொழி கண்டறிதல் எவ்வாறு வேலை செய்கிறது¶

    • i18next-browser-languagedetector பயன்படுத்துகிறது
    • முதல் வருகையில் உலாவி அமைப்புகளிலிருந்து தானாக கண்டறிகிறது
    • பயனர்கள் வழிசலவை பட்டியில் உள்ள மொழி தேர்வியின் மூலம் கையேட்டில் மொழியை மாற்றலாம்
    • தேர்வு localStorage இல் சேமிக்கப்படுகிறது மற்றும் உலாவி அமைப்புகளை விட முன்னுரிமை பெறுகிறது
    • தேர்ந்தெடுக்கப்பட்ட மொழி ஆதரிக்கப்படவில்லை என்றால் ஆங்கிலத்திற்கு திரும்புகிறது

    புதிய மொழியை சேர்ப்பது எப்படி¶

    1. மொழிபெயர்ப்பு கோப்பை உருவாக்கவும்

    2. src/i18n/locales/ க்கு செல்லவும்

    3. புதிய கோப்பை உருவாக்கவும்: [language-code].json (எ.கா., பிரெஞ்சுக்கு fr.json)

    4. en.json இலிருந்து அமைப்பை நகலெடுக்கவும்

    5. அனைத்து சொற்றொடர்களையும் மொழிபெயர்க்கவும்

    6. அனைத்து key-value pairs-ஐ மொழிபெயர்க்கவும்

    7. ஒரே JSON அமைப்பை வைத்திருக்கவும்

    8. {{variable}} போன்ற placeholder மாறிகளை பராமரிக்கவும்

    9. மொழியை பதிவு செய்யவும்

    10. src/i18n/index.ts திருத்தவும்
    11. உங்கள் மொழிபெயர்ப்பு கோப்பை இறக்குமதி செய்யவும்
    12. resources object-இல் சேர்க்கவும்

    பிரெஞ்சுக்கு எடுத்துக்காட்டு:

    import fr from './locales/fr.json';

resources: {
  en: { translation: en },
  fr: { translation: fr },
  // ... other languages
}

    முற்போக்கான வலை பயன்பாடு (PWA)¶

    Ministry Mapper Vite PWA plugin மூலம் PWA ஆதரவை உள்ளடக்குகிறது.

    PWA அம்சங்கள்¶

    • நிறுவக்கூடியது: மொபைல்/desktop home screen இல் நிறுவலாம்
    • ஆஃப்லைன் சொத்துக்கள்: வேகமான ஏற்றத்திற்காக நிலையான கோப்புகள் கேச் செய்யப்படுகின்றன
    • Service Worker: தானாக உருவாக்கப்பட்டு பதிவு செய்யப்படுகிறது
    • Cache Strategy: வெளிப்புற சொத்துக்களுக்கு StaleWhileRevalidate, எழுத்துருக்களுக்கு CacheFirst

    PWA உள்ளமைவு¶

    vite.config.js இல் உள்ளமைவு:

    VitePWA({
  registerType: "autoUpdate", // Auto-update service worker
  manifest: false, // No manifest (can be added if needed)
  workbox: {
    globPatterns: ["**/*.{js,css,html,ico,png,svg,woff2}"],
    skipWaiting: true, // Activate new service worker immediately
    clientsClaim: true, // Take control of clients immediately
    cleanupOutdatedCaches: true, // Remove old caches
  },
});

    Caching உத்தி¶

    வெளிப்புற சொத்துக்கள் (எ.கா., assets.ministry-mapper.com இலிருந்து):

    • உத்தி: StaleWhileRevalidate
    • Cache பெயர்: "external-assets"
    • காலாவதி: 7 நாட்கள், அதிகபட்சம் 50 நுழைவுகள்

    எழுத்துருக்கள்:

    • உத்தி: CacheFirst
    • Cache பெயர்: "fonts"
    • காலாவதி: 30 நாட்கள், அதிகபட்சம் 30 நுழைவுகள்

    குறிப்பு: தரவு செயல்பாடுகளுக்கு (PocketBase இணைப்பு) பயன்பாட்டிற்கு இணையம் தேவை. நிலையான சொத்துக்கள் மட்டுமே ஆஃப்லைனில் கேச் செய்யப்படும்.

    • உலாவியில் நேரடியாக பின்னணி URL-ஐ சோதிக்கவும்
    • PocketBase இல் CORS அமைப்புகளை சரிபார்க்கவும் (உங்கள் முன்னோடி டொமைனை அனுமதிக்க வேண்டும்)
    • சரியான பிழையை காண உலாவி DevTools → Network tab திறக்கவும்
    • உங்கள் பயன்படுத்தல் இடத்திலிருந்து பின்னணி அணுகக்கூடியதா என உறுதிப்படுத்தவும்

    உருவாக்க பிழைகள்¶

    சிக்கல்: npm run build தோல்வியடைகிறது

    தீர்வுகள்:

    • அனைத்து சூழல் மாறிகளும் சரியாக அமைக்கப்பட்டுள்ளனவா என உறுதிப்படுத்தவும்
    • சார்புகளை புதுப்பிக்க npm install இயக்கவும்
    • node_modules நீக்கி மீண்டும் நிறுவவும்: rm -rf node_modules && npm install
    • Node.js பதிப்பை சரிபார்க்கவும்: node --version (>=24.0.0 தேவை)
    • TypeScript பிழைகளை சரிபார்க்கவும்: குறிப்பிட்ட பிழை செய்திகளை பார்க்கவும்
    • வடிவமைப்பு சிக்கல்களை சரிசெய்ய npm run prettier:fix இயக்கவும்
    • Vite cache-ஐ அழிக்கவும்: rm -rf node_modules/.vite

    TypeScript பிழைகள்¶

    சிக்கல்: உருவாக்கத்தின்போது type பிழைகள்

    தீர்வுகள்:

    • type definitions-க்கு src/utils/interface.ts சரிபார்க்கவும்
    • imports சரியானதா என உறுதிப்படுத்தவும்
    • அனைத்து type பிழைகளையும் காண npx tsc --noEmit இயக்கவும்
    • அனைத்து சார்புகளும் நிறுவப்பட்டுள்ளனவா என சரிபார்க்கவும்

    மெதுவான செயல்திறன்¶

    சிக்கல்: பயன்பாடு மெதுவாக அல்லது தடையாக உள்ளது

    தீர்வுகள்:

    • இணைய இணைப்பு வேகத்தை சரிபார்க்கவும்
    • உலாவி cache-ஐ அழித்து மீண்டும் ஏற்றவும்
    • PocketBase பின்னணி வேகமாக பதிலளிக்கிறதா என சரிபார்க்கவும்
    • JavaScript பிழைகளுக்கு உலாவி console சரிபார்க்கவும்
    • சிக்கலை தனிமைப்படுத்த வேறு சாதனம்/உலாவியில் சோதிக்கவும்
    • Lighthouse செயல்திறன் மதிப்பெண்ணை சரிபார்க்கவும்
    • Service worker வேலை செய்கிறதா என சரிபார்க்கவும்: DevTools → Application → Service Workers

    PWA நிறுவல் சிக்கல்கள்¶

    சிக்கல்: "Add to Home Screen" தோன்றவில்லை

    தீர்வுகள்:

    • HTTPS மூலம் வழங்கப்பட வேண்டும்
    • DevTools இல் service worker பதிவை சரிபார்க்கவும்
    • PWA தேவைகள் பூர்த்தியாகின்றனவா என சரிபார்க்கவும் (manifest, service worker, முதலியன)
    • வேறு உலாவி முயற்சிக்கவும் (Safari, Chrome வெவ்வேறு PWA ஆதரவு கொண்டுள்ளன)

    நிகழ்நேர புதுப்பிப்புகள் வேலை செய்யவில்லை¶

    சிக்கல்: மற்ற பயனர்களுக்கு மாற்றங்கள் தோன்றவில்லை

    தீர்வுகள்:

    • PocketBase realtime subscriptions வேலை செய்கின்றனவா என சரிபார்க்கவும்
    • SSE (Server-Sent Events) பிழைகளுக்கு உலாவி console சரிபார்க்கவும்
    • பின்னணி SSE endpoint அணுகக்கூடியதா என உறுதிப்படுத்தவும்
    • பல பயனர்கள் உண்மையில் ஒரே பகுதியை பார்க்கிறார்களா என சரிபார்க்கவும்
    • மீண்டும் இணைக்க பக்கத்தை புதுப்பிக்கவும்

    பராமரிப்பு¶

    சார்புகளை புதுப்பித்திருக்கவும்¶

    # களஞ்சியத்திலிருந்து சமீபத்திய மாற்றங்களை இழுக்கவும்
git pull origin main

# புதுப்பிக்கப்பட்ட சார்புகளை நிறுவவும்
npm install

# பாதுகாப்பு பாதிப்புகளை சரிபார்க்கவும்
npm audit

# பாதுகாப்பு சிக்கல்களை தானாக சரிசெய்யவும் (சாத்தியமான போது)
npm audit fix

# காலாவதியான தொகுப்புகளை சரிபார்க்கவும்
npm outdated

# குறிப்பிட்ட தொகுப்பை புதுப்பிக்கவும்
npm update package-name

    கண்காணிப்பு¶

    Sentry Dashboard:

    • புதிய பிழைகளுக்கு வாராந்திர சரிபார்ப்பு
    • பிழை போக்குகளை மதிப்பாய்வு செய்யவும்
    • முக்கியமான சிக்கல்களை உடனடியாக சரிசெய்யவும்
    • உயர் முன்னுரிமை பிழைகளுக்கு விழிப்பூட்டல்களை அமைக்கவும்

    Google Cloud Console:

    • quota-க்குள் இருக்க API பயன்பாட்டை கண்காணிக்கவும்
    • பயன்பாட்டில் அசாதாரண கூர்முனைகளை சரிபார்க்கவும்
    • billing விழிப்பூட்டல்களை மதிப்பாய்வு செய்யவும்
    • APIs இயக்கப்பட்டிருக்கின்றனவா என உறுதிப்படுத்தவும்

    செயல்திறன்:

    • Lighthouse தணிக்கைகளை அவ்வப்போது இயக்கவும்
    • பக்க ஏற்றல் நேரங்களை கண்காணிக்கவும்
    • Core Web Vitals சரிபார்க்கவும்
    • மெதுவான இணைப்புகளில் சோதிக்கவும்

    பயனர் கருத்து:

    • சபை உறுப்பினர்களை கேட்கவும்
    • பொதுவான சிக்கல்களை கண்காணிக்கவும்
    • தீர்வுகளை ஆவணப்படுத்தவும்
    • தேவைப்படும்போது ஆவணங்களை புதுப்பிக்கவும்

    பதிப்பு புதுப்பிப்புகள்¶

    Ministry Mapper semantic versioning பயன்படுத்துகிறது:

    • முக்கிய பதிப்பு (x.0.0): மாற்றங்களை உடைக்கும்
    • சிறு பதிப்பு (1.x.0): புதிய அம்சங்கள்
    • இணைப்பு பதிப்பு (1.9.x): பிழை திருத்தங்கள்

    அடுத்த படிகள்¶

    பயனர்களுக்கு:

    • பயன்பாட்டை எவ்வாறு பயன்படுத்துவது என்று கற்றுக்கொள்ள பயனர் வழிகாட்டி படிக்கவும்

    நிர்வாகிகளுக்கு:

    • PocketBase பின்னணியை அமைக்கவும் (ministry-mapper-be களஞ்சியம் பார்க்கவும்)
    • சபை அமைப்புகளை உள்ளமைக்கவும்
    • பயனர்களை அழைத்து பங்குகளை ஒதுக்கவும்
    • பகுதிகளை உருவாக்கி முகவரிகளை சேர்க்கவும்
