    Ministry Mapperユーザーガイド¶

    はじめに¶

    Ministry Mapperへようこそ。これは、会衆が野外奉仕の区域を効率的に管理するために設計された最新のウェブベースアプリケーションです。このガイドでは、始め方からアプリケーションを最大限に活用する方法まで、知っておくべきすべてのことを説明します。

    Ministry Mapperとは？

    Ministry Mapperは、従来の紙ベースの方法に代わるデジタル区域管理システムです。会衆が以下のことを可能にします：

    • 区域をデジタルで整理し、割り当てる
    • 野外奉仕の訪問をリアルタイムで追跡
    • 複数の伝道者間で活動を調整
    • インターネットに接続された任意のデバイスから区域にアクセス

    主な利点：

    • ✓ エコフレンドリー - 紙の無駄を排除
    • ✓ クラウド同期によるリアルタイム更新
    • ✓ 任意のデバイス（デスクトップ、タブレット、モバイル）で動作
    • ✓ 簡単なナビゲーションのための統合インタラクティブマップ
    • ✓ 安全なロールベースのアクセス制御

    はじめに¶

    アカウントの作成¶

    図1：新しいMinistry Mapperアカウントを作成するための登録フォーム

    ステップ1：登録ページにアクセス

    図2：サインアップオプションのあるログインページ

    1. 会衆のMinistry Mapper URLにアクセス
    2. ログインページの「サインアップ」ボタンをクリック

    ステップ2：登録方法の選択

    従来のサインアップ Google OAuth（推奨）
    メール、パスワード（6文字以上、数字1つ、大文字1つ）、メール確認が必要 Googleアカウントを使用したワンクリック登録
    手動でのメール確認が必要 自動メール確認
    別のパスワードを覚える必要がある パスワード管理不要
    プライバシーポリシーと利用規約に同意 Googleによる強化されたセキュリティ

    従来のサインアップ： 1. 入力：名前、メール、パスワード、パスワード確認 2. プライバシーポリシーと利用規約に同意 3. 「アカウントを作成」をクリック

    Google OAuthサインアップ： 1. 「または次で続行」の下の「Googleでサインイン」ボタンをクリック 2. Googleアカウントを選択 3. Ministry Mapperに基本的なプロフィールアクセスを許可 4. アカウントが自動的に作成

    ステップ3：メールの確認

    図3：アカウント作成後に送信されるメール確認メッセージ

    1. Ministry Mapperからの確認メッセージがメール受信トレイにあるか確認
    2. メール内の確認リンクをクリック
    3. アカウントが確認されたという確認が表示されます

    ステップ4：会衆アクセスを待つ

    確認後：

    1. ログインページに戻る
    2. メールとパスワードでサインイン
    3. ワンタイムパスワード（OTP）が有効な場合は、メールでコードを確認

    図4：ワンタイムパスワード（OTP）確認画面

    図5：ログイン確認用のOTPコードが含まれるメール

    1. 重要：まだ区域は表示されません - 管理者が最初に会衆へのアクセスを許可する必要があります
    2. 会衆の区域担当者または管理者に連絡してアクセスをリクエスト

    Ministry Mapperへのログイン¶

    図6：より速く安全な認証のためのGoogle OAuthサインインオプション

    アカウントが確認され、会衆アクセスが付与されたら：

    標準ログイン： 1. 会衆のMinistry Mapper URLに移動 2. メールアドレスとパスワードを入力 3. 「サインイン」をクリック

    Google OAuthログイン（より速い）： 1. 「Googleでサインイン」ボタンをクリック 2. Googleアカウントを選択 3. 自動的にサインイン（パスワード不要）

    ステップ3：OTP確認を完了（有効な場合）

    会衆がワンタイムパスワードセキュリティを有効にしている場合：

    1. 資格情報を入力後、OTP確認画面が表示されます
    2. メールで確認コードを確認
    3. メールからの6桁のコードを入力
    4. 「確認」または「送信」をクリック
    5. コードは5〜10分で期限切れ

    ステップ4：ダッシュボードにアクセス

    • 伝道者：ダッシュボードは表示されません - 送られた割り当てリンクを使用します
    • 読み取り専用、司会者、管理者：ロール固有のダッシュボードが表示されます

    💡 ヒント：信頼できるデバイスではログイン状態を維持すると便利ですが、共有コンピューターでは必ずログアウトしてください。

    ユーザーロールの理解¶

    Ministry Mapperは4層のアクセス制御システムを使用しています。ロールによって、アクセスできる機能と実行できるアクションが決まります。

    ロール階層¶

    管理者（フルアクセス）
    ↓
司会者（割り当て管理）
    ↓
読み取り専用（閲覧のみ）
    ↓
伝道者（リンクアクセスのみ）

    👤 伝道者¶

    アクセス方法：管理者または司会者から送られた割り当てリンク経由

    伝道者ができること：

    • ✓ 共有リンク経由で区域にアクセス
    • ✓ インタラクティブマッピングで区域マップを表示
    • ✓ 訪問後に住所ステータスを更新
    • 完了、不在、訪問不可、無効としてマーク
    • ✓ 住所に訪問メモを追加
    • ✓ 「不在」の試行回数を追跡
    • ✓ 住所の詳細と順序を表示

    伝道者ができないこと：

    • ✗ ダッシュボードにアクセスできない
    • ✗ すべての会衆区域を表示できない
    • ✗ 区域を作成・管理できない
    • ✗ 割り当てリンクは設定された時間後に期限切れ（デフォルト：24時間、会衆管理者が設定可能）

    最適な対象：野外奉仕を行う通常の伝道者

    👓 読み取り専用¶

    アクセス方法：読み取り専用権限でのダッシュボードログイン

    読み取り専用ユーザーができること：

    • ✓ 会衆のすべての区域を表示
    • ✓ 完全な住所情報を確認
    • ✓ 区域の進捗と統計を表示
    • ✓ 区域マップにアクセス
    • ✓ 会衆メッセージを表示

    読み取り専用ユーザーができないこと：

    • ✗ 区域や住所データを変更できない
    • ✗ 区域を作成・削除できない
    • ✗ 割り当てを管理できない
    • ✗ 会衆設定を変更できない

    最適な対象：編集機能なしで可視性が必要な監督者

    🎯 司会者¶

    アクセス方法：司会者権限でのダッシュボードログイン

    司会者の能力には以下が含まれます：

    • ✓ 読み取り専用のすべての機能、さらに：
    • ✓ 区域割り当ての作成と管理
    • ✓ 伝道者用の割り当てリンクを生成
    • ✓ すべての割り当て履歴を表示
    • ✓ 会衆メッセージへのアクセスと投稿
    • ✓ 会衆オプション（世帯ステータスタイプ）の管理
    • ✓ 区域完了状況の表示

    司会者ができないこと：

    • ✗ 区域を作成・削除できない
    • ✗ 住所の詳細（住所、ユニット、階）を編集できない
    • ✗ ユーザーロールや権限を管理できない
    • ✗ コア会衆設定を変更できない

    最適な対象：野外奉仕コーディネーターとグループ監督者

    👑 管理者（区域担当者）¶

    アクセス方法：完全な管理権限でのダッシュボードログイン

    管理者は完全な制御を持ちます：

    • ✓ 司会者のすべての機能、さらに：
    • ✓ 区域の作成、編集、削除
    • ✓ 住所の追加、変更、削除
    • ✓ 建物とユニットの管理
    • ✓ 会衆設定の構成
    • 最大「不在」試行回数の設定
    • リンク有効期限の設定
    • 世帯タイプオプションの設定
    • ✓ ユーザーの招待と管理
    • ✓ ユーザーロールと権限の割り当て
    • ✓ 区域と住所のリセット
    • ✓ ジオロケーション座標の管理

    最適な対象：区域担当者と会衆の区域システムを管理する人

    💡 注意：ロールの変更が必要な場合や、現在のロールが不明な場合は、会衆管理者に連絡してください。

    主な機能¶

    ダッシュボード概要¶

    図6：区域セレクターとメインコントロールを表示する管理者ダッシュボード

    ダッシュボードインターフェースはロールによって異なります：

    伝道者の場合¶

    伝道者はダッシュボードにアクセスできません。代わりに：

    • メールまたはメッセージで割り当てリンクを受け取る
    • リンクをクリックして割り当てられた区域にアクセス
    • 区域ビュー内で直接作業
    • リンクは設定された時間後に自動的に期限切れ（デフォルト：24時間）

    司会者と管理者の場合¶

    ダッシュボードは包括的な概要を提供します：

    1. 区域セレクター（上部ドロップダウン）

    • 表示または管理する区域を選択
    • 区域コードと説明を表示
    • 区域間のクイックナビゲーション

    2. メインアクションボタン

    • 📋 割り当て：割り当てリンクの表示と管理
    • 💬 メッセージ：会衆メッセージの投稿と表示
    • ⚙️ 設定：会衆構成へのアクセス（管理者のみ）
    • 👥 ユーザー：ユーザーロールと招待の管理（管理者のみ）

    3. 区域情報パネル

    • 区域コードと説明
    • 完了率を示すプログレスバー
    • 最終更新のタイムスタンプ
    • 総ユニット数対完了ユニット数

    4. 区域表示オプション

    • 🗺️ マップビュー：インタラクティブマップ表示
    • 📋 リストビュー：すべての住所の表形式表示

    5. スピードダイヤルメニュー（フローティングアクションボタン）

    図6a：一般的なアクションへのクイックアクセスを提供するスピードダイヤルフローティングアクションボタン

    ➕ボタン（右下隅）はクイックアクセスを提供します： - 🗺️ マップビューモード：フルスクリーンマップビューの切り替え - 📋 クイックリンク：割り当てリンクの迅速な作成（司会者と管理者） - 現在のビューに基づくコンテキスト対応アクション

    区域の表示¶

    伝道者区域ビュー¶

    図7：マップと住所リストを含む割り当てられた区域の伝道者ビュー

    割り当てへのアクセス：

    1. 管理者/司会者から送られた割り当てリンクをクリック
    2. 区域が自動的に読み込まれ、以下が表示されます：
    3. すべての住所を表示するインタラクティブGoogle Map
    4. 各場所のクリック可能なマーカー
    5. 訪問する住所/ユニットのリスト
    6. 現在の進捗率

    表示される区域情報：

    • 区域コード：識別子（例：「T-001」）
    • 説明：区域名またはエリア
    • プログレスバー：視覚的な完了状況
    • マップ：住所マーカー付きのインタラクティブマップ
    • 住所リスト：現在のステータスを含むすべての住所

    管理者/司会者区域ビュー¶

    図8：区域セレクターと管理オプションを含む司会者ダッシュボード

    区域の表示：

    1. ダッシュボードにログイン
    2. ドロップダウンメニューから区域を選択
    3. 詳細情報を表示：
    4. 区域コードと説明
    5. 完了統計（例：「15/20完了 - 75%」）
    6. すべての場所を含むインタラクティブマップ
    7. 詳細付きの完全な住所/ユニットリスト
    8. 管理ボタン（編集、削除、リセット）

    管理オプション：

    • ✏️ 区域を編集：名前、コード、説明を変更
    • 🗑️ 区域を削除：区域全体を削除
    • 🔄 区域をリセット：すべての住所ステータスをクリア
    • ➕ 住所を追加：区域に新しい住所を作成

    住所の操作¶

    住所情報の理解¶

    図9：すべての住所詳細と現在のステータスを表示する住所カード

    Ministry Mapperの各住所またはユニットには包括的な情報が表示されます：

    基本情報：

    • 住所/ユニット番号：場所識別子（例：「#05-123」）
    • 階：建物の階（複数階建ての物件の場合）
    • タイプ：会衆オプションに基づく世帯分類
    • 例：中国語、英語、タミル語、スペイン語
    • 設定されている場合は複数のタイプを選択可能
    • 順序：訪問順序番号

    ステータス情報：

    • ステータス：現在の訪問ステータス（以下のステータスタイプを参照）
    • 不在カウント：応答がなかった訪問試行回数
    • 訪問不可日：DNCがマークされた日付（該当する場合）

    アクティビティ追跡：

    • メモ：訪問や世帯主に関する重要な情報
    • 最終更新：最新の更新の日時
    • 更新者：最後の変更を行った人のユーザー名

    住所ステータスタイプ¶

    Ministry Mapperは5つの標準ステータスタイプを使用します：

    ステータス 色 説明 使用法
    未完了 白/デフォルト まだ訪問していない すべての住所の初期ステータス
    完了 緑 正常に連絡がとれた 世帯主が在宅で連絡がとれた
    不在 黄/オレンジ 誰も応答しなかった 最大試行回数まで追跡（設定可能）
    訪問不可 赤 訪問を拒否された 世帯主がこれ以上の連絡を拒否
    無効 グレー アクセス不可 住所が存在しないかアクセスできない

    💡 ヒント：「不在」が最大試行回数（管理者が設定）に達すると、住所は自動的に進捗計算で完了としてマークされます。

    住所ステータスの更新¶

    図10：すべてのフィールドとオプションを表示する更新ステータスモーダル

    ステップバイステップのプロセス：

    1. 住所を探す

    • 住所リストをスクロール
    • またはマップ上のマーカーをクリック
    • 複数階建ての建物では住所は階ごとにグループ化

    2. 更新モーダルを開く

    • 住所/ユニットカードをクリック
    • 現在の情報を含む更新モーダルが開く

    3. ステータスを更新（必須）

    適切なステータスを選択：

    📗 完了 - 正常に連絡がとれたとき

    • 誰かが応答したときに選択
    • 会話が行われたか、出版物が配置された
    • 「不在」カウントがリセットされる

    🏠 不在 - 誰も応答しなかったとき

    • 自動的に「不在」カウントが増加
    • システムが試行回数を追跡
    • 最大試行回数に達すると、完了として扱われる

    🚫 訪問不可 - 訪問しないよう要求されたとき

    図11：日付選択を含む訪問不可ステータス更新

    • このステータスを選択
    • オプションでDNC日を設定（デフォルトは今日）
    • 必要に応じて理由を説明するメモを追加
    • 重要：常に世帯主の希望を尊重

    ❌ 無効 - 住所にアクセスできないとき

    • 住所が存在しない
    • 建設中
    • 恒久的に閉鎖
    • アクセスできない

    4. 世帯タイプを更新（利用可能な場合）

    会衆が世帯タイプを設定している場合：

    • 単一または複数のタイプを選択
    • 例：言語の好み、特別な状況
    • 必要に応じて既存のタイプをクリア

    5. メモを追加または更新（オプションですが推奨）

    ベストプラクティス：

    • ✓ 個人ではなく、物件の詳細に焦点を当てる
    • ✓ アクセス手順とタイミングを記録
    • ✓ 簡潔で敬意を持って
    • ✗ 世帯主に関する個人情報を含めない
    • ✗ センシティブな情報を含めない

    良い例：

    • 「ゲート付き物件 - 最初に警備室に電話」
    • 「最適な時間：週末の午後2時以降」
    • 「ドライブウェイ経由でサイドエントランスにアクセス可能」

    悪い例：

    • 住人に関する個人的な詳細
    • 医療または財務情報
    • 個人の名前や説明

    6. 不在カウントを調整（必要な場合）

    システムは自動的に不在試行を追跡しますが、必要に応じて手動で調整できます。

    7. 訪問不可日を設定（DNCステータスのみ）

    訪問不可としてマークするとき：

    • システムはデフォルトで今日の日付
    • 特定のDNCリクエストに応じて調整
    • 日付は潜在的に再訪問するタイミングを追跡するのに役立つ

    8. ジオロケーションを更新（管理者のみ）

    一戸建て区域の場合：

    • 「ジオロケーションを更新」ボタンをクリック
    • マップを使用して正確な位置を設定
    • ナビゲーションとマッピングの精度を向上

    9. 変更を保存

    • すべての更新を確認
    • 「保存」ボタンをクリック
    • 変更はすべてのユーザーに即座に同期
    • 成功メッセージで更新を確認

    10. 物件を削除（管理者のみ - 一戸建て）

    削除が必要な個人物件の場合：

    • 下部の「物件を削除」ボタンをクリック
    • 削除を確認
    • 警告：このアクションは取り消せません

    不足している住所をその場で追加する¶

    マップに物理的な住所が不足していることに気づいた場合、マッピングビューを離れることなくその場で追加できます：

    1. マップ上の住所リストの一番下までスクロール
    2. リストの末尾にある 「+」 カードをタップ
    3. 表示されるフォームに住所の詳細を入力
    4. 「保存」 をタップ — 住所がすぐにマップに追加されます

    Note

    この機能は伝道者が利用でき、テリトリー記録をまだ整備中の会衆向けに設計されています。新しく追加された住所は管理者によるレビューのためにフラグが立てられます。

    マップ機能の使用¶

    図12：マーカーとナビゲーションコントロールを表示するインタラクティブマップビュー

    Ministry Mapperは直感的な区域ナビゲーションのためのインタラクティブマッピングを統合しています。

    マップ機能¶

    インタラクティブマーカー：

    • 各住所はマップ上にマーク
    • 🔴 赤いマーカーは目的地を示す
    • 🔵 青い点滅マーカーは現在地を示す
    • 任意のマーカーをクリックしてその住所を表示/編集

    マップコントロール：

    • ズーム：+と-ボタンまたはピンチジェスチャー
    • パン：クリックしてドラッグで移動
    • 衛星ビュー：マップと衛星画像を切り替え
    • フルスクリーン：マップをフルスクリーンに拡大（またはスピードダイヤル→マップビューモードを使用）
    • 区域を中心に：すべての住所を表示するようにビューをリセット

    フルスクリーンマップビューモード（管理者と司会者）：

    図12a：スピードダイヤルメニューからアクセスするフルスクリーンマップビューモード

    スピードダイヤル（➕）→マップビューアイコンでアクセス。以下に最適： - 野外奉仕前のルート計画 - 区域カバレッジ分析 - ミーティングのプレゼンテーション - 住所クラスターの特定

    住所への道順を取得する¶

    Ministry Mapper には、テリトリー内の住所に向かうためのルーティングサービスが組み込まれています：

    1. マップビューで住所リストから任意の住所をタップ
    2. 「道順」 ボタン（またはルートアイコン）をタップ
    3. 希望する移動モードを選択：
    4. 🚗 ドライブ — 車によるナビゲーション
    5. 🚶 ウォーキング — 歩行者ルート
    6. 🚴 サイクリング — 自転車対応ルート
    7. ルートがマップ上に表示され、ターンバイターンの案内が提供されます

    Tip

    人口密集エリアで活動する場合は、より正確な歩行者ルートを取得するために ウォーキング モードをご利用ください。

    ナビゲーションのヒント¶

    1. 出発前：

    2. マップを見てルートを計画

    3. 住所のクラスターを特定

    4. メモから特別なアクセス要件を確認

    5. 現場で：

    6. マップを使用して住所間を移動

    7. 最適なルーティングのために順序番号に従う

    8. 各訪問後にマーカーをタップしてステータスを迅速に更新

    9. 電話のGPSを使用：

    10. 位置情報サービスを有効化
    11. マップに現在位置が表示
    12. 次の住所に直接ナビゲート
    13. マップタイルがキャッシュされている場合はオフラインでも動作（制限あり）

    区域割り当て（司会者と管理者）¶

    Ministry Mapperはリンクベースの割り当てシステムを使用しています。司会者と管理者は、伝道者が区域にアクセスするために使用する共有可能なリンクを作成します。

    リンクベースの割り当ての理由¶

    • ✓ 簡単な配布：メール、メッセージ、テキストでリンクを送信
    • ✓ 自動期限切れ：リンクは設定された時間後に期限切れ（デフォルト：24時間）
    • ✓ アカウント不要：伝道者はリンクを通じて直接作業
    • ✓ セキュリティ：期限切れのリンクはアクセス不可
    • ✓ 追跡：誰がいつ何にアクセスしたかを確認

    割り当ての作成¶

    割り当てを作成する2つの方法：

    図13a：スピードダイヤルメニューからアクセスするクイックリンク作成インターフェース

    方法 アクセス 最適な用途
    クイックリンク スピードダイヤル（➕）→クイックリンク 現在の区域の迅速な作成
    標準 割り当てボタン→新規作成 すべてのオプションとカスタマイズ

    ステップ1：割り当て作成にアクセス

    • クイックリンク：スピードダイヤル（➕）→クイックリンクアイコンをクリック（現在の区域が事前入力）
    • 標準：「割り当て」→「新しい割り当てを作成」または「+」をクリック

    ステップ2：割り当てフォームに入力

    フィールド 説明 必須
    タイプ 通常の割り当てまたは個人スリップ はい
    区域 会衆の区域から選択 はい
    伝道者名 追跡用（名前なしでもリンクは機能） オプション
    リンク有効期限 リンクが期限切れになるまでの時間（デフォルト：24） はい

    ステップ3：生成して共有

    1. 「割り当てを作成」をクリック
    2. システムが一意のリンクを生成
    3. 以下の方法で伝道者とリンクを共有：
    4. メール
    5. テキストメッセージ
    6. インスタントメッセージアプリ
    7. 任意のコミュニケーション方法

    割り当てリンクの例：

    https://your-ministry-mapper.com/map/abc123xyz456

    マップ固有の割り当て作成¶

    中央の割り当てモーダルに移動せずに、マップビューから直接割り当てを作成できます。

    アクセス：

    1. 割り当てたい区域に移動
    2. いずれかをクリック：
    3. 「割り当て」ボタン - 通常の区域割り当てを作成
    4. 「個人」ボタン - 個人スリップ割り当てを作成

    通常の区域割り当て：

    図13b：マップ固有の通常割り当て作成フォーム

    「割り当て」をクリックして通常の区域割り当てを作成。フォームには以下が表示されます：

    • ヘッダーの区域情報
    • 伝道者名フィールド - 割り当てられた伝道者の名前を入力
    • キャンセルと確認ボタン

    個人スリップ割り当て：

    図13c：カレンダー付きのマップ固有の個人スリップ作成フォーム

    「個人」をクリックして個人スリップ割り当てを作成。フォームには以下が含まれます：

    • ヘッダーの区域情報
    • カレンダーピッカー - 割り当て日を選択
    • 伝道者名フィールド - 割り当てられた伝道者の名前を入力
    • キャンセルと確認ボタン

    利点：

    • ✓ 区域を表示しながら素早く割り当て作成
    • ✓ 割り当てモーダルに切り替える必要なし
    • ✓ 割り当てられる区域の即座のコンテキスト

    割り当ての管理¶

    図14：すべての区域のすべてのアクティブな割り当てを表示する割り当て管理モーダル

    割り当て管理インターフェースにより、司会者と管理者は会衆内のすべての既存の割り当てを表示および管理できます。

    すべての割り当てを表示：

    1. ダッシュボードから「割り当て」ボタンをクリック
    2. 割り当てモーダルが開き、すべてのアクティブな割り当てリンクを表示
    3. 中央のリストですべての区域の割り当てを表示

    表示される割り当て情報：

    各割り当てについて、以下が確認できます：

    • 区域コードと場所：場所の説明を含む区域識別子（例：「187A, Marsiling Road (M01)」）
    • 割り当てタイプ：通常の割り当ての場合は「Assign」
    • 伝道者名：割り当てられた人の名前（例：Jon、Erli、Pety）
    • 作成日時：割り当てが作成された日時（例：「Dec 22, 2025, 9:38 PM」）
    • 有効期限：割り当てリンクが期限切れになる日時（例：「Dec 23, 2025, 9:38 AM」または「11:38 PM」）
    • 削除ボタン：個々の割り当てを削除するゴミ箱アイコン（🗑️）

    割り当ての削除：

    1. リスト内で割り当てを見つける
    2. 割り当ての右側にあるゴミ箱アイコン（🗑️）ボタンをクリック
    3. プロンプトが表示されたら削除を確認
    4. 割り当てリンクは即座に伝道者にアクセス不可になる

    削除するタイミング：

    • 区域が伝道者から早期に返却された
    • 間違ったリンクが作成または送信された
    • 伝道者がもうアクセスを必要としない
    • 割り当てを別の人に再割り当てする必要がある
    • セキュリティの懸念または侵害

    割り当てモーダルを閉じる：

    • 下部の「キャンセル」ボタンをクリックしてモーダルを閉じ、ダッシュボードに戻る

    マップ固有の割り当て管理¶

    マップビューから直接、個々の区域の割り当てを表示および管理できます。

    アクセス：

    1. 区域に移動
    2. いずれかをクリック：
    3. 「割り当てリンク」 - 通常の区域割り当て用
    4. 「個人リンク」 - 個人スリップ割り当て用

    図14a：特定の区域の割り当てリンクモーダル

    図14b：特定の区域の個人リンクモーダル

    表示される情報：

    • 伝道者名
    • 作成日時と有効期限
    • 削除ボタン（🗑️）で割り当てを削除

    使用するタイミング：

    • 現在その区域を誰が割り当てられているか確認
    • 区域を表示しながら割り当てを削除
    • 通常と個人の割り当てを別々に管理

    割り当てのベストプラクティス¶

    作成前：

    • ✓ 区域が準備できていることを確認（住所が更新され、指示が明確）
    • ✓ デフォルトの有効期限について会衆設定を確認
    • ✓ 区域サイズに適した有効期限を計画

    共有時：

    • ✓ メッセージに指示を含める
    • ✓ 伝道者に有効期限を通知
    • ✓ 質問がある場合の連絡先を提供
    • ✓ 適切な時間帯に送信

    監視：

    • ✓ 期限切れの割り当てを定期的に確認
    • ✓ 古い割り当てを定期的にクリーンアップ
    • ✓ 区域が返却されない場合はフォローアップ
    • ✓ 完了率を追跡

    メッセージと指示¶

    図15：固定オプション付きの投稿メッセージを表示するメッセージモーダル

    司会者メッセージ：

    • 司会者と管理者に表示

    • 管理メモ

    管理者メッセージ：

    • 管理者のみに表示
    • システム管理メモ

    1. 「メッセージ」ボタンをクリック
    2. 「新しいメッセージ」または「+」をクリック
    3. メッセージを入力
    4. メッセージタイプを選択（伝道者/司会者/管理者）

    会衆オプション（世帯タイプ）¶

    図20：世帯タイプ設定を表示する会衆オプション管理

    区域用のカスタム世帯分類タイプを設定します。

    会衆オプションとは？

    • 世帯を分類するためのカスタムカテゴリ
    • 例：言語グループ（中国語、英語、タミル語）
    • 会衆が使用する任意の分類システムを表すことができる
    • 設定されている場合、単一の世帯に複数のタイプを割り当て可能

    オプションの管理：

    1. オプションを表示

    2. 設定で「会衆オプション」セクションを見つける

    3. 設定されているすべてのタイプのリストを表示

    4. 新しいオプションを追加

    5. 「オプションを追加」または「+」をクリック

    6. 入力：
      • コード：短い識別子（例：「CHI」、「ENG」）
      • 説明：フルネーム（例：「中国語」、「英語」）
      • カウント可能か：区域進捗にカウントするかチェック
      • デフォルトか：デフォルト選択にするかチェック
      • 順序：表示順序番号

    7. 「保存」をクリック

    8. オプションを編集

    9. 既存のオプションをクリック

    10. フィールドを変更

    11. 変更を保存

    12. オプションを削除

    13. オプションの削除ボタンをクリック
    14. 削除を確認
    15. 警告：このタイプを使用しているすべての住所に影響

    オプションフラグ：

    カウント可能か： - このタイプの住所が区域進捗にカウントされるかを制御 - チェック済み：進捗計算に含まれる（例：中国語、英語、タミル語） - 未チェック：進捗から除外（例：ビジネス、建設中） - 例：総住所100、「ビジネス」（カウント不可）としてマーク10 = 進捗は90住所のみに基づく

    デフォルトか： - 新しい住所を作成するときにこのタイプを自動選択 - デフォルトは1つのオプションのみにする - 最も一般的な世帯タイプに使用

    順序： - ドロップダウンメニューでの表示順序を制御 - 小さい数字が先に表示（1、2、3...） - 設定でドラッグ＆ドロップで管理 - 単にオプションをドラッグして並べ替え - 住所世帯ドロップダウンでの順序に影響 - ヒント：最も一般的なものから最も珍しいものへ順序付け

    複数選択の設定：

    • 世帯が複数のタイプを持てる場合に有効化
    • 例：世帯が中国語と英語の両方を話す
    • 無効時、世帯ごとに1つのタイプのみ

    マップ設定¶

    図34：管理者向けの高度なマップ設定オプション

    会衆でのマップの表示と動作を設定します。管理者は区域のメンテナンスと組織化のための強力なマップ管理機能にアクセスできます。

    マップ設定にアクセスするには：

    1. 設定に移動（管理者のみ）
    2. ドロップダウンから区域を選択
    3. 選択した区域のマップビューを開く
    4. マップ設定メニューにアクセス（通常は設定アイコンまたはメニュー経由）

    利用可能なマップ設定機能：

    1. 場所の変更 - マップマーカーを別の住所に移動 - 区域ユニットの地理的位置を更新 - 住所が変更されたか、初期位置が間違っていた場合に便利 - マップ上で新しい位置を選択するだけ

    2. 区域の変更 - 住所またはユニットを別の区域に移動 - 区域境界の再編成に役立つ - 区域サイズのバランスを取るのに便利 - 転送中にすべての住所データと履歴を維持

    3. 順序の変更 - 住所の訪問順序番号を変更 - 野外奉仕効率のためにルートを最適化 - 小さい数字が先に訪問 - 論理的な訪問パターンを作成するのに役立つ

    マップ順序の更新（ドラッグ＆ドロップ）¶

    区域内のすべてのマップ順序を並べ替えるためのドラッグ＆ドロップインターフェース

    アクセス： 住所メニュー → 「順序の変更」

    仕組み： 1. 各カードには現在の順序番号付きの住所が表示 2. ドラッグ＆ドロップして並べ替え - 番号は自動的に更新 3. 「保存」をクリックして適用、または「キャンセル」で破棄

    ベストプラクティス： - 近くの住所をグループ化してバックトラッキングを最小化 - 複数階建ての建物では階をまとめる - 端から端への論理的なフローを作成 - 順序付け後にマップビューを確認

    4. 名前の変更 - 住所/ユニットの名前または識別子を変更 - 建物名やユニット番号を更新 - データを実世界の変更に合わせて最新に保つ - 初期入力エラーの修正に便利

    5. ユニット番号を追加 - 既存の住所に新しいユニット番号を追加 - 追加ユニットで複数階建ての建物を拡張 - 建物に新しいアパートが追加されたときに便利 - 建物構造の組織を維持

    6. 上の階を追加 - 追加の階で建物を上方に拡張 - 拡張された建物や最初に過小評価された建物用 - 建物パターンに基づいて新しい階のユニットを自動作成 - 区域データを建設変更に合わせて最新に保つ

    7. 下の階を追加 - 現在の最低階より下に階を追加 - 地下レベルや新しくアクセス可能になった下の階に便利 - 負の階番号を追加可能（例：B1、B2） - 一貫した階番号システムを維持

    8. ステータスをリセット - 住所のステータスを「未完了」にクリア - 「完了」と「不在」ステータスのみを削除 - 「訪問不可」や「無効」ステータスは削除しない - メモやその他の住所情報を保持 - 以前に完了した住所での作業を再開するときに便利 - 区域内の他の住所には影響しない

    9. 削除 - 区域から住所、ユニット、または階を恒久的に削除 - 取り消し不可 - 注意して使用 - 重複エントリや存在しない住所の削除に役立つ - 削除前に慎重に確認

    一般的な使用例： - エラーの修正：名前の変更と場所の変更 - 区域のリバランス：区域の変更 - 建物の更新：上/下の階を追加 - ルート最適化：順序の変更 - データクリーンアップ：重複の削除 - 季節の更新：ステータスのリセット

    ⚠️ 警告：変更はすべてのユーザーに即座に影響します。削除は恒久的で取り消しできません。

    住所タイプ¶

    Ministry Mapperは2種類の住所をサポートしています：

    • アパート、マンション

    • 各階に複数のユニット

    2. 個人住所（一戸建て）

    • 個別の住宅、ショップハウス
    • 1つの住所を持つ単一の物件
    • 例：戸建て物件、独立した建物
    • 階/ユニット構造なし

    公共住所の追加（複数階建て）¶

    ステップ1：作成を開始

    1. 区域を選択

    • 郵便番号/住所：建物識別子
    • 郵便番号または通りの住所を入力
    • システムが位置を自動入力する可能性あり
    • ジオコーディングとマップ表示に使用
    • 建物名：オプションの建物名
    • 例：「ブロック123A」、「サニーハイツ」
    • 伝道者が建物を識別するのに役立つ

    ステップ3：階を設定

    図25：階とユニットの組織を備えた複数階建ての建物管理

    • 開始階：最下階の番号
    • 地下レベルの場合は負の値可能
    • 例：-2（B2）、1（地上）、0
    • 最上階：最上階の番号
    • 最大：50
    • 例：10、25、40
    • 階の選択：特定の階を選択
    • ユニットのない階をスキップ（例：機械階）

    プライバシーとセキュリティ¶

    情報の保護¶

    Ministry Mapperはセンシティブな住所と個人情報を扱います。以下のガイドラインを遵守してください：

    アカウントセキュリティ：

    • ✓ ログイン資格情報を誰とも共有しない
    • ✓ 強力でユニークなパスワードを使用（最小6文字、数字と大文字を含む）
    • ✓ 共有デバイスでは常にログアウト
    • ✓ 利用可能な場合はOTPを有効化して追加のセキュリティ
    • ✓ 不審なアクティビティを直ちに報告

    データプライバシー：

    • ✓ メモには必要な情報のみを記録
    • ✓ すべての説明で敬意を持ち事実に基づく
    • ✓ センシティブな個人データなし（医療、財務など）
    • ✓ プライバシーに関する世帯主のリクエストに従う
    • ✓ プライバシー法を遵守（GDPR、CCPA、地域の規制）

    法的コンプライアンス：

    • ⚠️ GDPR（ヨーロッパ）：個人データ保護要件
    • ⚠️ CCPA（カリフォルニア）：消費者プライバシー権
    • ⚠️ LGPD（ブラジル）：データ保護規制
    • ⚠️ 地域の法律：地域の要件を確認

    保存される情報¶

    ユーザーデータ：

    • アカウント詳細（名前、メール、確認ステータス）
    • 会衆ロール割り当て
    • 作成/アクセスした割り当てリンク
    • アクティビティタイムスタンプ

    区域データ：

    • 住所とユニット情報
    • ステータス更新と履歴
    • メモと訪問情報
    • ジオロケーション座標
    • 進捗追跡

    システムデータ：

    • ログインセッション
    • リアルタイムサブスクリプション
    • メッセージ履歴
    • 設定設定

    会衆の野外奉仕活動をサポートするためにMinistry Mapperをご利用いただきありがとうございます。このモダンなウェブベースのソリューションは、区域管理に効率性、コラボレーション、環境への利点をもたらします。
