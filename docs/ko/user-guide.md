# Ministry Mapper 사용자 가이드

## 소개

Ministry Mapper에 오신 것을 환영합니다. 회중이 야외 봉사 구역을 효율적으로 관리할 수 있도록 설계된 현대적인 웹 기반 애플리케이션입니다. 이 가이드는 시작하고 애플리케이션을 최대한 활용하기 위해 알아야 할 모든 것을 안내합니다.

**Ministry Mapper란?**

Ministry Mapper는 전통적인 종이 기반 방법을 대체하는 디지털 구역 관리 시스템입니다. 회중이 다음을 수행할 수 있게 합니다:

- 구역을 디지털로 구성하고 배정
- 야외 봉사 방문을 실시간으로 추적
- 여러 전도인 간 활동 조정
- 인터넷이 있는 모든 기기에서 구역 접근

**주요 이점:**

- ✓ 친환경 - 종이 낭비 제거
- ✓ 클라우드 동기화를 통한 실시간 업데이트
- ✓ 모든 기기에서 작동 (데스크톱, 태블릿, 모바일)
- ✓ 쉬운 네비게이션을 위한 통합 대화형 지도
- ✓ 안전한 역할 기반 접근 제어

## 시작하기

### 계정 만들기

![회원가입 페이지](assets/screenshots/sign_up.png)

*그림 1: 새 Ministry Mapper 계정 생성을 위한 등록 양식*

**1단계: 등록 페이지 접근**

![로그인](assets/screenshots/sign_in.png)

*그림 2: 회원가입 옵션이 있는 로그인 페이지*

1. 회중의 Ministry Mapper URL 방문
2. 로그인 페이지에서 **"회원가입"** 버튼 클릭

**2단계: 등록 방법 선택**

| 일반 회원가입 | Google OAuth (권장) |
|---------------------|---------------------------|
| 이메일, 비밀번호 (6자 이상, 숫자 1개, 대문자 1개), 이메일 인증 필요 | Google 계정을 사용한 원클릭 등록 |
| 수동 이메일 인증 필요 | 자동 이메일 인증 |
| 다른 비밀번호 기억 필요 | 비밀번호 관리 불필요 |
| 개인정보 보호정책 및 약관 동의 | Google을 통한 강화된 보안 |

**일반 회원가입:**
1. 입력: 이름, 이메일, 비밀번호, 비밀번호 확인
2. 개인정보 보호정책 및 서비스 약관 동의
3. **"계정 만들기"** 클릭

**Google OAuth 회원가입:**
1. "또는 계속하기" 아래의 **"Google로 로그인"** 버튼 클릭
2. Google 계정 선택
3. Ministry Mapper에 기본 프로필 접근 권한 부여
4. 계정 자동 생성

**3단계: 이메일 인증**

![회원가입 인증 이메일](assets/screenshots/sign_up_verification_email.png)

*그림 3: 계정 생성 후 전송된 이메일 인증 메시지*

3. 계정이 인증되었다는 확인 표시

1. 로그인 페이지로 돌아가기
2. 이메일과 비밀번호로 로그인
3. 일회용 비밀번호 (OTP)가 활성화된 경우 이메일에서 코드 확인

![OTP 인증 페이지](assets/screenshots/otp_verification_page.png)

*그림 4: 일회용 비밀번호 (OTP) 인증 화면*

![OTP 인증 이메일](assets/screenshots/otp_verification_email.png)

*그림 5: 로그인 인증을 위한 OTP 코드가 포함된 이메일*

4. **중요**: 아직 구역이 표시되지 않습니다 - 관리자가 먼저 회중 접근 권한을 부여해야 합니다
5. 접근 요청을 위해 회중의 구역 종 또는 관리자에게 연락하세요

### Ministry Mapper에 로그인

![Google OAuth 로그인](assets/screenshots/google_oauth_sign_in.png)

*그림 6: 더 빠르고 안전한 인증을 위한 Google OAuth 로그인 옵션*

계정이 인증되고 회중 접근 권한이 부여된 후:

**일반 로그인:**
1. 회중의 Ministry Mapper URL로 이동
2. 이메일 주소와 비밀번호 입력
3. **"로그인"** 클릭

**Google OAuth 로그인 (더 빠름):**
1. **"Google로 로그인"** 버튼 클릭
2. Google 계정 선택
3. 자동 로그인 (비밀번호 불필요)

**3단계: OTP 인증 완료 (활성화된 경우)**

회중에서 일회용 비밀번호 보안을 활성화한 경우:

1. 자격 증명 입력 후 OTP 인증 화면이 표시됩니다
2. 이메일에서 인증 코드 확인
3. 이메일의 6자리 코드 입력
4. **"인증"** 또는 **"제출"** 클릭
5. 코드는 5-10분 후 만료됩니다

**4단계: 대시보드 접근**

- **전도인**: 대시보드가 표시되지 않습니다 - 전송된 배정 링크 사용
- **읽기 전용, 인도자, 관리자**: 역할별 대시보드가 표시됩니다

> **팁**: 신뢰할 수 있는 기기에서는 편의를 위해 로그인 상태를 유지하되, 공용 컴퓨터에서는 항상 로그아웃하세요.

### 사용자 역할 이해하기

Ministry Mapper는 4단계 접근 제어 시스템을 사용합니다. 역할에 따라 접근할 수 있는 기능과 수행할 수 있는 작업이 결정됩니다.

#### 역할 계층

```
관리자 (전체 접근)
    ↓
인도자 (배정 관리)
    ↓
읽기 전용 (보기만)
    ↓
전도인 (링크 접근만)
```

---

#### 전도인

**접근 방법**: 관리자 또는 인도자가 보낸 배정 링크를 통해

**전도인이 할 수 있는 것:**

- ✓ 공유 링크를 통해 구역 접근
- ✓ 대화형 매핑으로 구역 지도 보기
- ✓ 방문 후 주소 상태 업데이트
  - 완료, 부재, 방문 금지, 유효하지 않음으로 표시
- ✓ 주소에 방문 메모 추가
- ✓ "부재" 시도 횟수 추적
- ✓ 주소 세부 정보 및 순서 보기

**전도인이 할 수 없는 것:**

- ✗ 대시보드 접근 없음
- ✗ 모든 회중 구역 볼 수 없음
- ✗ 구역 생성 또는 관리 불가
- ✗ 배정 링크 접근은 설정된 시간 후 만료됨 (기본값: 24시간, 회중 관리자가 구성 가능)

**최적 대상**: 야외 봉사를 하는 일반 전도인

---

#### 읽기 전용

**접근 방법**: 읽기 전용 권한이 있는 사용자 계정으로 대시보드 로그인

**읽기 전용 사용자가 할 수 있는 것:**

- ✓ 회중의 모든 구역 보기
- ✓ 완전한 주소 정보 보기
- ✓ 구역 진행 상황 및 통계 보기
- ✓ 구역 지도 접근
- ✓ 회중 메시지 보기

**읽기 전용 사용자가 할 수 없는 것:**

- ✗ 구역 또는 주소 데이터 수정 불가
- ✗ 구역 생성 또는 삭제 불가
- ✗ 배정 관리 불가
- ✗ 회중 설정 변경 불가

**최적 대상**: 편집 기능 없이 가시성이 필요한 감독자

---

#### 인도자

**접근 방법**: 인도자 권한이 있는 사용자 계정으로 대시보드 로그인

**인도자 기능:**

- ✓ 읽기 전용이 할 수 있는 **모든 것**, 추가로:
- ✓ 구역 배정 생성 및 관리
- ✓ 전도인을 위한 배정 링크 생성
- ✓ 모든 배정 이력 보기
- ✓ 회중 메시지 접근 및 게시
- ✓ 회중 옵션 (가구 상태 유형) 관리
- ✓ 구역 완료 상태 보기

**인도자가 할 수 없는 것:**

- ✗ 구역 생성 또는 삭제 불가
- ✗ 주소 세부 정보 (주소, 세대, 층) 편집 불가
- ✗ 사용자 역할 또는 권한 관리 불가
- ✗ 핵심 회중 설정 수정 불가

**최적 대상**: 야외 봉사 조정자 및 그룹 감독자

---

#### 관리자 (구역 종)

**접근 방법**: 전체 관리 권한이 있는 사용자 계정으로 대시보드 로그인

**관리자는 완전한 제어권 보유:**

- ✓ 인도자가 할 수 있는 **모든 것**, 추가로:
- ✓ 구역 생성, 편집 및 삭제
- ✓ 주소 추가, 수정 및 제거
- ✓ 건물 및 세대 관리
- ✓ 회중 설정 구성
  - 최대 "부재" 시도 횟수 설정
  - 링크 만료 시간 구성
  - 가구 유형 옵션 설정
- ✓ 사용자 초대 및 관리
- ✓ 사용자 역할 및 권한 할당
- ✓ 구역 및 주소 초기화
- ✓ 위치 좌표 관리

**최적 대상**: 구역 종 및 회중의 구역 시스템 관리자

---

> **참고**: 역할 변경이 필요하거나 현재 역할이 확실하지 않은 경우 회중 관리자에게 문의하세요.

## 주요 기능

### 대시보드 개요

![관리자 대시보드](assets/screenshots/admin_dashboard.png)

*그림 6: 구역 선택기와 주요 컨트롤이 표시된 관리자 대시보드*

대시보드 인터페이스는 역할에 따라 다릅니다:

#### 전도인의 경우

전도인은 **대시보드 접근 권한이 없습니다**. 대신:

- 이메일 또는 메시지로 배정 링크 수신
- 링크를 클릭하여 배정된 구역 접근
- 구역 보기 내에서 직접 작업
- 링크는 구성된 시간 후 자동 만료 (기본값: 24시간)

#### 인도자 및 관리자의 경우

대시보드는 종합적인 개요를 제공합니다:

**1. 구역 선택기** (상단 드롭다운)

- 보거나 관리할 구역 선택
- 구역 코드 및 설명 표시
- 구역 간 빠른 네비게이션

**2. 주요 작업 버튼**

- 📋 **배정**: 배정 링크 보기 및 관리
- 💬 **메시지**: 회중 메시지 게시 및 보기
- ⚙️ **설정**: 회중 구성 접근 (관리자만)
- 👥 **사용자**: 사용자 역할 및 초대 관리 (관리자만)

**3. 구역 정보 패널**

- 구역 코드 및 설명
- 완료 비율을 보여주는 진행률 바
- 마지막 업데이트 타임스탬프
- 총 세대 vs 완료된 세대

**4. 구역 보기 옵션**

- 🗺️ **지도 보기**: 대화형 지도 표시
- 📋 **목록 보기**: 모든 주소의 표 형식 표시

**5. 스피드 다이얼 메뉴** (플로팅 액션 버튼)

![스피드 다이얼 관리자 페이지](assets/screenshots/speeddial_admin_page.png)

*그림 6a: 일반적인 작업에 빠르게 접근할 수 있는 스피드 다이얼 플로팅 액션 버튼*

**➕** 버튼 (오른쪽 하단)은 다음에 빠르게 접근할 수 있습니다:
- 🗺️ **지도 보기 모드**: 전체 화면 지도 보기 전환
- 📋 **빠른 링크**: 배정 링크 빠르게 생성 (인도자 및 관리자)
- 현재 보기에 따른 컨텍스트 인식 작업

### 구역 보기

#### 전도인 구역 보기

![전도인 배정 화면](assets/screenshots/publisher_assignment_screen.png)

*그림 7: 지도와 주소 목록이 있는 전도인의 배정된 구역 보기*

**배정 접근:**

1. 관리자/인도자가 보낸 배정 링크 클릭
2. 다음과 함께 구역이 자동으로 로드됩니다:
   - 모든 주소를 보여주는 대화형 Google 지도
   - 각 위치의 클릭 가능한 마커
   - 방문할 주소/세대 목록
   - 현재 진행률 비율

**표시되는 구역 정보:**

- **구역 코드**: 식별자 (예: "T-001")
- **설명**: 구역 이름 또는 지역
- **진행률 바**: 시각적 완료 상태
- **지도**: 주소 마커가 있는 대화형 지도
- **주소 목록**: 현재 상태가 있는 모든 주소

#### 관리자/인도자 구역 보기

![인도자 대시보드](assets/screenshots/conductor_dashboard.png)

*그림 8: 구역 선택기와 관리 옵션이 있는 인도자 대시보드*

**구역 보기:**

1. 대시보드에 로그인
2. 드롭다운 메뉴에서 구역 선택
3. 자세한 정보 보기:
   - 구역 코드 및 설명
   - 완료 통계 (예: "15/20 완료 - 75%")
   - 모든 위치가 있는 대화형 지도
   - 세부 정보가 있는 완전한 주소/세대 목록
   - 관리 버튼 (편집, 삭제, 초기화)

**관리 옵션:**

- ✏️ **구역 편집**: 이름, 코드 또는 설명 변경
- 🗑️ **구역 삭제**: 전체 구역 제거
- 🔄 **구역 초기화**: 모든 주소 상태 지우기
- ➕ **주소 추가**: 구역에 새 주소 생성

### 주소 작업하기

#### 주소 정보 이해하기

![주소 상태 세부 정보](assets/screenshots/address_status_details.png)

*그림 9: 모든 주소 세부 정보와 현재 상태를 보여주는 주소 카드*

Ministry Mapper의 각 주소 또는 세대는 종합적인 정보를 표시합니다:

**기본 정보:**

- **주소/세대 번호**: 위치 식별자 (예: "#05-123")
- **층**: 건물 층 (다층 건물의 경우)
- **유형**: 회중 옵션에 따른 가구 분류
  - 예: 중국어, 영어, 타밀어, 스페인어
  - 구성된 경우 여러 유형 선택 가능
- **순서**: 방문 순서 번호

**상태 정보:**

- **상태**: 현재 방문 상태 (아래 상태 유형 참조)
- **부재 횟수**: 응답 없는 방문 시도 횟수
- **방문 금지 날짜**: DNC가 표시된 날짜 (해당되는 경우)

**활동 추적:**

- **메모**: 방문 또는 가구주에 대한 중요 정보
- **마지막 업데이트**: 가장 최근 업데이트 날짜 및 시간
- **업데이트한 사람**: 마지막 변경을 한 사용자 이름

#### 주소 상태 유형

Ministry Mapper는 5가지 표준 상태 유형을 사용합니다:

| 상태          | 색상         | 설명            | 용도                                      |
| --------------- | ------------- | ---------------------- | ------------------------------------------ |
| **미완료**    | 흰색/기본 | 아직 방문하지 않음        | 모든 주소의 초기 상태           |
| **완료**        | 녹색         | 성공적으로 연락됨 | 가구주가 집에 있어서 연락함         |
| **부재**    | 노란색/주황색 | 아무도 응답 안 함        | 최대 시도 횟수까지 추적 (구성 가능)       |
| **방문 금지** | 빨간색           | 방문 거부 요청   | 가구주가 더 이상의 방문을 원하지 않음   |
| **유효하지 않음**     | 회색          | 접근 불가           | 주소가 존재하지 않거나 방문 불가 |

> **팁**: "부재"가 최대 시도 횟수 (관리자가 설정)에 도달하면 주소가 진행률 계산에서 자동으로 완료로 표시됩니다.

#### 주소 상태 업데이트

![주소 부재 상태](assets/screenshots/address_not_home_status.png)

*그림 10: 모든 필드와 옵션을 보여주는 상태 업데이트 모달*

**단계별 프로세스:**

**1. 주소 찾기**

- 주소 목록 스크롤
- 또는 지도에서 마커 클릭
- 다층 건물의 경우 주소가 층별로 그룹화됨

**2. 업데이트 모달 열기**

- 주소/세대 카드 클릭
- 현재 정보가 있는 업데이트 모달이 열림

**3. 상태 업데이트** (필수)

적절한 상태 선택:

**📗 완료** - 성공적으로 연락한 경우

- 누군가 응답한 경우 선택
- 대화가 이루어졌거나 문서가 전달됨
- "부재" 카운트 증가 초기화됨

**🏠 부재** - 아무도 응답하지 않은 경우

- 자동으로 "부재" 카운트 증가
- 시스템이 시도 횟수 추적
- 최대 시도 도달 후 완료로 처리

**🚫 방문 금지** - 방문하지 말 것을 요청받은 경우

![주소 방문 금지 상태](assets/screenshots/address_do_not_call_status.png)

*그림 11: 날짜 선택이 있는 방문 금지 상태 업데이트*

- 이 상태 선택
- 선택적으로 DNC 날짜 설정 (기본값: 오늘)
- 적절한 경우 이유를 설명하는 메모 추가
- **중요**: 가구주의 요청을 항상 존중

**❌ 유효하지 않음** - 주소에 접근할 수 없는 경우

- 주소가 존재하지 않음
- 공사 중
- 영구적으로 폐쇄됨
- 접근 불가

**4. 가구 유형 업데이트** (가능한 경우)

회중에서 가구 유형을 구성한 경우:

- 단일 또는 여러 유형 선택
- 예: 언어 선호도, 특수 상황
- 필요한 경우 기존 유형 지우기

**5. 메모 추가 또는 업데이트** (선택 사항이지만 권장)

**모범 사례:**

- ✓ 개인이 아닌 건물 세부 정보에 집중
- ✓ 접근 지침 및 시간 기록
- ✓ 간결하고 정중하게
- ✗ 가구주에 대한 개인 정보 절대 포함하지 않음
- ✗ 민감한 정보 절대 포함하지 않음

**좋은 예:**

- "문이 잠긴 건물 - 경비실에 먼저 전화"
- "최적 시간: 주말 오후 2시 이후"
- "진입로를 통해 측면 입구 접근 가능"

**나쁜 예:**

- 거주자에 대한 개인 세부 정보
- 의료 또는 재정 정보
- 개인의 이름 또는 설명

**6. 부재 횟수 조정** (필요한 경우)

시스템이 자동으로 부재 시도를 추적하지만 필요한 경우 수동으로 조정할 수 있습니다.

**7. 방문 금지 날짜 설정** (DNC 상태만)

방문 금지로 표시할 때:

- 시스템이 오늘 날짜를 기본값으로 설정
- 특정 DNC 요청에 맞게 필요한 경우 조정
- 날짜는 잠재적 재방문 시기 추적에 도움

**8. 위치 정보 업데이트** (관리자만)

단층 구역의 경우:

- "위치 정보 업데이트" 버튼 클릭
- 지도를 사용하여 정확한 위치 설정
- 네비게이션 및 매핑 정확도에 도움

**9. 변경 사항 저장**

- 모든 업데이트 검토
- **"저장"** 버튼 클릭
- 변경 사항이 모든 사용자에게 즉시 동기화
- 성공 메시지로 업데이트 확인

**10. 건물 삭제** (관리자만 - 단층)

제거해야 하는 개인 건물의 경우:

- 하단의 "건물 삭제" 버튼 클릭
- 삭제 확인
- **경고**: 이 작업은 취소할 수 없습니다

### 지도 기능 사용하기

![지도 길 안내](assets/screenshots/map_directions.png)

*그림 12: 마커와 네비게이션 컨트롤이 있는 대화형 지도 보기*

Ministry Mapper는 직관적인 구역 네비게이션을 위해 대화형 매핑을 통합합니다.

#### 지도 기능

**대화형 마커:**

- 각 주소가 지도에 표시됨
- 🔴 빨간 마커는 목적지를 나타냄
- 🔵 파란 깜박이는 마커는 현재 위치를 나타냄
- 아무 마커나 클릭하여 해당 주소 보기/편집

**지도 컨트롤:**

- **줌**: + 및 - 버튼 또는 핀치 제스처
- **팬**: 클릭하고 드래그하여 이동
- **위성 보기**: 지도와 위성 이미지 간 전환
- **전체 화면**: 지도를 전체 화면으로 확장 (또는 스피드 다이얼 → 지도 보기 모드 사용)
- **구역에 중심 맞추기**: 모든 주소를 표시하도록 보기 재설정

**전체 화면 지도 보기 모드** (관리자 및 인도자):

![관리자 지도 보기 모드](assets/screenshots/admin_map_view_mode.png)

*그림 12a: 스피드 다이얼 메뉴를 통해 접근하는 전체 화면 지도 보기 모드*

스피드 다이얼 (➕) → 지도 보기 아이콘을 통해 접근. 다음에 이상적:
- 야외 봉사 전 경로 계획
- 구역 커버리지 분석
- 회의 프레젠테이션
- 주소 클러스터 식별

#### 네비게이션 팁

1. **집을 떠나기 전:**

   - 지도를 보고 경로 계획
   - 주소 클러스터 식별
   - 메모에서 특별한 접근 요구 사항 확인

2. **현장에서:**

   - 지도를 사용하여 주소 간 네비게이션
   - 최적 경로를 위해 순서 번호 따르기
   - 각 방문 후 마커를 탭하여 상태 빠르게 업데이트

3. **휴대폰 GPS와 함께 사용:**
   - 위치 서비스 활성화
   - 지도에 현재 위치 표시
   - 다음 주소로 직접 네비게이션
   - 지도 타일이 캐시된 경우 오프라인에서도 작동 (제한적)

### 구역 배정 (인도자 및 관리자)

Ministry Mapper는 링크 기반 배정 시스템을 사용합니다. 인도자와 관리자가 전도인이 구역에 접근할 수 있는 공유 가능한 링크를 생성합니다.

#### 링크 기반 배정의 이유

- ✓ **간편한 배포**: 이메일, 메시지 또는 문자로 링크 전송
- ✓ **자동 만료**: 설정된 시간 후 링크 만료 (기본값: 24시간)
- ✓ **계정 불필요**: 전도인은 링크를 통해 직접 작업
- ✓ **보안**: 만료된 링크는 접근 불가
- ✓ **추적**: 누가 무엇에 언제 접근했는지 확인

#### 배정 생성

**배정을 생성하는 두 가지 방법:**

![관리자 인도자 빠른 링크](assets/screenshots/admin_conductor_quicklink.png)

*그림 13a: 스피드 다이얼 메뉴를 통해 접근하는 빠른 링크 생성 인터페이스*

| 방법 | 접근 | 최적 용도 |
|--------|--------|----------|
| **빠른 링크** | 스피드 다이얼 (➕) → 빠른 링크 | 현재 구역에 대한 빠른 생성 |
| **표준** | 배정 버튼 → 새로 만들기 | 전체 옵션 및 사용자 정의 |

**1단계: 배정 생성 접근**

- **빠른 링크**: 스피드 다이얼 (➕) → 빠른 링크 아이콘 클릭 (현재 구역 미리 입력됨)
- **표준**: **"배정"** → **"새 배정 만들기"** 또는 **"+"** 클릭

**2단계: 배정 양식 작성**

| 필드 | 설명 | 필수 |
|-------|-------------|----------|
| **유형** | 일반 배정 또는 개인 전도지 | 예 |
| **구역** | 회중 구역에서 선택 | 예 |
| **전도인 이름** | 추적용 (이름 없이도 링크 작동) | 선택 |
| **링크 만료** | 링크 만료까지 시간 (기본값: 24) | 예 |

**3단계: 생성 및 공유**

1. **"배정 만들기"** 클릭
2. 시스템이 고유 링크 생성
3. 다음을 통해 전도인에게 링크 공유:
   - 이메일
   - 문자 메시지
   - 인스턴트 메시징 앱
   - 모든 통신 방법

**배정 링크 예:**

```
https://your-ministry-mapper.com/map/abc123xyz456
```

#### 지도별 배정 생성

중앙 배정 모달로 이동하지 않고 지도 보기에서 직접 배정을 생성합니다.

**접근:**

1. 배정하려는 구역으로 이동
2. 다음 중 하나를 클릭:

- 헤더에 구역 정보

- ✓ 배정 중인 구역의 즉각적인 컨텍스트

#### 배정 관리

![배정 관리](assets/screenshots/assignment_management.png)

*그림 14: 모든 구역의 활성 배정을 보여주는 배정 관리 모달*

배정 관리 인터페이스는 인도자와 관리자가 회중의 모든 기존 배정을 보고 관리할 수 있게 합니다.

**모든 배정 보기:**

1. 대시보드에서 **"배정"** 버튼 클릭
2. 모든 활성 배정 링크를 표시하는 배정 모달이 열림
3. 중앙 집중식 목록에서 모든 구역의 배정 보기

**표시되는 배정 정보:**

각 배정에 대해 다음을 볼 수 있습니다:

- **구역 코드 및 위치**: 위치 설명이 있는 구역 식별자 (예: "187A, Marsiling Road (M01)")
- **배정 유형**: 일반 배정의 경우 "배정"
- **전도인 이름**: 배정된 사람 이름 (예: Jon, Erli, Pety)
- **생성 날짜/시간**: 배정이 생성된 시간 (예: "2025년 12월 22일 오후 9:38")
- **만료 날짜/시간**: 배정 링크가 만료되는 시간 (예: "2025년 12월 23일 오전 9:38" 또는 "오후 11:38")
- **삭제 버튼**: 개별 배정을 제거하는 휴지통 아이콘 (🗑️)

**배정 삭제:**

1. 목록에서 배정 찾기
2. 배정 오른쪽의 **휴지통 아이콘** (🗑️) 버튼 클릭
3. 프롬프트가 표시되면 삭제 확인
4. 배정 링크가 전도인에게 즉시 접근 불가능해짐

**삭제해야 할 때:**

- 전도인이 구역을 조기에 반환함
- 잘못된 링크가 생성되거나 전송됨
- 전도인이 더 이상 접근이 필요하지 않음
- 배정을 다른 사람에게 재배정해야 함
- 보안 우려 또는 침해

**배정 모달 닫기:**

- 하단의 **"취소"** 버튼을 클릭하여 모달을 닫고 대시보드로 돌아가기

#### 지도별 배정 관리

지도 보기에서 직접 개별 구역의 배정을 보고 관리합니다.

**접근:**

1. 구역으로 이동
2. 다음 중 하나를 클릭:
   - **"배정 링크"** - 일반 구역 배정용
   - **"개인 링크"** - 개인 전도지 배정용

![일반 지도 배정](assets/screenshots/map_assign_links.png)

*그림 14a: 특정 구역의 배정 링크 모달*

![개인 지도 배정](assets/screenshots/map_personal_links.png)

*그림 14b: 특정 구역의 개인 링크 모달*

**표시되는 정보:**

- 전도인 이름
- 생성 및 만료 날짜/시간
- 배정을 제거하는 삭제 버튼 (🗑️)

**사용 시기:**

- 현재 누가 구역을 배정받았는지 확인
- 구역을 보면서 배정 제거
- 일반 및 개인 배정을 별도로 관리

#### 배정 모범 사례

**생성 전:**

- ✓ 구역이 준비되었는지 확인 (주소 업데이트됨, 지침 명확)
- ✓ 기본 만료 시간에 대한 회중 설정 확인
- ✓ 구역 크기에 적절한 만료 기간 계획

**공유할 때:**

- ✓ 메시지에 지침 포함
- ✓ 전도인에게 만료 시간 알림
- ✓ 질문을 위한 연락처 제공
- ✓ 합리적인 시간에 전송

**모니터링:**

- ✓ 만료된 배정 정기 확인

5. 변경 사항이 즉시 적용

### 회중 설정 (관리자만)

![회중 설정](assets/screenshots/congregation_settings.png)

*그림 19: 모든 구성 옵션이 있는 회중 설정 페이지*

회중에 맞게 Ministry Mapper 작동 방식을 구성합니다.

#### 설정 접근

1. **"설정"** 버튼 또는 ⚙️ 아이콘 클릭
2. 회중 구성 패널 보기

#### 주요 설정

**1. 최대 "부재" 시도 횟수**

- 기본값: 1
- 범위: 1-4회 시도
- 도달 시 주소가 진행률 계산에서 완료로 간주
- 구역이 완료로 표시되는 시점에 영향

**예:** 3으로 설정된 경우:

- 첫 번째 "부재": 횟수 = 1
- 두 번째 "부재": 횟수 = 2
- 세 번째 "부재": 횟수 = 3, 완료로 표시

**2. 배정 링크 만료 (시간)**

- 기본값: 24시간
- 범위: 1-168시간 (1주일)
- 배정 링크가 활성 상태로 유지되는 시간
- 새로 생성된 링크에 적용

**3. 회중 원점/위치**

- 회중의 위치 설정
- 지도 중심 맞추기 및 길 안내에 사용
- 도시 이름 또는 좌표 가능
- 경로 계획에 도움

**4. OTP (일회용 비밀번호)**

- 로그인 시 이메일 OTP 활성화/비활성화
- 추가 보안 계층 추가
- 사용자가 로그인 시 이메일로 코드 수신
- 민감한 회중 데이터에 권장

#### 회중 옵션 (가구 유형)

![가구 옵션 설정](assets/screenshots/household_options_settings.png)

*그림 20: 가구 유형 구성을 보여주는 회중 옵션 관리*

구역에 대한 사용자 정의 가구 분류 유형을 구성합니다.

**회중 옵션이란?**

- 가구 분류를 위한 사용자 정의 카테고리
- 예: 언어 그룹 (중국어, 영어, 타밀어)
- 회중이 사용하는 모든 분류 시스템을 나타낼 수 있음
- 구성된 경우 단일 가구에 여러 유형 할당 가능

**옵션 관리:**

1. **옵션 보기**

   - 설정에서 "회중 옵션" 섹션 찾기
   - 구성된 모든 유형 목록 보기

2. **새 옵션 추가**

   - "옵션 추가" 또는 "+" 클릭
   - 입력:
     - **코드**: 짧은 식별자 (예: "CHI", "ENG")
     - **설명**: 전체 이름 (예: "중국어", "영어")
     - **계산 가능**: 구역 진행률에 포함할지 체크
     - **기본값**: 기본 선택으로 설정할지 체크
     - **순서**: 표시 순서 번호
   - "저장" 클릭

3. **옵션 편집**

   - 기존 옵션 클릭
   - 필드 수정
   - 변경 사항 저장

4. **옵션 삭제**
   - 옵션의 삭제 버튼 클릭
   - 삭제 확인
   - **경고**: 이 유형을 사용하는 모든 주소에 영향

**옵션 플래그:**

**계산 가능:**
- 이 유형의 주소가 구역 진행률에 포함되는지 제어
- **체크됨**: 진행률 계산에 포함 (예: 중국어, 영어, 타밀어)
- **체크 해제**: 진행률에서 제외 (예: 사업체, 공사 중)
- 예: 총 100개 주소, 10개가 "사업체" (계산 불가)로 표시 = 진행률은 90개 주소 기준

**기본값:**
- 새 주소 생성 시 이 유형 자동 선택
- 하나의 옵션만 기본값이어야 함
- 가장 일반적인 가구 유형에 사용

**순서:**
- 드롭다운 메뉴의 표시 순서 제어
- 낮은 숫자가 먼저 나타남 (1, 2, 3...)
- 설정에서 **드래그 앤 드롭으로 관리** - 옵션을 드래그하여 순서 변경
- 주소 가구 드롭다운의 순서에 영향
- 팁: 가장 일반적인 것부터 가장 적은 것 순으로 정렬

**다중 선택 구성:**

- 가구에 여러 유형이 있을 수 있는 경우 활성화
- 예: 가구가 중국어와 영어 모두 사용
- 비활성화 시 가구당 하나의 유형만

#### 지도 구성

![지도 구성](assets/screenshots/map_configurations.png)

*그림 34: 관리자를 위한 고급 지도 구성 옵션*

회중에서 지도가 표시되고 동작하는 방식을 구성합니다. 관리자는 구역 유지 관리 및 구성을 위한 강력한 지도 관리 기능에 접근할 수 있습니다.

**지도 구성 접근:**

1. **설정**으로 이동 (관리자만)
2. 드롭다운에서 구역 선택
3. 선택한 구역의 지도 보기 열기
4. 지도 구성 메뉴 접근 (일반적으로 설정 아이콘 또는 메뉴를 통해)

**사용 가능한 지도 구성 기능:**

**1. 위치 변경**
   - 지도 마커를 다른 주소로 이동
   - 구역 단위의 지리적 위치 업데이트
   - 주소가 변경되었거나 초기 위치가 잘못된 경우 유용
   - 지도에서 새 위치 선택

**2. 구역 변경**
   - 주소 또는 단위를 다른 구역으로 이동
   - 구역 경계 재구성에 도움
   - 구역 크기 균형에 유용
   - 전송 중 모든 주소 데이터 및 이력 유지

**3. 순서 변경**
   - 주소의 방문 순서 번호 수정
   - 야외 봉사 효율성을 위한 경로 최적화
   - 낮은 숫자가 먼저 방문됨
   - 논리적인 방문 패턴 생성에 도움

##### 지도 순서 업데이트 (드래그 앤 드롭)

![지도 순서 업데이트](assets/screenshots/update_map_sequence.png)

*구역의 모든 지도 순서를 재정렬하는 드래그 앤 드롭 인터페이스*

**접근:** 주소 메뉴 → "순서 변경"

**작동 방식:**
1. 각 카드에 현재 순서 번호가 있는 주소 표시
2. 드래그 앤 드롭으로 재정렬 - 번호가 자동 업데이트
3. **"저장"**을 클릭하여 적용 또는 **"취소"**로 버리기

**모범 사례:**
- 근처 주소를 그룹화하여 되돌아가기 최소화
- 다층 건물에서 층을 함께 그룹화
- 한쪽 끝에서 다른 쪽으로 논리적 흐름 생성
- 순서 지정 후 지도 보기 검토

**4. 이름 변경**
   - 주소/단위의 이름 또는 식별자 변경
   - 건물 이름 또는 세대 번호 업데이트
   - 실제 변경 사항으로 데이터 최신 유지
   - 초기 입력 오류 수정에 유용

**5. 세대 번호 추가**
   - 기존 주소에 새 세대 번호 추가
   - 추가 세대로 다층 건물 확장
   - 건물에 새 아파트가 추가될 때 유용
   - 건물 구조 구성 유지

**6. 위층 추가**
   - 추가 층으로 건물 위로 확장
   - 확장되었거나 처음에 과소평가된 건물용
   - 건물 패턴에 따라 새 층에 대한 세대 자동 생성
   - 건설 변경에 따라 구역 데이터 최신 유지에 도움

**7. 아래층 추가**
   - 현재 가장 낮은 층 아래에 층 추가
   - 지하 층 또는 새로 접근 가능한 하층에 유용
   - 음수 층 번호 추가 가능 (예: B1, B2)
   - 일관된 층 번호 시스템 유지

**8. 상태 초기화**
   - 주소 상태를 "미완료"로 지우기
   - "완료" 및 "부재" 상태만 제거
   - "방문 금지" 또는 "유효하지 않음" 상태는 제거하지 않음
   - 메모 및 기타 주소 정보 보존
   - 이전에 완료된 주소에서 작업을 다시 시작할 때 유용
   - 구역의 다른 주소에는 영향 없음

**9. 삭제**
   - 구역에서 주소, 세대 또는 층을 영구적으로 제거
   - 실행 취소 불가 - 주의해서 사용
   - 중복 항목 또는 존재하지 않는 주소 제거에 도움
   - 삭제 전 신중히 확인

**일반적인 사용 사례:**
- 오류 수정: 이름 변경 및 위치 변경
- 구역 재균형: 구역 변경
- 건물 업데이트: 위층/아래층 추가
- 경로 최적화: 순서 변경
- 데이터 정리: 중복 삭제
- 계절별 업데이트: 상태 초기화

> **경고**: 변경 사항은 모든 사용자에게 즉시 영향을 미칩니다. 삭제는 영구적이며 실행 취소할 수 없습니다.

### 주소 관리 (관리자만)

![단층 지도](assets/screenshots/single_story_map.png)

*그림 24: 개인/단층 건물을 위한 주소 관리 인터페이스*

관리자는 구역 내 주소를 추가하고 관리할 수 있습니다.

#### 주소 유형

Ministry Mapper는 두 가지 유형의 주소를 지원합니다:

**1. 공공 주소 (다층)**

- 아파트 건물, 콘도미니엄
- 여러 층과 세대
- 예: HDB 아파트, 아파트 단지
- 각 층에 여러 세대

**2. 개인 주소 (단층)**

- 개별 주택, 상가 주택
- 하나의 주소를 가진 단일 건물
- 예: 단독 주택, 독립 건물
- 층/세대 구조 없음

#### 공공 주소 추가 (다층)

**1단계: 생성 시작**

1. 구역 선택
2. **"주소 추가"** 또는 **"+"** 버튼 클릭
3. **"공공 주소"** 유형 선택

**2단계: 건물 정보 입력**

- **우편번호/주소**: 건물 식별자
  - 우편번호 또는 도로명 주소 입력
  - 시스템이 위치를 자동 채울 수 있음
  - 지오코딩 및 지도 표시에 사용
- **건물 이름**: 선택적 건물 이름
  - 예: "블록 123A", "써니 하이츠"
  - 전도인이 건물 식별에 도움

**3단계: 층 구성**

![다층 지도](assets/screenshots/multi_story_map.png)

*그림 25: 층 및 세대 구성이 있는 다층 건물 관리*

- **시작 층**: 가장 낮은 층 번호
  - 지하 층의 경우 음수 가능
  - 예: -2 (B2), 1 (지상), 0
- **최상층**: 가장 높은 층 번호
  - 최대: 50
  - 예: 10, 25, 40
- **층 선택**: 특정 층 선택
  - 세대가 없는 층 건너뛰기 (예: 기계 층)
  - 일반적: 시작부터 최상층까지 모든 층

**4단계: 세대 구성**

- **층당 세대**: 각 층의 세대 수
  - 예: 8, 12, 16
  - 자동으로 세대 생성
- **세대 번호 형식**: 세대 번호 지정 방법
  - 패턴: 층 + 세대 (예: 01-01, 01-02)
  - 사용자 정의: 나중에 세대 번호 수동 입력

**5단계: 생성 및 채우기**

1. **"생성"** 클릭
2. 시스템이 모든 층과 세대 생성
3. 주소가 모든 세대와 함께 구역에 나타남

**예:**

- 건물: 블록 123
- 층: 1부터 12까지
- 층당 세대: 8
- 결과: 96개 세대 생성 (12층 × 8세대)

#### 개인 주소 추가 (단층)

**1단계: 생성 시작**

1. 구역 선택
2. **"주소 추가"** 또는 **"+"** 클릭
3. **"개인 주소"** 유형 선택

**2단계: 건물 정보 입력**

- **건물 우편번호/주소**: 고유 식별자
  - 도로명 주소 또는 우편번호
  - 각 건물이 하나의 레코드
- **건물 이름**: 선택적 집 이름
  - 예: "123 메인 스트리트", "빌라 선샤인"

**3단계: 위치 설정 (선택 사항)**

- **"위치 정보 설정"** 클릭
- 지도를 사용하여 정확한 위치 지정
- 네비게이션에 도움
- 나중에 업데이트 가능

**4단계: 생성**

1. **"건물 생성"** 클릭
2. 단일 주소 단위 생성됨
3. 구역 목록에 나타남

#### 기존 주소 관리

**주소 이름 편집:**

1. 주소가 있는 구역 선택
2. 주소의 편집 옵션 클릭
3. 이름/우편번호 업데이트
4. 변경 사항 저장

**우편번호 변경:**

1. 주소 편집 모드 접근
2. 우편번호 필드 업데이트
3. 지오코딩에 영향을 줄 수 있음
4. 저장하고 지도 위치 확인

**주소 초기화:**

- 주소의 모든 세대 상태 지우기
- 메모 보존됨
- 주소가 완전히 작업되었을 때 사용

**주소 삭제:**

- 전체 주소와 모든 세대 제거
- 실행 취소 불가
- 삭제 전 신중히 확인

#### 세대 관리 (공공 주소만)

**세대 추가:**

1. 주소 선택
2. **"세대 추가"** 클릭
3. 추가할 세대 번호 지정
4. 세대 자동 생성

**세대 삭제:**

1. 특정 세대 클릭
2. 모달에서 **"세대 삭제"** 버튼 클릭
3. 삭제 확인
4. 세대가 주소에서 제거됨

**세대 순서 변경:**

1. 세대 편집 모달 열기
2. 순서 번호 업데이트
3. 방문 순서에 영향
4. 낮은 번호가 먼저 방문됨

**층 추가/삭제:**

1. 층 관리 접근
2. 새 층 번호 추가
3. 또는 전체 층 제거
4. 층의 모든 세대에 영향

**세대 위치 정보 업데이트:**

- 세대에 특정 좌표 설정
- 대형 건물에 유용
- 정확한 네비게이션에 도움
- 선택적 기능

## 일반적인 문제 해결

### 로그인 문제

| 문제 | 증상 | 해결 방법 |
|-------|----------|-----------|
| **잘못된 비밀번호** | "잘못된 자격 증명" 오류 | 이메일 확인 • Caps Lock 확인 • "비밀번호 찾기" 사용 → 이메일로 재설정 |
| **계정 미인증** | "이메일 미인증" 메시지 | 받은편지함/스팸에서 인증 이메일 확인 • 링크 클릭 • 만료된 경우 새로 요청 |
| **회중 접근 없음** | 로그인했지만 구역이 표시되지 않음 | 관리자에게 연락하여 초대 및 역할 할당 요청 • 접근 권한 부여 후 로그아웃/로그인 |
| **OTP 문제** | OTP를 받지 못함/유효하지 않음 | 스팸 확인 • 코드는 5-10분 후 만료 • 새 코드 요청 • 이메일 주소 확인 |

## 개인정보 보호 및 보안

### 정보 보호

Ministry Mapper는 민감한 주소 및 개인 정보를 처리합니다. 다음 지침을 준수하세요:

**계정 보안:**

- ✓ **로그인 자격 증명을 절대 공유하지 않음**
- ✓ **강력하고 고유한 비밀번호 사용** (최소 6자, 숫자 및 대문자 포함)
- ✓ **공유 기기에서 항상 로그아웃**
- ✓ **가능한 경우 OTP 활성화** 추가 보안을 위해
- ✓ **의심스러운 활동 즉시 보고**

**데이터 개인정보 보호:**

- ✓ 메모에 **필요한 정보만 기록**
- ✓ 모든 설명에서 **정중하고 사실적**
- ✓ **민감한 개인 데이터 없음** (의료, 재정 등)
- ✓ 개인정보에 대한 **가구주 요청 따르기**
- ✓ **개인정보 보호법 준수** (GDPR, CCPA, 지역 규정)

**법적 준수:**

- ⚠️ **GDPR (유럽)**: 개인 데이터 보호 요구 사항
- ⚠️ **CCPA (캘리포니아)**: 소비자 개인정보 보호 권리
- ⚠️ **LGPD (브라질)**: 데이터 보호 규정
- ⚠️ **지역 법률**: 해당 지역의 요구 사항 확인

Ministry Mapper를 사용하여 회중의 야외 봉사 활동을 지원해 주셔서 감사합니다. 이 현대적인 웹 기반 솔루션은 구역 관리에 효율성, 협업 및 환경적 이점을 제공합니다.

| **전도인** | 링크로 접근 • 방문 후 즉시 업데이트 • 정중한 메모 작성 • 순서 따르기 |
| **인도자** | 배정 생성 • 진행 상황 모니터링 • 메시지 게시 • 활동 조정 |
| **관리자** | 설정 구성 • 구역 관리 • 역할 할당 • 보안 보장 |

### 누락된 주소 추가하기

구역 기록을 구축 중인 회중을 위해 Ministry Mapper는 전도인이 지도 작업 중 누락된 주소를 직접 추가할 수 있습니다.

**단계:**

1. 주소 목록의 맨 아래로 스크롤합니다
2. 목록 끝에 있는 **"+"** 카드를 탭합니다
3. 주소 세부 정보(주소 코드, 세대 정보 등)를 입력합니다
4. **"저장"** 버튼을 탭하여 주소를 추가합니다

![주소 목록 끝에 누락된 주소를 추가하는 "+" 카드](assets/screenshots/add_more_add.png)

> **참고:** 이 기능은 구역 기록을 아직 구축 중인 회중에 이상적입니다. 추가된 주소는 관리자 검토를 위해 즉시 동기화됩니다.

### 길 안내 받기

Ministry Mapper의 내장 라우팅 서비스를 사용하여 주소로 이동하는 방법을 확인할 수 있습니다.

**단계:**

1. 주소 목록이나 지도에서 원하는 **주소**를 탭합니다
2. 주소 모달에서 **"길 안내"** 버튼을 탭합니다
3. 이동 수단을 선택합니다:
   - 🚗 **자동차** — 도로 경로
   - 🚶 **도보** — 보행자 경로
   - 🚲 **자전거** — 자전거 경로
4. 선택한 경로가 지도에 표시됩니다

![이동 수단 옵션과 지도에 그려진 경로를 보여주는 라우팅 패널](assets/screenshots/map_routing.png)

> **팁:** 건물이 밀집된 지역에서는 도보 모드를 사용하면 보다 정확한 경로 안내를 받을 수 있습니다.
