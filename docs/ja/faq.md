# よくある質問（FAQ）

## 一般的な質問

### Ministry Mapper とは何ですか？

Ministry Mapper は、会衆がフィールド奉仕のテリトリーをデジタルで管理するのに役立つ最新のウェブアプリケーションです。紙のテリトリー伝票の代わりに、Ministry Mapper を使えば、インターネット接続のある任意のデバイスからすべてをデジタルで整理・追跡できます。フロントエンドに React、バックエンドに PocketBase、インタラクティブマッピングに Leaflet を使用しています。

### Ministry Mapper にアクセスするにはどうすればいいですか？

**推奨**：私たちのホストサービス **[ministry-mapper.com](https://ministry-mapper.com)** を使用してください。

手順：
1. [ministry-mapper.com](https://ministry-mapper.com) にアクセス
2. サインアップページでアカウントを作成
3. メールアドレスを確認
4. 会衆の管理者に適切な権限を依頼

**代替**：セルフホスト（推奨しません）- 詳細については[セルフホストガイド](self-hosting.md)を参照してください。セルフホストには相当な技術的専門知識と継続的なメンテナンスが必要です。

### Ministry Mapper は無料で使用できますか？

**ホストサービス**：料金とプランについては [ministry-mapper.com](https://ministry-mapper.com) をご覧ください。

**セルフホスト**：ソースコードは無料でオープンソースですが、以下を自分で用意する必要があります：
- フロントエンドのホスティング（例：Vercel、Netlify、AWS）
- PocketBase バックエンドのデプロイ
- オプション：エラー監視用の Sentry アカウント

注意：セルフホストのコスト（サーバー、API 費用、メンテナンス時間）は、ホストサービスの費用を超えることが多いです。

### Ministry Mapper はどの言語をサポートしていますか？

現在サポートされている言語：

- 英語
- 日本語
- 韓国語（한국語）
- 中国語（中文）
- インドネシア語（Bahasa Indonesia）
- マレー語（Bahasa Melayu）

インターフェースはブラウザの言語を自動検出します。アプリで言語を手動で切り替えることもできます。

### Ministry Mapper を使うのに技術知識は必要ですか？

**伝道者/ユーザーとして**：技術知識は必要ありません！スマートフォンやコンピュータを使えるなら、[ministry-mapper.com](https://ministry-mapper.com) のホストサービスを通じて Ministry Mapper を使用できます。

**セルフホスト管理者として**：はい、相当な技術知識が必要です：

- Linux サーバー管理の経験
- Docker とコンテナ化の理解
- 環境変数と設定の知識
- SSL/TLS 証明書の知識
- Webサーバー設定（Nginx/Apache）
- DNS とドメイン設定の知識

**セルフホストではなくホストサービスの使用を強く推奨します。** それでもセルフホストをご希望の場合は、[セルフホストガイド](self-hosting.md)を参照してください。

### Ministry Mapper はオフラインで動作しますか？

いいえ、アプリが正常に機能するにはインターネット接続が必要です。理由は以下の通りです：

- PocketBase バックエンドに接続する必要があります
- ユーザー間のリアルタイム更新には接続が必要です

WiFi が利用できない場合でも、モバイルデータ通信でうまく動作します。

## プライバシーと法律

### どのようなプライバシー法を考慮すべきですか？

**重要**：Ministry Mapper は住宅の住所を追跡するため、データプライバシー法の対象となる可能性があります。これらの法律は国や地域によって大きく異なります：

- **ヨーロッパ**：GDPR（一般データ保護規則）
- **カリフォルニア**：CCPA（カリフォルニア州消費者プライバシー法）
- **ブラジル**：LGPD（Lei Geral de Proteção de Dados）
- **その他多数**：各地の規制

**Ministry Mapper を使用する前に、地域の規制を十分に確認し、コンプライアンスを確保してください。** 特定の地域の適切な法的アドバイスについては、法律の専門家にご相談ください。

**ホストサービス**：[ministry-mapper.com](https://ministry-mapper.com) のホストサービスはコンプライアンス支援を提供することがありますが、地域の法律に従う責任はあなたにあります。

**セルフホスト**：すべてのプライバシー法コンプライアンスに対して完全に責任を負います。

### Ministry Mapper はどのようなデータを保存しますか？

**ユーザーデータ（PocketBase が管理）：**

- メールアドレス
- 名前
- パスワード（暗号化）
- ユーザーロールと権限

**テリトリーデータ：**

- テリトリーの境界と名前
- 住所
- 状態情報
- 世帯に関するメモ
- 割り当て履歴
- 更新タイムスタンプ

**技術データ（Sentry が有効な場合）：**

- エラーログ
- パフォーマンスメトリクス

すべてのデータ保存はセキュリティのベストプラクティスに準拠しています。ホストサービスのデータポリシーについては、[ministry-mapper.com](https://ministry-mapper.com) を確認してください。

## 技術的なセットアップ（セルフホスト）

!!! warning "セルフホストは推奨しません"
    以下のセクションは、Ministry Mapper をセルフホストしたい上級ユーザー向けです。**代わりに [ministry-mapper.com](https://ministry-mapper.com) のホストサービスを使用することを強く推奨します。** セルフホストには相当な技術的専門知識、継続的なメンテナンス、セキュリティの責任が必要です。
    
    完全なセルフホスト手順については、[セルフホストガイド](self-hosting.md)を参照してください。

### システム要件は何ですか？

**ホストサービスの使用の場合：**
- 最新のウェブブラウザ（Chrome、Firefox、Safari、Edge）
- インターネット接続
- メールアドレス
- モバイルデバイスまたはコンピュータ

**セルフホスト開発の場合：**

- Node.js >= 22.0.0
- npm パッケージマネージャ
- Git
- Docker（バックエンド用）

**セルフホスト本番環境の場合：**

- クラウドホスティングサービス（Railway、Render、DigitalOcean、AWS など）
- ドメイン名
- SSL/TLS 証明書
- PocketBase バックエンド（別途デプロイ）
- （オプション）監視用の Sentry アカウント
- （オプション）通知用のメールサービス

### どの環境変数が必要ですか？

**セルフホストのみ** - このセクションはセルフホストの場合にのみ適用されます。ホストサービスはすべての設定を自動的に処理します。

**フロントエンド（.env ファイル）：**

```
VITE_SYSTEM_ENVIRONMENT=local
VITE_VERSION=$npm_package_version
VITE_POCKETBASE_URL=your_pocketbase_backend_url
VITE_OPENROUTE_API_KEY=your_openrouteservice_key
VITE_LOCATIONIQ_API_KEY=your_locationiq_key
VITE_SENTRY_DSN=your_sentry_dsn (optional)
VITE_PRIVACY_URL=your_privacy_policy_url
VITE_TERMS_URL=your_terms_url
VITE_ABOUT_URL=your_about_page_url
```

**本番環境の場合：** 上記と同じ変数ですが、`VITE_SYSTEM_ENVIRONMENT=production`

詳細については[フロントエンドセットアップガイド](frontend-setup.md)を参照してください。

### フロントエンドはどのようにデプロイしますか？

**セルフホストのみ** - ホストサービスを使用している場合はスキップしてください。

1. **アプリケーションのビルド：**

   ```bash
   npm run build
   ```

2. 上記で指定した**環境変数を設定**

3. **`build/` フォルダをホスティングプロバイダーにデプロイ：**

   - Vercel：GitHub リポジトリを接続して環境変数を設定
   - Netlify：ビルドフォルダをドラッグ＆ドロップまたは Git 経由で接続
   - AWS S3：ビルドフォルダをアップロードして静的ウェブサイトホスティングを設定

4. **PocketBase バックエンドがデプロイされ**、`VITE_POCKETBASE_URL` で指定した URL でアクセス可能であることを確認

詳細な手順については[フロントエンドセットアップガイド](frontend-setup.md)と[セルフホストガイド](self-hosting.md)を参照してください。

### PocketBase バックエンドはどのように設定しますか？

**セルフホストのみ** - ホストサービスを使用している場合はスキップしてください。

バックエンドは別のリポジトリで管理されています：[ministry-mapper-be](https://github.com/rimorin/ministry-mapper-be)

**クイック概要：**

1. バックエンドリポジトリをクローン
2. 環境変数を設定（[バックエンドセットアップガイド](backend-setup.md)を参照）
3. Docker または Railway/Render を使用してデプロイ
4. スキーマでデータベースを初期化
5. メール通知用の SMTP を設定（オプション）

**詳細な手順：** [バックエンドセットアップガイド](backend-setup.md)と[セルフホストガイド](self-hosting.md)を参照してください。

**重要**：フロントエンドをデプロイする前に、バックエンドがデプロイされてアクセス可能である必要があります。

### Sentry 監視はどのように設定しますか？

**セルフホストのみ** - ホストサービスを使用している場合はスキップしてください。

1. [Sentry](https://sentry.io/) アカウントを作成
2. React プロジェクトを作成
3. 設定に進み DSN キーを取得
4. 以下の環境変数を設定：
   - `VITE_SENTRY_DSN`：Sentry プロジェクトの DSN
   - `VITE_SYSTEM_ENVIRONMENT`：本番環境では "production" に設定（トレーシングサンプルレートに影響）
   - `VITE_VERSION`：Sentry のリリース追跡に使用

注意：Sentry はオプションですが、本番デプロイには推奨されます。

## 開発

### アプリをローカルで実行するにはどうすればいいですか？

**開発者のみ** - このセクションは Ministry Mapper に貢献する開発者向けです。

1. **リポジトリをクローン：**

   ```bash
   git clone https://github.com/rimorin/ministry-mapper-v2.git
   cd ministry-mapper-v2
   ```

2. **依存関係をインストール：**

   ```bash
   npm install
   ```

3. 必要な環境変数で **.env ファイルを設定**（`.env.example` を参照）

4. **開発サーバーを起動：**
   ```bash
   npm start
   ```

アプリは `http://localhost:3000` で実行されます。

注意：アプリが機能するには、実行中の PocketBase バックエンドが必要です。[バックエンドセットアップガイド](backend-setup.md)を参照してください。

### テストはどのように実行しますか？

```bash
npm test
```

テストは Vitest と Testing Library を使用して作成されています。テストファイルはソースファイルと同じ場所に `.test.ts` 拡張子で配置されています。

### コードのフォーマットはどうすればいいですか？

**フォーマットの確認：**

```bash
npm run prettier
```

**フォーマットの修正適用：**

```bash
npm run prettier:fix
```

プロジェクトはコードフォーマットに Prettier を使用し、Husky 経由でプリコミットフックが設定されています。

### 技術スタックは何ですか？

- **フロントエンドフレームワーク**：React 19（TypeScript）
- **ビルドツール**：Vite
- **UIライブラリ**：React Bootstrap（Bootstrap 5）
- **マップ**：Leaflet と OpenStreetMap
- **状態管理**：React Context + カスタムフック
- **ルーティング**：Wouter
- **スタイリング**：SCSS
- **バックエンド**：PocketBase（別リポジトリ）
- **監視**：Sentry
- **テスト**：Vitest + Testing Library

アーキテクチャの詳細については、リポジトリの CLAUDE.md を参照してください。

## トラブルシューティング

### アプリが「バックエンドに接続できません」と表示される

**ホストサービスユーザーへ**：このエラーが表示される場合、サービスに問題が発生している可能性があります。ステータスページを確認するかサポートに連絡してください。

**セルフホストユーザーへ**：

**考えられる原因：**

1. PocketBase バックエンドが実行されていないかアクセスできない
2. `VITE_POCKETBASE_URL` 環境変数が間違っている
3. CORS の問題（バックエンドがフロントエンドドメインを許可していない）

**解決策：**

- バックエンドが実行されてアクセス可能であることを確認
- 環境変数の URL を確認
- フロントエンドドメインを許可するように PocketBase の CORS 設定を構成

### 再ビルド後に変更が反映されない

**セルフホストのみ**：

**考えられる原因：**

1. ブラウザキャッシュ
2. CDN キャッシュ（使用している場合）
3. ビルドアーティファクトが正しくデプロイされていない

**解決策：**

- ブラウザキャッシュをクリアするかシークレットモードを使用
- 該当する場合は CDN キャッシュを無効化
- ビルドフォルダがホスティングサービスで完全に置き換えられたことを確認
- ブラウザコンソールのエラーを確認

### ユーザー認証が機能しない

**ホストサービスユーザーへ**：認証の問題がある場合はサポートに連絡してください。

**セルフホストユーザーへ**：

**考えられる原因：**

1. PocketBase バックエンドのユーザー管理が設定されていない
2. ネットワーク接続の問題
3. PocketBase の URL が間違っている
4. CORS 設定の問題

**解決策：**

- PocketBase バックエンドが正しく設定されていることを確認
- ブラウザコンソールのエラーメッセージを確認
- バックエンド API エンドポイントを直接テスト
- PocketBase の CORS 設定を確認

### アプリが遅い、または反応しない

**考えられる原因：**

1. 大きなテリトリーデータセット
2. ネットワークの遅延
3. サーバーリソース不足（セルフホスト）
4. ブラウザのパフォーマンスの問題

**解決策：**

- インターネット接続を確認
- 別のブラウザを試す
- セルフホストの場合：テリトリーデータを最適化（不要なフィールドを削減）
- セルフホストの場合：より近いホスティングリージョンを使用
- セルフホストの場合：必要に応じてサーバーリソースをアップグレード
- ブラウザキャッシュとデータをクリア
- ブラウザコンソールのパフォーマンス警告を確認

## 貢献

### バグを報告するにはどうすればいいですか？

1. [GitHub Issues](https://github.com/rimorin/ministry-mapper-v2/issues) にバグが既に報告されていないか確認
2. 報告されていない場合は、以下を含む新しい Issue を作成：
   - 問題の明確な説明
   - 再現手順
   - 期待される動作と実際の動作
   - ブラウザとデバイス情報
   - スクリーンショットまたはエラーメッセージ
   - 環境設定（機密データなし）

### 機能をリクエストできますか？

はい！GitHub で機能リクエストを作成してください：

- 機能の明確な説明
- ユースケースとメリット
- 動作方法
- 誰が恩恵を受けるか

注意：これはボランティアが維持しているプロジェクトなので、実装は利用可能な時間とリソースによります。

### コードに貢献できますか？

もちろんです！貢献を歓迎します：

**貢献の方法：**

1. リポジトリをフォーク
2. フィーチャーブランチを作成
3. 既存のコードスタイルに従って変更を加える
4. 新機能のテストを作成
5. `npm test` と `npm run prettier` を実行して品質を確認
6. 明確な説明で Pull Request を提出

**貢献ガイドライン：**

- TypeScript と React のベストプラクティスに従う
- コードベースの既存パターンを使用する
- 必要に応じてドキュメントを更新する
- 変更を集中的かつ最小限に保つ
- 提出前に徹底的にテストする

アーキテクチャの詳細とコーディング規則については、リポジトリの CLAUDE.md を参照してください。

### アプリを新しい言語に翻訳できますか？

はい！アプリは翻訳に国際化（i18n）を使用しています：

1. [フロントエンドリポジトリ](https://github.com/rimorin/ministry-mapper-v2)の `src/i18n/` ディレクトリで既存の翻訳を確認
2. 既存の構造に従って新しい言語ファイルを作成
3. 翻訳を追加
4. 言語セレクターコンポーネントを更新
5. Pull Request を提出

### 新しいバージョンにアップグレードするにはどうすればいいですか？

**ホストサービスユーザーへ**：アップデートは自動的に適用されます。何もする必要はありません！

**セルフホストユーザーへ**：

**フロントエンド：**

```bash
git pull origin main
npm install
npm run build
# ホスティングサービスに新しいビルドをデプロイ
```

**バックエンド：**
バックエンドのアップグレード手順については [Ministry Mapper BE リポジトリ](https://github.com/rimorin/ministry-mapper-be)を参照してください。

**重要：**

- まずデータをバックアップする
- 破壊的変更については CHANGELOG.md を読む
- 可能であればステージング環境でテストする
- 必要に応じて環境変数を更新する

## サポート

### ヘルプはどこで得られますか？

**ホストサービスユーザーへ**：[ministry-mapper.com](https://ministry-mapper.com) ウェブサイトのサポートを通じてお問い合わせください。

**すべてのユーザーへ**：

1. **ドキュメント**：このドキュメントサイト（[はじめかた](getting-started.md)、[ユーザーガイド](user-guide.md)など）を確認
2. **GitHub Issues**：既存の Issue を検索するか [ministry-mapper-v2 リポジトリ](https://github.com/rimorin/ministry-mapper-v2/issues)で新しい Issue を作成
3. **GitHub Discussions**：質問とアイデアの共有
4. **コードレビュー**：実装の詳細についてはコードとコメントを読む（開発者向け）

### 商用サポートはありますか？

**ホストサービス**：プロフェッショナルサポートは [ministry-mapper.com](https://ministry-mapper.com) を通じて利用可能です。

**セルフホスト**：セルフホストインスタンスに対する公式の商用サポートはありません。これはボランティアが維持するオープンソースプロジェクトです。ただし：

- GitHub によるコミュニティサポート
- 包括的なドキュメント
- カスタマイズのために独立した開発者を雇うことができます

### アプリはどのくらいの頻度でアップデートされますか？

**ホストサービス**：利用可能になり次第、アップデートが自動的に適用されます。

**セルフホスト**：アップデートはボランティアの可用性に依存します。確認方法：

- バージョン履歴については **CHANGELOG.md**
- リリースノートについては **GitHub Releases**
- 最近の変更については **コミット履歴**

### 自分のニーズに合わせて Ministry Mapper をカスタマイズできますか？

はい！オープンソースなので以下が可能です：

- UI のカスタマイズ（色、レイアウト、ブランディング）
- カスタム機能の追加
- ワークフローの変更
- 他のシステムとの統合

**要件：**

- React 19、TypeScript、PocketBase の知識
- コードベースの理解（リポジトリの CLAUDE.md 参照）
- 変更の適切なテスト
- プロジェクトへの改善のフィードバックを検討

**注意**：カスタマイズにはセルフホストが必要です。ホストサービスは標準バージョンを使用します。
