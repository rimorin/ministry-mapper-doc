# Ministry Mapperユーザーガイド

## はじめに

Ministry Mapperへようこそ。これは、会衆が野外奉仕の区域を効率的に管理するために設計された最新のウェブベースアプリケーションです。このガイドでは、始め方からアプリケーションを最大限に活用する方法まで、知っておくべきすべてのことを説明します。

**Ministry Mapperとは？**

Ministry Mapperは、従来の紙ベースの方法に代わるデジタル区域管理システムです。会衆が以下のことを可能にします：

- 区域をデジタルで整理し、割り当てる
- 野外奉仕の訪問をリアルタイムで追跡
- 複数の伝道者間で活動を調整
- インターネットに接続された任意のデバイスから区域にアクセス

**主な利点：**

- ✓ エコフレンドリー - 紙の無駄を排除
- ✓ クラウド同期によるリアルタイム更新
- ✓ 任意のデバイス（デスクトップ、タブレット、モバイル）で動作
- ✓ 簡単なナビゲーションのための統合インタラクティブマップ
- ✓ 安全なロールベースのアクセス制御

### Ministry Mapperへのログイン

![Google OAuthサインイン](assets/screenshots/google_oauth_sign_in.png)

*図6：より速く安全な認証のためのGoogle OAuthサインインオプション*

アカウントが確認され、会衆アクセスが付与されたら：

**標準ログイン：**
1. 会衆のMinistry Mapper URLに移動
2. メールアドレスとパスワードを入力
3. **「サインイン」**をクリック

**Google OAuthログイン（より速い）：**
1. **「Googleでサインイン」**ボタンをクリック
2. Googleアカウントを選択
3. 自動的にサインイン（パスワード不要）

**ステップ3：OTP確認を完了（有効な場合）**

会衆がワンタイムパスワードセキュリティを有効にしている場合：

1. 資格情報を入力後、OTP確認画面が表示されます
2. メールで確認コードを確認
3. メールからの6桁のコードを入力
4. **「確認」**または**「送信」**をクリック
5. コードは5〜10分で期限切れ

**ステップ4：ダッシュボードにアクセス**

- **伝道者**：ダッシュボードは表示されません - 送られた割り当てリンクを使用します
- **読み取り専用、司会者、管理者**：ロール固有のダッシュボードが表示されます

> **💡 ヒント**：信頼できるデバイスではログイン状態を維持すると便利ですが、共有コンピューターでは必ずログアウトしてください。

### ユーザーロールの理解

Ministry Mapperは4層のアクセス制御システムを使用しています。ロールによって、アクセスできる機能と実行できるアクションが決まります。

#### ロール階層

```
管理者（フルアクセス）
    ↓
司会者（割り当て管理）
    ↓
読み取り専用（閲覧のみ）
    ↓
伝道者（リンクアクセスのみ）
```

---

#### 👤 伝道者

**アクセス方法**：管理者または司会者から送られた割り当てリンク経由

**伝道者ができること：**

- ✓ 共有リンク経由で区域にアクセス
- ✓ インタラクティブマッピングで区域マップを表示
- ✓ 訪問後に住所ステータスを更新
  - 完了、不在、訪問不可、無効としてマーク
- ✓ 住所に訪問メモを追加
- ✓ 「不在」の試行回数を追跡
- ✓ 住所の詳細と順序を表示

**伝道者ができないこと：**

- ✗ ダッシュボードにアクセスできない
- ✗ すべての会衆区域を表示できない
- ✗ 区域を作成・管理できない
- ✗ 割り当てリンクは設定された時間後に期限切れ（デフォルト：24時間、会衆管理者が設定可能）

**最適な対象**：野外奉仕を行う通常の伝道者

---

#### 👓 読み取り専用

**アクセス方法**：読み取り専用権限でのダッシュボードログイン

**読み取り専用ユーザーができること：**

- ✓ 会衆のすべての区域を表示
- ✓ 完全な住所情報を確認
- ✓ 区域の進捗と統計を表示
- ✓ 区域マップにアクセス
- ✓ 会衆メッセージを表示

**読み取り専用ユーザーができないこと：**

- ✗ 区域や住所データを変更できない
- ✗ 区域を作成・削除できない
- ✗ 割り当てを管理できない
- ✗ 会衆設定を変更できない

**最適な対象**：編集機能なしで可視性が必要な監督者

---

#### 🎯 司会者

**アクセス方法**：司会者権限でのダッシュボードログイン

**司会者の能力には以下が含まれます：**

- ✓ **読み取り専用のすべての機能**、さらに：
- ✓ 区域割り当ての作成と管理
- ✓ 伝道者用の割り当てリンクを生成
- ✓ すべての割り当て履歴を表示
- ✓ 会衆メッセージへのアクセスと投稿
- ✓ 会衆オプション（世帯ステータスタイプ）の管理
- ✓ 区域完了状況の表示

**司会者ができないこと：**

- ✗ 区域を作成・削除できない
- ✗ 住所の詳細（住所、ユニット、階）を編集できない
- ✗ ユーザーロールや権限を管理できない
- ✗ コア会衆設定を変更できない

**最適な対象**：野外奉仕コーディネーターとグループ監督者

---

#### 👑 管理者（区域担当者）

**アクセス方法**：完全な管理権限でのダッシュボードログイン

**管理者は完全な制御を持ちます：**

- ✓ **司会者のすべての機能**、さらに：
- ✓ 区域の作成、編集、削除
- ✓ 住所の追加、変更、削除
- ✓ 建物とユニットの管理
- ✓ 会衆設定の構成
  - 最大「不在」試行回数の設定
  - リンク有効期限の設定
  - 世帯タイプオプションの設定
- ✓ ユーザーの招待と管理
- ✓ ユーザーロールと権限の割り当て
- ✓ 区域と住所のリセット
- ✓ ジオロケーション座標の管理

**最適な対象**：区域担当者と会衆の区域システムを管理する人

---

> **💡 注意**：ロールの変更が必要な場合や、現在のロールが不明な場合は、会衆管理者に連絡してください。

伝道者は**ダッシュボードにアクセスできません**。代わりに：

- メールまたはメッセージで割り当てリンクを受け取る
- リンクをクリックして割り当てられた区域にアクセス
- 区域ビュー内で直接作業

- 💬 **メッセージ**：会衆メッセージの投稿と表示

**3. 区域情報パネル**

- 区域コードと説明
- 完了率を示すプログレスバー
- 最終更新のタイムスタンプ
- 総ユニット数対完了ユニット数

**4. 区域表示オプション**

- 🗺️ **マップビュー**：インタラクティブマップ表示
- 📋 **リストビュー**：すべての住所の表形式表示

**5. スピードダイヤルメニュー**（フローティングアクションボタン）

- 🗺️ **マップビューモード**：フルスクリーンマップビューの切り替え
- 📋 **クイックリンク**：割り当てリンクの迅速な作成（司会者と管理者）

### 住所の操作

#### 住所情報の理解

![住所ステータス詳細](assets/screenshots/address_status_details.png)

*図9：すべての住所詳細と現在のステータスを表示する住所カード*

Ministry Mapperの各住所またはユニットには包括的な情報が表示されます：

**基本情報：**

- **住所/ユニット番号**：場所識別子（例：「#05-123」）
- **階**：建物の階（複数階建ての物件の場合）
- **タイプ**：会衆オプションに基づく世帯分類
  - 例：中国語、英語、タミル語、スペイン語
  - 設定されている場合は複数のタイプを選択可能
- **順序**：訪問順序番号

**ステータス情報：**

- **ステータス**：現在の訪問ステータス（以下のステータスタイプを参照）
- **不在カウント**：応答がなかった訪問試行回数
- **訪問不可日**：DNCがマークされた日付（該当する場合）

**アクティビティ追跡：**

- **メモ**：訪問や世帯主に関する重要な情報
- **最終更新**：最新の更新の日時
- **更新者**：最後の変更を行った人のユーザー名

---

#### 住所ステータスタイプ

Ministry Mapperは5つの標準ステータスタイプを使用します：

| ステータス | 色 | 説明 | 使用法 |
| --------------- | ------------- | ---------------------- | ------------------------------------------ |
| **未完了** | 白/デフォルト | まだ訪問していない | すべての住所の初期ステータス |
| **完了** | 緑 | 正常に連絡がとれた | 世帯主が在宅で連絡がとれた |
| **不在** | 黄/オレンジ | 誰も応答しなかった | 最大試行回数まで追跡（設定可能） |
| **訪問不可** | 赤 | 訪問を拒否された | 世帯主がこれ以上の連絡を拒否 |
| **無効** | グレー | アクセス不可 | 住所が存在しないかアクセスできない |

> **💡 ヒント**：「不在」が最大試行回数（管理者が設定）に達すると、住所は自動的に進捗計算で完了としてマークされます。

---

#### 住所ステータスの更新

![住所不在ステータス](assets/screenshots/address_not_home_status.png)

*図10：すべてのフィールドとオプションを表示する更新ステータスモーダル*

**ステップバイステップのプロセス：**

**1. 住所を探す**

- 住所リストをスクロール
- またはマップ上のマーカーをクリック
- 複数階建ての建物では住所は階ごとにグループ化

**2. 更新モーダルを開く**

- 住所/ユニットカードをクリック
- 現在の情報を含む更新モーダルが開く

**3. ステータスを更新**（必須）

適切なステータスを選択：

**📗 完了** - 正常に連絡がとれたとき

- 誰かが応答したときに選択
- 会話が行われたか、出版物が配置された
- 「不在」カウントがリセットされる

**🏠 不在** - 誰も応答しなかったとき

- 自動的に「不在」カウントが増加
- システムが試行回数を追跡
- 最大試行回数に達すると、完了として扱われる

**🚫 訪問不可** - 訪問しないよう要求されたとき

![住所訪問不可ステータス](assets/screenshots/address_do_not_call_status.png)

*図11：日付選択を含む訪問不可ステータス更新*

- このステータスを選択
- オプションでDNC日を設定（デフォルトは今日）
- 必要に応じて理由を説明するメモを追加
- **重要**：常に世帯主の希望を尊重

**❌ 無効** - 住所にアクセスできないとき

- 住所が存在しない
- 建設中
- 恒久的に閉鎖
- アクセスできない

**4. 世帯タイプを更新**（利用可能な場合）

会衆が世帯タイプを設定している場合：

- 単一または複数のタイプを選択
- 例：言語の好み、特別な状況
- 必要に応じて既存のタイプをクリア

**5. メモを追加または更新**（オプションですが推奨）

**ベストプラクティス：**

- ✓ 個人ではなく、物件の詳細に焦点を当てる
- ✓ アクセス手順とタイミングを記録
- ✓ 簡潔で敬意を持って
- ✗ 世帯主に関する個人情報を含めない
- ✗ センシティブな情報を含めない

**良い例：**

- 「ゲート付き物件 - 最初に警備室に電話」
- 「最適な時間：週末の午後2時以降」
- 「ドライブウェイ経由でサイドエントランスにアクセス可能」

**悪い例：**

- 住人に関する個人的な詳細
- 医療または財務情報
- 個人の名前や説明

**6. 不在カウントを調整**（必要な場合）

システムは自動的に不在試行を追跡しますが、必要に応じて手動で調整できます。

**7. 訪問不可日を設定**（DNCステータスのみ）

訪問不可としてマークするとき：

- システムはデフォルトで今日の日付
- 特定のDNCリクエストに応じて調整
- 日付は潜在的に再訪問するタイミングを追跡するのに役立つ

**8. ジオロケーションを更新**（管理者のみ）

一戸建て区域の場合：

- 「ジオロケーションを更新」ボタンをクリック
- マップを使用して正確な位置を設定
- ナビゲーションとマッピングの精度を向上

**9. 変更を保存**

- すべての更新を確認
- **「保存」**ボタンをクリック
- 変更はすべてのユーザーに即座に同期
- 成功メッセージで更新を確認

**10. 物件を削除**（管理者のみ - 一戸建て）

削除が必要な個人物件の場合：

- 下部の「物件を削除」ボタンをクリック
- 削除を確認
- **警告**：このアクションは取り消せません

### マップ機能の使用

![マップの道順](assets/screenshots/map_directions.png)

*図12：マーカーとナビゲーションコントロールを表示するインタラクティブマップビュー*

Ministry Mapperは直感的な区域ナビゲーションのためのインタラクティブマッピングを統合しています。

#### マップ機能

**インタラクティブマーカー：**

- 各住所はマップ上にマーク
- 🔴 赤いマーカーは目的地を示す
- 🔵 青い点滅マーカーは現在地を示す
- 任意のマーカーをクリックしてその住所を表示/編集

### 区域割り当て（司会者と管理者）

Ministry Mapperはリンクベースの割り当てシステムを使用しています。司会者と管理者は、伝道者が区域にアクセスするために使用する共有可能なリンクを作成します。

#### リンクベースの割り当ての理由

- ✓ **簡単な配布**：メール、メッセージ、テキストでリンクを送信
- ✓ **自動期限切れ**：リンクは設定された時間後に期限切れ（デフォルト：24時間）
- ✓ **アカウント不要**：伝道者はリンクを通じて直接作業
- ✓ **セキュリティ**：期限切れのリンクはアクセス不可
- ✓ **追跡**：誰がいつ何にアクセスしたかを確認

### リアルタイムデータ同期

#### 仕組み

**自動同期：**

- 変更はすべての接続されたデバイス間で即座に同期

- ✓ 住所ステータスの変更
- ✓ 新しいメモと更新
- ✓ 区域の進捗

**パフォーマンス：**

- 区域/ページが開いているときのみ更新

- 最小限の帯域幅使用
- モバイルデータ用に最適化

#### ユーザーロールの変更

![ユーザー管理詳細](assets/screenshots/manage_users_details.png)

*図18：ユーザー詳細とロール管理インターフェース*

1. ユーザーリストでユーザーを見つける
2. ユーザーまたは**「編集」**ボタンをクリック
3. ドロップダウンから新しいロールを選択：
   - **伝道者**：リンク経由の基本的な区域アクセス
   - **読み取り専用**：閲覧のみのダッシュボードアクセス
   - **司会者**：割り当て作成とメッセージ管理が可能
   - **管理者**：フルコントロール
4. **「保存」**または**「更新」**をクリック
5. 変更は即座に有効

**一時的な削除：**

1. ユーザーのロールを**「アクセスなし」**または**「アクセスを削除」**に変更
2. ユーザーはすべての権限を失う
3. アカウントは残るが会衆データにアクセス不可

**恒久的な削除：**

1. ユーザーの**「削除」**ボタンをクリック
2. 削除を確認
3. ユーザーが会衆から完全に削除
4. 必要に応じてユーザーを再招待可能

- 通常にログイン可能

**未確認ユーザー（✗）：**

- メールがまだ確認されていない
- 制限付きまたはアクセスなし
- メールを確認して確認リンクをクリックする必要あり

**確認を再送信するには：**

- 一部のシステムでは確認メールの再送信が可能
- またはユーザーに「パスワードを忘れた」機能を使用するよう依頼

### 会衆設定（管理者のみ）

![会衆設定](assets/screenshots/congregation_settings.png)

*図19：すべての設定オプションを含む会衆設定ページ*

会衆でMinistry Mapperがどのように機能するかを設定します。

#### 設定へのアクセス

1. **「設定」**ボタンまたは⚙️アイコンをクリック
2. 会衆設定パネルを表示

#### 主要設定

**1. 最大「不在」試行回数**

- デフォルト：1
- 範囲：1〜4回
- 達成時、住所は進捗計算で完了とみなされる
- 区域が完了として表示されるタイミングに影響

**例：** 3に設定した場合：

- 1回目の「不在」：カウント = 1
- 2回目の「不在」：カウント = 2
- 3回目の「不在」：カウント = 3、完了としてマーク

**2. 割り当てリンクの有効期限（時間）**

- デフォルト：24時間
- 範囲：1〜168時間（1週間）
- 割り当てリンクがアクティブな期間
- 新しく作成されたリンクに適用

**3. 会衆の原点/場所**

- 会衆の場所を設定
- マップのセンタリングと道順に使用
- 都市名または座標を指定可能
- ルート計画に役立つ

**4. OTP（ワンタイムパスワード）**

- ログイン用のメールOTPを有効/無効
- 追加のセキュリティレイヤーを追加
- ログイン時にユーザーがメールでコードを受信
- センシティブな会衆データに推奨

- 例：言語グループ（中国語、英語、タミル語）

**オプションの管理：**

   - オプションの削除ボタンをクリック
   - 削除を確認
   - **警告**：このタイプを使用しているすべての住所に影響

**オプションフラグ：**

**カウント可能か：**
- このタイプの住所が区域進捗にカウントされるかを制御
- **チェック済み**：進捗計算に含まれる（例：中国語、英語、タミル語）
- **未チェック**：進捗から除外（例：ビジネス、建設中）
- 例：総住所100、「ビジネス」（カウント不可）としてマーク10 = 進捗は90住所のみに基づく

**デフォルトか：**
- 新しい住所を作成するときにこのタイプを自動選択
- デフォルトは1つのオプションのみにする
- 最も一般的な世帯タイプに使用

**順序：**
- ドロップダウンメニューでの表示順序を制御
- 小さい数字が先に表示（1、2、3...）
- 設定で**ドラッグ＆ドロップで管理** - 単にオプションをドラッグして並べ替え
- 住所世帯ドロップダウンでの順序に影響
- ヒント：最も一般的なものから最も珍しいものへ順序付け

**複数選択の設定：**

- 世帯が複数のタイプを持てる場合に有効化
- 例：世帯が中国語と英語の両方を話す
- 無効時、世帯ごとに1つのタイプのみ

**利用可能なマップ設定機能：**

   - マップマーカーを別の住所に移動
   - 区域ユニットの地理的位置を更新
   - 住所が変更されたか、初期位置が間違っていた場合に便利
   - マップ上で新しい位置を選択するだけ

**2. 区域の変更**
   - 住所またはユニットを別の区域に移動
   - 区域境界の再編成に役立つ
   - 区域サイズのバランスを取るのに便利

   - 住所の訪問順序番号を変更
   - 野外奉仕効率のためにルートを最適化
   - 小さい数字が先に訪問
   - 論理的な訪問パターンを作成するのに役立つ

##### マップ順序の更新（ドラッグ＆ドロップ）

![マップ順序の更新](assets/screenshots/update_map_sequence.png)

*区域内のすべてのマップ順序を並べ替えるためのドラッグ＆ドロップインターフェース*

**アクセス：** 住所メニュー → 「順序の変更」

**仕組み：**
1. 各カードには現在の順序番号付きの住所が表示
2. ドラッグ＆ドロップして並べ替え - 番号は自動的に更新
3. **「保存」**をクリックして適用、または**「キャンセル」**で破棄

**ベストプラクティス：**
- 近くの住所をグループ化してバックトラッキングを最小化
- 複数階建ての建物では階をまとめる
- 端から端への論理的なフローを作成
- 順序付け後にマップビューを確認

**4. 名前の変更**
   - 住所/ユニットの名前または識別子を変更
   - 建物名やユニット番号を更新
   - データを実世界の変更に合わせて最新に保つ
   - 初期入力エラーの修正に便利

**5. ユニット番号を追加**
   - 既存の住所に新しいユニット番号を追加
   - 追加ユニットで複数階建ての建物を拡張
   - 建物に新しいアパートが追加されたときに便利
   - 建物構造の組織を維持

**6. 上の階を追加**
   - 追加の階で建物を上方に拡張
   - 拡張された建物や最初に過小評価された建物用
   - 建物パターンに基づいて新しい階のユニットを自動作成
   - 区域データを建設変更に合わせて最新に保つ

**7. 下の階を追加**
   - 現在の最低階より下に階を追加
   - 地下レベルや新しくアクセス可能になった下の階に便利
   - 負の階番号を追加可能（例：B1、B2）
   - 一貫した階番号システムを維持

**8. ステータスをリセット**
   - 住所のステータスを「未完了」にクリア
   - 「完了」と「不在」ステータスのみを削除
   - 「訪問不可」や「無効」ステータスは削除**しない**
   - メモやその他の住所情報を保持
   - 以前に完了した住所での作業を再開するときに便利
   - 区域内の他の住所には影響しない

**9. 削除**
   - 区域から住所、ユニット、または階を恒久的に削除
   - 取り消し不可 - 注意して使用
   - 重複エントリや存在しない住所の削除に役立つ
   - 削除前に慎重に確認

**一般的な使用例：**
- エラーの修正：名前の変更と場所の変更
- 区域のリバランス：区域の変更
- 建物の更新：上/下の階を追加
- ルート最適化：順序の変更
- データクリーンアップ：重複の削除
- 季節の更新：ステータスのリセット

> **⚠️ 警告**：変更はすべてのユーザーに即座に影響します。削除は恒久的で取り消しできません。

4. ✓ 効率的なルーティングのために順序番号に従う
5. ✓ 不要なときは画面を暗くしてバッテリーを節約

**データ使用のヒント：**

- Ministry Mapperは最小限のデータを使用
- マップタイルはより多くのデータを使用する可能性あり
- ほとんどの更新は各1KB未満
- モバイルデータ使用に適切

### オフライン機能と制限

**インターネット接続が必要：**

Ministry Mapperは以下にアクティブなインターネットが必要：

- ✗ 区域データの読み込み
- ✗ ステータス更新の保存
- ✗ リアルタイム同期
- ✗ マップの表示
- ✗ ユーザー認証

**限定的なオフライン機能：**

- 静的アセットがキャッシュ（アプリシェル）
- 以前に読み込まれた区域が表示される可能性あり
- **オフラインで更新はできない**
- **更新は後の同期用にキューされない**

**接続損失の処理：**

- システムが接続損失を検出
- 接続ステータス警告を表示
- 利用可能になると自動再接続
- 接続復旧後に作業を再開

**推奨事項：**

- ✓ 開始前に信頼できる接続を確保
- ✓ 区域場所で接続をテスト
- ✓ インターネットがないエリアのバックアッププランを用意
- ✓ 必要に応じてポータブルWiFiホットスポットを検討
- ✓ 接続がアクティブな間に住所を更新

### 伝道者向け

| フェーズ | ベストプラクティス |
|-------|---------------|
| **開始前** | マップで区域を確認 • メモ/指示を確認 • 順序を使用してルートを計画 • 「訪問不可」住所をメモ • デバイスが充電されていることを確認 |

- 区域サイズに基づいて適切なリンク有効期限を設定
- 追跡用に伝道者名を含める
- 期限切れの割り当てを定期的にクリーンアップ
- 区域完了を監視し、期限超過の割り当てをフォローアップ
- 明確なメッセージと指示を投稿

### 管理者向け

| エリア | ベストプラクティス |
|------|---------------|
| **区域設定** | 一貫した命名 • 短く意味のあるコード • 明確な説明 • マップ位置を確認 • 論理的な順序 |

### 一般原則

- **事前に計画**：出発前に確認
- **体系的に作業**：一度に1つのセクションを完了
- **迅速に更新**：情報を記録するのを待たない
- **明確にコミュニケーション**：理解しやすく、敬意あるメモを書く
- **徹底的に**：すべての住所を粘り強くカバー
- **柔軟に**：区域のニーズに適応

## プライバシーとセキュリティ

### 情報の保護

Ministry Mapperはセンシティブな住所と個人情報を扱います。以下のガイドラインを遵守してください：

**アカウントセキュリティ：**

- ✓ **ログイン資格情報を誰とも共有しない**
- ✓ **強力でユニークなパスワードを使用**（最小6文字、数字と大文字を含む）
- ✓ **共有デバイスでは常にログアウト**
- ✓ **利用可能な場合はOTPを有効化**して追加のセキュリティ
- ✓ **不審なアクティビティを直ちに報告**

**データプライバシー：**

- ✓ メモには**必要な情報のみを記録**
- ✓ すべての説明で**敬意を持ち事実に基づく**
- ✓ **センシティブな個人データなし**（医療、財務など）
- ✓ プライバシーに関する**世帯主のリクエストに従う**
- ✓ **プライバシー法を遵守**（GDPR、CCPA、地域の規制）

**法的コンプライアンス：**

- ⚠️ **GDPR（ヨーロッパ）**：個人データ保護要件
- ⚠️ **CCPA（カリフォルニア）**：消費者プライバシー権
- ⚠️ **LGPD（ブラジル）**：データ保護規制
- ⚠️ **地域の法律**：地域の要件を確認

### 保存される情報

**ユーザーデータ：**

- アカウント詳細（名前、メール、確認ステータス）

**システムデータ：**

- ログインセッション
