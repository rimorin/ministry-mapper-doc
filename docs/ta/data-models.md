# தரவு மாதிரிகள் & திட்டம்

## மேலோட்டம்

Ministry Mapper பல-குத்தகை சபை நிர்வாகத்திற்காக வடிவமைக்கப்பட்ட படிநிலை தரவு மாதிரியுடன் PocketBase மூலம் SQLite பயன்படுத்துகிறது. இந்த ஆவணம் அனைத்து தரவு மாதிரிகள், உறவுகள் மற்றும் வணிக விதிகளின் விரிவான ஆவணப்படுத்தலை வழங்குகிறது.

## நிறுவன உறவு வரைபடம்

```
Congregation (Root entity - multi-tenant isolation)
    │
    ├─── Territory (1:M)
    │      │
    │      └─── Map (1:M)
    │             │
    │             ├─── Address (1:M)
    │             │      ├─── addresses_log (1:M)
    │             │      └─── address_options (1:M)
    │             ├─── Assignment (1:M)
    │             └─── Message (1:M)
    │
    ├─── Option (1:M)
    │      └─── address_options (M:M bridge with Address)
    │
    ├─── User (1:M via Role)
    │      │
    │      ├─── Role (1:M)
    │      └─── Assignment (1:M)
    │
    ├─── Message (1:M)
    │
    └─── aggregates (1:M from Territory and Map)
```

## முக்கிய தொகுப்புகள்

### 1. Congregations

**நோக்கம்:** பல-குத்தகை தனிமைப்படுத்தல் மற்றும் நிறுவன அமைப்புகளுக்கான மையமான நிறுவனம்

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `code` | text | ஆம் | நிறுவன குறியீடு |
| `name` | text | ஆம் | நிறுவன பெயர் |
| `expiry_hours` | number | ஆம் | பணிப்பு இணைப்பு கால அளவு (மணிநேரங்கள்) |
| `max_tries` | number | ஆம் | முடிந்ததாக குறிக்கும் முன் அதிகபட்ச "வீட்டில் இல்லை" முயற்சிகள் |
| `origin` | select | ஆம் | நாடு/பிராந்தியம் (sg, my) |
| `timezone` | select | ஆம் | தேதி கணக்கீடுகளுக்கான நேர மண்டலம் |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**அணுகல் விதிகள்:**
- **பட்டியல்:** சபையில் பாத்திரமுடைய பயனர்
- **பார்வை:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு-அடிப்படையிலான அணுகல்
- **புதுப்பிப்பு:** நிர்வாகி மட்டும்
- **நீக்கம்:** நிர்வாகி மட்டும்

**வணிக விதிகள்:**
- அனைத்து வரைபடங்களுக்கும் இயல்புநிலை பணிப்பு கால அளவை கட்டுப்படுத்துகிறது
- `max_tries` "வீட்டில் இல்லை" முகவரிகள் எப்போது முடிந்ததாக கணக்கிடப்படும் என்பதை தீர்மானிக்கிறது
- நேர மண்டலம் தேதி-அடிப்படையிலான கணக்கீடுகள் மற்றும் அறிக்கைகளை பாதிக்கிறது

**எடுத்துக்காட்டு:**
```json
{
  "id": "abc123",
  "code": "CONG001",
  "name": "North Congregation",
  "expiry_hours": 24,
  "max_tries": 3,
  "origin": "sg",
  "timezone": "Asia/Singapore"
}
```

### 2. Territories

**நோக்கம்:** வயல் சேவையை ஒழுங்கமைப்பதற்காக சபைக்குள் புவியியல் பிரிவுகள்

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `congregation` | relation | ஆம் | Congregations க்கு FK (cascade delete) |
| `code` | text | ஆம் | பகுதி அடையாளங்காட்டி (தானியங்கி: "0") |
| `description` | text | இல்லை | மனிதர் படிக்கக்கூடிய விளக்கம் |
| `progress` | number | தானியங்கி | நிறைவு சதவீதம் (0-100) |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**குறியீடுகள்:**
- `(congregation, code)` - சபை வழியாக விரைவான தேடல்
- `(congregation)` - சபை வடிகட்டி

**உறவுகள்:**
- Maps உடன் 1:M
- Addresses உடன் 1:M (செயல்திறனுக்காக de-normalized)

**அணுகல் விதிகள்:**
- **பட்டியல்:** சபையில் பாத்திரமுடைய பயனர்
- **பார்வை:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **உருவாக்கு/புதுப்பிக்கு/நீக்கு:** Conductor அல்லது Administrator

**முன்னேற்ற கணக்கீடு:**
பகுதிக்குள் உள்ள அனைத்து வரைபடங்களின் ஒருங்கிணைப்பில் இருந்து தானியங்கியாக கணக்கிடப்படுகிறது

**எடுத்துக்காட்டு:**
```json
{
  "id": "terr_abc",
  "congregation": "cong_123",
  "code": "1A",
  "description": "Downtown District",
  "progress": 65.5
}
```

### 3. Maps

**நோக்கம்:** முகவரிகளை கொண்ட குறிப்பிட்ட இடங்கள் அல்லது கட்டிடங்களை பிரதிபலிக்கிறது

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `territory` | relation | ஆம் | Territories க்கு FK (cascade delete) |
| `congregation` | relation | ஆம் | Congregations க்கு FK (cascade delete) |
| `sequence` | number | தானியங்கி | பகுதிக்குள் வரிசை (தானியங்கி-அதிகரிப்பு) |
| `code` | text | ஆம் | வரைபட அடையாளங்காட்டி/எண் (தானியங்கி: "0") |
| `description` | text | இல்லை | இருப்பிட விளக்கம் |
| `type` | select | ஆம் | `single` அல்லது `multi` (தள வகை) |
| `coordinates` | JSON | இல்லை | புவியியல் இருப்பிடம் `{lat, lng}` |
| `aggregates` | JSON | தானியங்கி | தேக்கப்பட்ட புள்ளிவிவரங்கள் (அதிகபட்சம் 2MB) |
| `progress` | number | தானியங்கி | நிறைவு சதவீதம் |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**குறியீடுகள்:**
- `(territory, code)` - முதன்மை தேடல்
- `(territory, sequence)` - வரிசைப்படுத்தல்
- `(territory)` - பகுதி வடிகட்டி

**Cascade நீக்கங்கள்:**
- Addresses
- Assignments
- Messages

**Aggregates கட்டமைப்பு:**
```json
{
  "done": 45,
  "not_done": 30,
  "dnc": 5,
  "invalid": 2,
  "not_home_max_tries": 8
}
```

**வரைபட வகைகள்:**
- **single**: தனியார் முகவரிகள் (வீடுகள்)
- **multi**: பொது முகவரிகள் (பல-தள கட்டிடங்கள்)

**அணுகல் விதிகள்:**
- **பட்டியல்:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **பார்வை:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **உருவாக்கு/புதுப்பிக்கு:** Conductor அல்லது Administrator
- **நீக்கு:** நிர்வாகி மட்டும்

**எடுத்துக்காட்டு:**
```json
{
  "id": "map_xyz",
  "territory": "terr_abc",
  "congregation": "cong_123",
  "sequence": 5,
  "code": "12",
  "description": "Main Street Apartments",
  "type": "multi",
  "coordinates": {"lat": 1.3521, "lng": 103.8198},
  "aggregates": {
    "done": 20,
    "not_done": 10,
    "dnc": 2,
    "invalid": 1,
    "not_home_max_tries": 3
  },
  "progress": 66.7
}
```

### 4. Addresses

**நோக்கம்:** வரைபடங்களுக்குள் தனிப்பட்ட அலகுகள் அல்லது வீடுகள்

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `map` | relation | ஆம் | Maps க்கு FK (cascade delete) |
| `congregation` | relation | ஆம் | Congregations க்கு FK (de-normalized) |
| `territory` | relation | ஆம் | Territories க்கு FK (de-normalized) |
| `floor` | number | ஆம் | தள எண் (0-அடிப்படையிலான) |
| `code` | text | ஆம் | அலகு/கதவு குறியீடு (எ.கா., "101", "A") |
| `sequence` | number | ஆம் | வரைபடத்திற்குள் வரிசை |
| `status` | select | ஆம் | முகவரி வருகை நிலை |
| `not_home_tries` | number | தானியங்கி | "வீட்டில் இல்லை" முயற்சிகளுக்கான எண்ணிக்கை |
| `type` | relation | இல்லை | Options க்கு M:M (JSON array) |
| `notes` | text | இல்லை | வயல் தொழிலாளர் குறிப்புகள் |
| `last_notes_updated` | date | தானியங்கி | குறிப்புகள் கடைசியாக மாறிய நேரம் (hook வழியாக) |
| `last_notes_updated_by` | text | தானியங்கி | கடைசி புதுப்பித்தவரின் பயனர்பெயர் (hook வழியாக) |
| `dnc_time` | date | இல்லை | "அழைக்க வேண்டாம்" என குறிக்கப்பட்ட நேரம் |
| `coordinates` | JSON | இல்லை | நுட்பமான GPS `{lat, lng}` |
| `updated_by` | text | இல்லை | புதுப்பிப்புகளை யார் செய்தார் என கண்காணிக்க |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**நிலை மதிப்புகள்:**
- `not_done` - இன்னும் பார்க்கப்படவில்லை (ஆரம்ப நிலை)
- `done` - வெற்றிகரமாக முடிந்தது
- `not_home` - வீட்டில் யாரும் இல்லை (மீண்டும் முயற்சிக்கலாம்)
- `do_not_call` - வேலையில் இருந்து விலக்கப்பட்டது
- `invalid` - முகவரி இல்லை

**குறியீடுகள் (மொத்தம் 7):**
- `(map)` - வரைபட வடிகட்டி
- `(map, status)` - நிலை விநியோக வினவல்கள்
- `(map, code)` - குறியீடு தேடல்
- `(map, floor)` - தள ஒழுங்கமைப்பு
- `(type, congregation)` - விருப்பம் விநியோகம்
- `(congregation, last_notes_updated_by)` - பயனரால் சமீபத்திய குறிப்புகள்
- `(territory, status)` - பகுதி-அளவிலான நிலை

**தானியங்கி Hooks:**
- புதுப்பிப்புகள் குறிப்புகள் மாறினால் `last_notes_updated` மற்றும் `last_notes_updated_by` புலங்களை புதுப்பிக்கின்றன
- வரைபடத்தில் aggregate மறு கணக்கீட்டை தூண்டுகின்றன
- வரைபட முன்னேற்றத்தை புதுப்பிக்கின்றன
- பகுதி முன்னேற்றத்தை புதுப்பிக்கின்றன

**அணுகல் விதிகள்:**
- **பட்டியல்:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **பார்வை:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **உருவாக்கு:** Conductor அல்லது Administrator
- **புதுப்பிக்கு:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல் (வெளியீட்டாளர்கள் புதுப்பிக்கலாம்)
- **நீக்கு:** நிர்வாகி மட்டும்

**எடுத்துக்காட்டு:**
```json
{
  "id": "addr_123",
  "map": "map_xyz",
  "congregation": "cong_123",
  "territory": "terr_abc",
  "floor": 2,
  "code": "201",
  "sequence": 5,
  "status": "not_home",
  "not_home_tries": 2,
  "type": ["opt_residential"],
  "notes": "Try after 6pm",
  "last_notes_updated": "2025-12-19T14:30:00Z",
  "last_notes_updated_by": "john.doe",
  "coordinates": {"lat": 1.3521, "lng": 103.8198}
}
```

### 5. Users

**நோக்கம்:** அங்கீகாரத்துடன் கணினி பயனர் கணக்குகள்

**வகை:** Auth தொகுப்பு (PocketBase user-auth-type)

**கணினி புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `email` | email | ஆம் | உள்நுழைவு அடையாளம் (தனித்துவமானது) |
| `emailVisibility` | bool | ஆம் | மின்னஞ்சல் பொதுவா என்பதை குறிக்கிறது |
| `verified` | bool | தானியங்கி | மின்னஞ்சல் சரிபார்ப்பு நிலை |
| `password` | password | ஆம் | Bcrypt hashed (cost: 10, குறைந்தது 6 எழுத்துகள்) |
| `tokenKey` | text | மறைக்கப்பட்டது | டோக்கன் உருவாக்கும் விசை (30-60 எழுத்துகள்) |

**தனிப்பயன் புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `name` | text | ஆம் | காட்சி பெயர் (2-50 எழுத்துகள்) |
| `disabled` | bool | இல்லை | கணக்கு முடக்கப்பட்ட கொடி |
| `last_login` | date | தானியங்கி | கடைசி அங்கீகார நேரமுத்திரை (hook வழியாக) |

**பெயர் சரிபார்ப்பு முறை:**
`^[A-Za-z][\w\s\.\-']*$` (எழுத்தில் தொடங்குகிறது, வார்த்தை எழுத்துகள், இடைவெளிகள், புள்ளிகள், கோடுகள், அபொஸ்ட்ராஃப்களை அனுமதிக்கிறது)

**தானியங்கி உருவாக்கம் முறை:**
`user[0-9]{5}[A-Za-z]` (எ.கா., user12345A)

**குறியீடுகள்:**
- `email` (தனித்துவமானது) - விரைவான auth தேடல்
- `tokenKey` (தனித்துவமானது) - டோக்கன் சரிபார்ப்பு

**Auth உள்ளமைவு:**
- **கடவுச்சொல் Auth:** இயக்கப்பட்டது (குறைந்தது 6 எழுத்துகள்)
- **OAuth2:** Google PKCE flow
  - OAuth வழங்குநரிடமிருந்து `name` ஐ மாப்பிங் செய்கிறது
  - முதல் உள்நுழைவில் பயனரை தானியங்கியாக உருவாக்குகிறது
- **MFA:** இயக்கப்பட்டது (கால அளவு: 1800s) - OAuth2 க்கு முடக்கப்பட்டது
- **OTP:** இயக்கப்பட்டது (கால அளவு: 180s, நீளம்: 4 இலக்கங்கள்)
- **மின்னஞ்சல் சரிபார்ப்பு:** கால அளவு 604800s (7 நாட்கள்)
- **கடவுச்சொல் மீட்டமைப்பு:** கால அளவு 1800s (30 நிமிடம்)
- **Auth எச்சரிக்கை:** இயக்கப்பட்டது (புதிய இடத்திலிருந்து உள்நுழைவில் அறிவிக்கிறது)

**அணுகல் விதிகள்:**
- **பட்டியல்:** நிர்வாகிகள் மட்டும்
- **பார்வை:** நிர்வாகிகள் அல்லது தாமே
- **உருவாக்கு:** பொது (OAuth அல்லது வெளிப்படையான பதிவு)
- **புதுப்பிக்கு:** தாமே மட்டும்
- **நீக்கு:** நிர்வாகிகள் மட்டும்

**கோப்பு டோக்கன்:** 120s கால அளவு

**உறவுகள்:**
- Roles உடன் 1:M
- Assignments உடன் 1:M

**எடுத்துக்காட்டு:**
```json
{
  "id": "user_abc",
  "email": "john.doe@example.com",
  "name": "John Doe",
  "verified": true,
  "disabled": false,
  "last_login": "2025-12-19T14:30:00Z"
}
```

### 6. Roles

**நோக்கம்:** பயனர்களை சபைகளுடன் இணைக்கும் அனுமதி ஒதுக்கீடுகள்

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `user` | relation | ஆம் | Users க்கு FK (cascade delete) |
| `congregation` | relation | ஆம் | Congregations க்கு FK (cascade delete) |
| `role` | select | ஆம் | அனுமதி நிலை |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**பாத்திர மதிப்புகள்:**
- `read_only` - பார்க்க மட்டுமே அணுகல்
- `conductor` - பணிப்புகளை நிர்வகிக்கவும் வரைபடங்களை நிர்வகிக்கவும் முடியும்
- `administrator` - முழு அணுகல்

**குறியீடுகள்:**
- `(user, congregation)` - கலவை விசை (சபைக்கு ஒரு பயனருக்கு ஒரு பாத்திரம்)
- `(user)` - பயனரின் பாத்திரங்கள்
- `(congregation, role)` - பாத்திர விநியோகம்
- `(user, congregation, role)` - முழு கலவை தேடல்

**அணுகல் விதிகள்:**
- **உருவாக்கு/புதுப்பிக்கு/நீக்கு:** நிர்வாகிகள் மட்டும்
- **பட்டியல்:** பயனரின் சொந்த பாத்திரங்கள்
- **பார்வை:** பயனரின் சொந்த பாத்திரங்கள்

**உறவுகள்:**
- Users மற்றும் Congregations இடையே M:N

**வணிக விதிகள்:**
- சபைக்கு ஒரு பயனருக்கு ஒரு பாத்திரம்
- பாத்திரம் அனைத்து செயல்களுக்கான அனுமதிகளை தீர்மானிக்கிறது

**எடுத்துக்காட்டு:**
```json
{
  "id": "role_xyz",
  "user": "user_abc",
  "congregation": "cong_123",
  "role": "conductor"
}
```

### 7. Assignments

**நோக்கம்:** நேர-வரையறுக்கப்பட்ட வரைபட அணுகலுக்கான வெளியீட்டாளர் அணுகல் டோக்கன்கள்

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(20) | ஆம் | **அநாமதேய அணுகல் டோக்கனாக செயல்படுகிறது** |
| `congregation` | relation | ஆம் | Congregations க்கு FK (cascade delete) |
| `map` | relation | ஆம் | Maps க்கு FK (cascade delete) |
| `user` | relation | இல்லை | வெளியீட்டாளர் பயனர் (விருப்பமானது) |
| `type` | select | ஆம் | பணிப்பு வகைப்படுத்தல் |
| `expiry_date` | date | ஆம் | தானியங்கி காலாவதி நேரமுத்திரை |
| `publisher` | text | இல்லை | வெளியீட்டாளர் பெயர்/அடையாளம் |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**பணிப்பு வகைகள்:**
- `normal` - வழக்கமான பகுதி வேலை
- `personal` - தனிப்பட்ட ஊழியப் பகுதி

**குறியீடுகள்:**
- `(map)` - வரைபடத்திற்கு செயலில் உள்ள பணிப்புகள்
- `(expiry_date)` - பின்னணி சுத்தம் செய்யும் வேலைக்காக
- `(user, created)` - பயனர் பணிப்பு வரலாறு
- `(map, expiry_date)` - வரைபடத்திற்கு காலாவதி சோதனை

**அணுகல் விதிகள்:**
- **உருவாக்கு/நீக்கு:** Conductor அல்லது Administrator
- **பார்வை:** பாத்திரமுடைய பயனர் அல்லது இந்த ID உடன் பொருந்தும் link_id + காலாவதியாகவில்லை
- **பட்டியல்:** பாத்திரமுடைய பயனர்

**பணிப்பாட்டு வழிமுறை:**
1. Conductor `/territory/link` endpoint வழியாக உருவாக்குகிறார்
2. வெளியீட்டாளர் 20-எழுத்து `link_id` பெறுகிறார் (இந்த பணிப்பின் ID)
3. வெளியீட்டாளர் HTTP header `@request.headers.link_id` வழியாக `link_id` அனுப்புகிறார்
4. PocketBase சரிபார்க்கிறது: link_id, assignment.id உடன் பொருந்துகிறதா மற்றும் expiry_date > இப்போது
5. பின்னணி வேலை ஒவ்வொரு 5 நிமிடத்திற்கும் காலாவதியான பணிப்புகளை நீக்குகிறது

**எடுத்துக்காட்டு:**
```json
{
  "id": "asgn_1234567890abcdef",
  "congregation": "cong_123",
  "map": "map_xyz",
  "user": "user_abc",
  "type": "normal",
  "expiry_date": "2025-12-20T14:30:00Z",
  "publisher": "John Doe"
}
```

### 8. Options

**நோக்கம்:** முகவரி வகை வகைப்படுத்தல்கள் (சபைக்கு ஏற்ப தனிப்பயனாக்கக்கூடியது)

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `congregation` | relation | ஆம் | Congregations க்கு FK (cascade delete) |
| `code` | text | ஆம் | விருப்பம் அடையாளங்காட்டி (எ.கா., "RES", "BUS") |
| `description` | text | ஆம் | மனிதர் படிக்கக்கூடிய தலைப்பு |
| `is_countable` | bool | ஆம் | முன்னேற்ற கணக்கீடுகளில் சேர்க்கவும் |
| `is_default` | bool | ஆம் | புதிய முகவரிகளுக்கு இயல்புநிலை விருப்பம் |
| `sequence` | number | ஆம் | காட்சி/வரிசைப்படுத்தும் வரிசை |
| `created` | date | தானியங்கி | உருவாக்கப்பட்ட நேரமுத்திரை |
| `updated` | date | தானியங்கி | கடைசி புதுப்பிப்பு நேரமுத்திரை |

**குறியீடுகள்:**
- `(congregation, sequence)` - சபைக்குள் வரிசைப்படுத்தல்
- `(congregation, is_default)` - இயல்புநிலை விருப்பத்தை கண்டுபிடி

**கட்டுப்பாடுகள்:**
- சபைக்கு சரியாக ஒரு `is_default = true` (பயன்பாட்டு அடுக்கில் அமலாக்கப்படுகிறது)
- குறியீடுகள் சபைக்கு தனித்துவமானவை
- வரிசைகள் சபைக்கு தனித்துவமானவை

**முன்னேற்றத்தில் தாக்கம்:**
- எண்ணக்கூடிய விருப்பங்கள் (`is_countable=true`) முன்னேற்ற % கணக்கீட்டில் சேர்க்கப்படுகின்றன
- எண்ண முடியாத விருப்பங்கள் தனியாக கண்காணிக்கப்படுகின்றன ஆனால் முன்னேற்றத்திலிருந்து விலக்கப்படுகின்றன
- விருப்பம் நீக்கப்படும்போது, முகவரிகளுக்கு தானியங்கியாக இயல்புநிலை விருப்பம் சேர்க்கப்படுகிறது

**அணுகல் விதிகள்:**
- **பட்டியல்/பார்வை:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **உருவாக்கு/புதுப்பிக்கு/நீக்கு:** `/options/update` endpoint பயன்படுத்தவும் (Conductor அல்லது Administrator)

**உறவுகள்:**
- Addresses உடன் M:M (address.type புலத்தில் JSON array ஆக சேமிக்கப்படுகிறது)

**எடுத்துக்காட்டு:**
```json
{
  "id": "opt_res",
  "congregation": "cong_123",
  "code": "RES",
  "description": "Residential",
  "is_countable": true,
  "is_default": true,
  "sequence": 1
}
```

### 9. Messages

**செய்தி வகைகள்:**
- `publisher` - வெளியீட்டாளர்களிடமிருந்து நிர்வாகிகளுக்கு

- `administrator` - நிர்வாகி அறிவுறுத்தல்கள் (சொருக முடியும்)

**குறியீடுகள்:**
- `(map, pinned, created)` - சமீபத்திய சொருகப்பட்ட செய்திகள்
- `(map, type, pinned)` - பெறுநர் வகை மற்றும் சொருகல் நிலை வழியாக வடிகட்டு
- `(map, type, read)` - வகைக்கு படிக்காத எண்ணிக்கை

**அணுகல் விதிகள்:**
- **உருவாக்கு:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்
- **புதுப்பிக்கு:** நிர்வாகிகள் மட்டும்
- **பட்டியல்:** பாத்திரமுடைய பயனர் அல்லது இணைப்பு அணுகல்

**பின்னணி செயலாக்கம்:**
- ஒவ்வொரு 30 நிமிடத்திற்கும்: படிக்காத செய்திகள் நிர்வாகிகளுக்கு மின்னஞ்சல் வழியாக அனுப்பப்படுகின்றன
- ஒவ்வொரு 30 நிமிடத்திற்கும்: சொருகப்பட்ட நிர்வாகி செய்திகள் வெளியீட்டாளர்களுக்கு மின்னஞ்சல் வழியாக அனுப்பப்படுகின்றன

### 10. Addresses Log

**நோக்கம்:** முகவரி நிலை மாற்றங்களின் தணிக்கை தடம் — ஒரு முகவரி நிலை புதுப்பிக்கப்படும் ஒவ்வொரு முறையும், இங்கே ஒரு பதிவு எழுதப்படுகிறது

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |
|-------|------|----------|-------------|
| `id` | text(15) | ஆம் | தானியங்கி-உருவாக்கப்பட்ட தனித்துவ ID |
| `address` | relation | ஆம் | Addresses க்கு FK (cascade delete) |
| `old_status` | text | ஆம் | முந்தைய முகவரி நிலை மதிப்பு |
| `new_status` | text | ஆம் | புதிய முகவரி நிலை மதிப்பு |
| `changed_by` | relation | இல்லை | Users க்கு FK — மாற்றம் செய்த பயனர் |
| `created` | date | தானியங்கி | மாற்றத்தின் நேரமுத்திரை |

**உறவுகள்:**
- Addresses உடன் M:1 (ஒவ்வொரு பதிவும் ஒரு முகவரியை சேர்ந்தது)
- Users உடன் M:1 (`changed_by` வழியாக)

**அணுகல் விதிகள்:**
- **பட்டியல்/பார்வை:** Conductor அல்லது Administrator
- **உருவாக்கு:** கணினி hook (முகவரி நிலை மாற்றத்தில் தானியங்கி)

### 11. Aggregates

**நோக்கம்:** பகுதிகள் மற்றும் வரைபடங்களுக்கான தேக்கப்பட்ட முன்னேற்ற snapshots. ஒவ்வொரு 10 நிமிடத்திற்கும் ஒரு பின்னணி வேலையால் மறு கணக்கிடப்படுகிறது

**வகை:** அடிப்படை தொகுப்பு

**புலங்கள்:**

| புலம் | வகை | தேவையா | விளக்கம் |

- மதிப்புகள் படிக்க மட்டும் snapshots; நேரடி முன்னேற்றம் கோரிக்கையின்போது கணக்கிடப்படுகிறது

**அணுகல் விதிகள்:**

## பகுப்பாய்வு தொகுப்புகள்

பகுப்பாய்வு தொகுப்புகள் அறிக்கையிடல் மற்றும் நுண்ணறிவுக்கான வரலாற்று தரவை சேமிக்கின்றன. இந்த தொகுப்புகள் உள்நாட்டில் நிர்வகிக்கப்படுகின்றன மற்றும் அவற்றின் சரியான புலங்கள் காலப்போக்கில் மாறலாம்; அவை தனிப்பட்ட புலங்களை விட நோக்கம் கொண்டு இங்கே ஆவணப்படுத்தப்பட்டுள்ளன.

### analytics_territories

**நோக்கம்:** காலப்போக்கில் பகுதி-அளவிலான அளவீடுகளின் (எ.கா., முன்னேற்ற சதவீதம், முகவரி எண்ணிக்கை) காலாண்டு snapshots சேமிக்கிறது. வரலாற்று போக்கு விளக்கப்படங்கள் மற்றும் பகுதி நிறைவு அறிக்கைகளை உருவாக்க பயன்படுத்தப்படுகிறது.

### analytics_maps

**நோக்கம்:** காலப்போக்கில் வரைபட-அளவிலான அளவீடுகளின் காலாண்டு snapshots சேமிக்கிறது. தனிப்பட்ட வரைபட நிறைவு விகிதங்கள் மற்றும் செயல்பாட்டு முறைகள் பற்றிய நுண்ணிய அறிக்கையிடலை இயக்குகிறது.

### analytics_daily_status

**நோக்கம்:** சபையில் முகவரி நிலைகளின் (முடிந்தது, முடிக்கப்படவில்லை, வீட்டில் இல்லை, அழைக்க வேண்டாம், செல்லாத) தினாந்திர திரட்டப்பட்ட எண்ணிக்கைகளை பதிவு செய்கிறது. நாளுக்கு நாள் மற்றும் வாரந்திர போக்கு dashboards இயக்குகிறது.

### analytics_not_home

**நோக்கம்:** திரும்பத்திரும்ப "வீட்டில் இல்லை" என குறிக்கப்பட்ட முகவரிகளை கண்காணிக்கிறது. conductors க்கு தொடர்தொடர் இலக்குகளை மேலே கொண்டு வர மற்றும் தொடர்ந்து எட்டமுடியாத முகவரிகளை அடையாளம் காண பயன்படுத்தப்படுகிறது.

### analytics_user_audit

**நோக்கம்:** பொறுப்பு மற்றும் தணிக்கை நோக்கங்களுக்காக உள்நுழைவுகள், முகவரி புதுப்பிப்புகள் மற்றும் பணிப்பு செயல்பாடுகள் உட்பட பயனர் செயல்களை பதிவு செய்கிறது. நிர்வாகிகள் செயல்பாட்டு வரலாற்றை மதிப்பாய்வு செய்ய மற்றும் முரண்பாடுகளை ஆராய இயக்குகிறது.

> **குறிப்பு:** பகுப்பாய்வு தொகுப்புகள் பின்னணி வேலைகளால் நிரப்பப்படுகின்றன மற்றும் அறிக்கையிடல் அடுக்கால் படிக்கப்படுகின்றன. பயன்பாட்டு குறியீட்டிலிருந்து நேரடி எழுத்துகளை தவிர்க்க வேண்டும்.

**பயன்படுத்தப்படும் SQL அம்சங்கள்:**
- நெகிழ்வான தரவுக்கு JSON புலங்கள் (ஆயங்கள், aggregates)

- ACID இணக்கத்திற்கு Transactions
- சிக்கலான திரட்டல்களுக்கு CTEs (Common Table Expressions)

**நன்மைகள்:**
- சுயேச்சையான கோப்பு-அடிப்படையிலான தரவுத்தளம்
- தனி தரவுத்தள சேவை தேவையில்லை
- சுய-ஹோஸ்ட் வரிசைப்படுத்தல்களுக்கு சரியானது
- CGO சார்பு இல்லாமல் நிலையான binary கட்டமைப்புகளை இயக்குகிறது
- பாதுகாப்பு திட்டுகளுடன் நிரூபிக்கப்பட்ட நிலைத்தன்மை

**எண்ணக்கூடிய முகவரிகள்:**
- `is_countable=true` விருப்பங்களுடன் மட்டுமே முகவரிகள்
- DNC மற்றும் செல்லாத முகவரிகளை விலக்குகிறது

**நிறைவு நிபந்தனைகள்:**
- நிலை = `done`, அல்லது
- நிலை = `not_home` மற்றும் `not_home_tries >= congregation.max_tries`

### முகவரி வாழ்க்கைச் சுழற்சி

```
not_done → not_home (retry attempts) → done
        → do_not_call (excluded)
        → invalid (excluded)
```

### தானியங்கி புதுப்பிப்பு Triggers

1. **முகவரி புதுப்பிப்பு:**
   - குறிப்புகள் மாறினால் `last_notes_updated` ஐ புதுப்பிக்கிறது
   - `last_notes_updated_by` ஐ தற்போதைய பயனருக்கு அமைக்கிறது
   - aggregate மறு கணக்கீட்டை தூண்டுகிறது
   - வரைபட முன்னேற்றத்தை புதுப்பிக்கிறது
   - பகுதி முன்னேற்றத்தை புதுப்பிக்கிறது

2. **வரைபட செயல்பாடுகள்:**
   - முகவரிகள் மாறும்போது aggregates ஐ மறு கணக்கிடுகிறது
   - பகுதி முன்னேற்றத்தை புதுப்பிக்கிறது

3. **பணிப்பு உருவாக்கம்:**
   - சபை அமைப்புகளின் அடிப்படையில் காலாவதியை அமைக்கிறது
   - அணுகல் டோக்கனாக தனித்துவமான 20-எழுத்து ID ஐ உருவாக்குகிறது
