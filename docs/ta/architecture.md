## தரவு கட்டமைப்பு

### நிறுவன உறவு மாதிரி

கணினி சபை-அளவிலான தனிமைப்படுத்தலுடன் படிநிலையாக தரவை ஒழுங்கமைக்கிறது:

```
Congregation (Organization boundary)
    ├── Territory (Geographic region)
    │   ├── Map (Building/Location)
    │   │   └── Address (Individual unit)
    │   └── Option (Address type classification)
    ├── User (System user)
    │   └── Role (Permission assignment)
    ├── Assignment (Publisher access)
    └── Message (Notification)
```

### முக்கிய தொகுப்புகள்

**1. Congregations** - நிறுவன அமைப்புகள் மற்றும் எல்லைகள்
- பல-குத்தகை தனிமைப்படுத்தலை கட்டுப்படுத்துகிறது
- பணிப்பு கால அளவு மற்றும் மீண்டும் முயற்சி வரம்புகளை வரையறுக்கிறது
- பிராந்திய அமைப்புகள் (நேர மண்டலம், நாடு)

**2. Territories** - புவியியல் பிரிவுகள்
- பல வரைபடங்களை உள்ளடக்கியது
- ஒருங்கிணைந்த நிறைவு சதவீதத்தை கண்காணிக்கிறது
- வயல் சேவை பகுதிகளை ஒழுங்கமைக்கிறது

**3. Maps** - குறிப்பிட்ட இடங்கள் (கட்டிடங்கள்)
- தனி அல்லது பல-தள கட்டமைப்புகள்
- புவியியல் ஆயங்கள்
- முன்னேற்ற கண்காணிப்பு மற்றும் புள்ளிவிவரங்கள்

**4. Addresses** - தனிப்பட்ட அலகுகள்/வீடுகள்
- நிலை கண்காணிப்பு (முடிக்கப்படவில்லை, முடிந்தது, வீட்டில் இல்லை, DNC)
- வருகை குறிப்புகள் மற்றும் வரலாறு
- தள மற்றும் அலகு தகவல்

**5. Users** - கணினி கணக்குகள்
- மின்னஞ்சல்/கடவுச்சொல் அல்லது OAuth2 அங்கீகாரம்
- பல-காரணி அங்கீகார ஆதரவு
- சபை-குறிப்பிட்ட பாத்திரங்கள்

**6. Roles** - அனுமதி ஒதுக்கீடுகள்
- மூன்று நிலைகள்: read_only, conductor, administrator
- சபை-குறிப்பிட்ட அனுமதிகள்
- பாத்திர-அடிப்படையிலான அணுகல் கட்டுப்பாடு

**7. Assignments** - வெளியீட்டாளர் அணுகல் டோக்கன்கள்
- நேர-வரையறுக்கப்பட்ட அணுகல் இணைப்புகள்
- அநாமதேய வயல் தொழிலாளர் அணுகல்
- தானியங்கி காலாவதி

**8. Options** - முகவரி வகைப்படுத்தல்கள்
- சபைக்கு ஏற்ப தனிப்பயனாக்கக்கூடியது
- எண்ணக்கூடிய மற்றும் எண்ண முடியாத வகைகள்
- முன்னேற்ற கணக்கீடுகளை பாதிக்கிறது

**9. Messages** - தகவல்தொடர்புகள்
- வெளியீட்டாளர் கருத்துக்கள்
- நிர்வாகி அறிவுறுத்தல்கள்
- மின்னஞ்சல் அறிவிப்புகள்

### நிலை நிர்வாக உத்தி

**நான்கு-அடுக்கு அணுகுமுறை:**

1. **Global Context** - கருப்பொருள், மொழி, பயன்பாட்டு முறை
2. **Custom Hooks** - டொமைன்-குறிப்பிட்ட தர்க்கம் மற்றும் தரவு
3. **Local Component State** - UI மாற்றிகள் மற்றும் படிவங்கள்
4. **Persistent State** - விருப்பத்தேர்வுகளுக்கு LocalStorage

**நன்மைகள்:**
- Redux மேல்பட்டை இல்லை
- எளிமையான தரவு ஓட்டம்
- சிறந்த குறியீடு பிரித்தல்
- எளிதான சோதனை

## பாதுகாப்பு கட்டமைப்பு

### அங்கீகார முறைகள்

1. **Email/Password** - bcrypt hashing உடன் நிலையான அங்கீகாரம்
2. **OAuth2 (Google)** - PKCE flow உடன் சமூக உள்நுழைவு
3. **One-Time Password (OTP)** - கூடுதல் பாதுகாப்பு அடுக்கு
4. **Multi-Factor Authentication (MFA)** - நிர்வாகிகளுக்கு மேம்பட்ட பாதுகாப்பு

### அங்கீகார மாதிரி

**பாத்திர-அடிப்படையிலான அணுகல் கட்டுப்பாடு (RBAC):**

| பாத்திரம் | அனுமதிகள் | பயன்பாட்டு வழக்கு |
|------|-------------|----------|
| **read_only** | தரவை மட்டும் பார்க்க | கண்காணிப்பாளர்கள், தணிக்கையாளர்கள் |
| **conductor** | பணிப்புகளை உருவாக்க, வரைபடங்களை நிர்வகிக்க | வயல் ஒருங்கிணைப்பாளர்கள் |
| **administrator** | முழு அணுகல் | கணினி நிர்வாகிகள் |

### அணுகல் கட்டுப்பாட்டு முறைகள்

**முறை 1: பாத்திரமுடைய அங்கீகரிக்கப்பட்ட பயனர்**
```
User must be logged in
+ User has role in roles collection
+ Role is for specific congregation
```

**முறை 2: இணைப்பு-அடிப்படையிலான அணுகல்**
```
Assignment ID passed in header
+ Assignment exists and not expired
+ Assignment belongs to requested map
```

**முறை 3: சபை தனிமைப்படுத்தல்**
```
Every entity has congregation foreign key
+ Access rules filter by congregation
+ Prevents cross-congregation data access
```

## நிகழ்-நேர ஒத்திசைவு

### SSE கட்டமைப்பு

- **PocketBase Real-time API**: Server-Sent Events (SSE)
- **தானியங்கி சந்தாக்கள்**: கூறுகள் தொகுப்புகளுக்கு சந்தா செய்கின்றன
- **நேரடி புதுப்பிப்புகள்**: மாற்றங்கள் இணைக்கப்பட்ட அனைத்து கிளையண்டுகளுக்கும் ஒளிபரப்பப்படுகின்றன
- **முரண்பாட்டு தீர்வு**: நேரமுத்திரை-அடிப்படையிலான தீர்வு
- **தெரிவுநிலை கண்டறிதல்**: தாவல் செயல்படும்போது தானியங்கி புதுப்பிப்பு

### சந்தா முறை

```typescript
// Components subscribe to data changes
useRealtimeSubscription('addresses', (data) => {
  // UI updates automatically
});
```

### அம்சக் கொடி கட்டுப்பாடு

அனைத்து பின்னணி வேலைகளும் LaunchDarkly அம்சக் கொடிகளால் கட்டுப்படுத்தப்படுகின்றன:
- கட்டமைப்பு இல்லாமல் வேலைகளை இயக்கு/நிறுத்து
- படிப்படியான வெளியீட்டு திறன்கள்
- அவசர நிறுத்தி

### AI/LLM ஒருங்கிணைப்பு

Ministry Mapper புத்திசாலித்தனமான உரை சுருக்கத்திற்காக OpenAI GPT மாதிரியுடன் ஒருங்கிணைகிறது. இது `enable-report-ai-summary` LaunchDarkly கொடியால் கட்டுப்படுத்தப்படும் ஒரு **விருப்பமான** அம்சமாகும்.

**மாதிரி**: GPT (சமீபத்திய கிடைக்கக்கூடிய `openai-go` SDK v3.29.0 வழியாக)  
**Temperature**: 0.3 (உண்மையான, நிர்ணயிக்கக்கூடிய வெளியீடு)  
**பதில் வடிவம்**: கட்டமைக்கப்பட்ட தரவுக்கு JSON முறை

**பயன்பாட்டு வழக்குகள்:**

| சூழல் | வேலை | வெளியீடு |
|---------|-----|--------|
| மாதாந்திர Excel அறிக்கைகள் | `generateMonthlyReport` | பகுதிக்கான `summary` மற்றும் `key_points` |
| வெளியீட்டாளர் செய்தி சுருக்கங்கள் | `processMessages` | `overview` மற்றும் `key_themes` |
| சபை குறிப்பு சுருக்கங்கள் | `processNotes` | `overview` மற்றும் `key_themes` |

**நேர்மையான தரமிறக்கம்**: `OPENAI_API_KEY` அமைக்கப்படவில்லை அல்லது `enable-report-ai-summary` கொடி முடக்கப்பட்டிருந்தால், கணினி AI-உருவாக்கப்பட்ட உள்ளடக்கம் இல்லாமல் அறிக்கைகள் மற்றும் மின்னஞ்சல்களை வழங்கும் — எந்த பிழைகளும் எழாது.



### முன்தள உகப்பாக்கங்கள்

- **குறியீடு பிரித்தல்**: சோம்பல் ஏற்றப்பட்ட routes
- **Bundle உகப்பாக்கம்**: Tree shaking, குறைத்தல்
- **React 19 Compiler**: தானியங்கி memoization
- **PWA தேக்கல்**: நிலையான சொத்துகளுக்கு service worker
- **மெய்நிகர் பட்டியல்கள்**: பெரிய தரவுத்தொகுப்புகளுக்கு React-window

### பின்தள உகப்பாக்கங்கள்

- **தரவுத்தள குறியீடுகள்**: விரைவான வினவல்களுக்கு 25+ குறியீடுகள்
- **Aggregate தேக்கல்**: பகுதி முன்னேற்ற புள்ளிவிவரங்கள் ஒரு தொகுத்த cron வேலை மூலம் முன்-கணக்கிடப்படுகின்றன (`updateTerritoryAggregates`, ஒவ்வொரு 10 நிமிடத்திற்கும் 11-நிமிட lookback சாளரத்துடன்), முந்தைய நிகழ்-நேர debouncer அணுகுமுறையை மாற்றி. முடிவுகள் விரைவான மீட்டெடுப்பிற்காக JSON ஆக சேமிக்கப்படுகின்றன.
- **இணைப்பு குளம்**: திறமையான தரவுத்தள இணைப்புகள்
- **Transaction தொகுத்தல்**: தரவுத்தள சுற்று-பயணங்களை குறைக்க

## தொழில்நுட்ப முடிவுகள்

### ஏன் இந்த தொழில்நுட்பங்கள்?

**React 19**: சமீபத்திய அம்சங்கள், தானியங்கி உகப்பாக்கம், சிறிய bundle

**TypeScript**: வகை பாதுகாப்பு, சிறந்த IDE ஆதரவு, குறைவான இயக்க நேர பிழைகள்

**Bootstrap**: போர்-சோதிக்கப்பட்ட, விரிவான கூறுகள், நல்ல அணுகல்தன்மை

**Leaflet**: இலவச வரைபடங்கள், API வரம்புகள் இல்லை, இலகுரக

**SQLite**: சுயேச்சையானது, தனி தரவுத்தள சர்வர் தேவையில்லை, சுய-ஹோஸ்டிங்கிற்கு சரியானது
