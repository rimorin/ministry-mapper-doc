## சொந்தமாக ஹோஸ்ட் செய்வதை நாங்கள் ஊக்குவிக்காத காரணம்

Ministry Mapper திறந்த மூலமாக இருந்தாலும், சொந்தமாக ஹோஸ்ட் செய்வதில் சவால்கள் உள்ளன:

- **தொழில்நுட்ப சிக்கலானது**: Docker, தரவுத்தளங்கள், வலை சேவையகங்கள் மற்றும் cloud உள்கட்டமைப்பு பற்றிய அறிவு தேவை
- **பாதுகாப்பு பொறுப்பு**: அனைத்தையும் பாதுகாப்பாகவும் புதுப்பித்தும் வைத்திருப்பதற்கு நீங்களே பொறுப்பு
- **பராமரிப்பு சுமை**: வழக்கமான புதுப்பிப்புகள், காப்புப்பிரதிகள் மற்றும் கண்காணிப்பு அவசியம்
- **தனியுரிமை இணக்கம்**: GDPR, CCPA மற்றும் பிற தனியுரிமை சட்ட இணக்கத்தை நீங்களே உறுதி செய்ய வேண்டும்
- **செலவு**: சேவையக செலவுகள், API செலவுகள் மற்றும் நேர முதலீடு பெரும்பாலும் ஹோஸ்ட் செய்யப்பட்ட சேவை கட்டணங்களை மீறும்
- **ஆதரவு**: சொந்தமாக ஹோஸ்ட் செய்யும் சிக்கல்களுக்கு சீமித சமூக ஆதரவு

## சொந்தமாக ஹோஸ்ட் செய்வதற்கான முன்நிபந்தனைகள்

எங்கள் பரிந்துரைகளை மீறி சொந்தமாக ஹோஸ்ட் செய்ய முடிவு செய்திருந்தால், பின்வருபவை உங்களிடம் உள்ளதா என உறுதிப்படுத்தவும்:

### தொழில்நுட்ப தேவைகள்
- Linux சேவையக நிர்வாகத்தில் அனுபவம்
- Docker மற்றும் containerization-ன் புரிதல்
- சூழல் மாறிகள் மற்றும் கட்டமைப்பு பற்றிய நடைமுறை அறிவு
- வலை சேவையக கட்டமைப்பின் அறிவு (Nginx/Apache)
- SSL/TLS சான்றிதழ்களில் அனுபவம்
- DNS மற்றும் டொமைன் கட்டமைப்பின் புரிதல்

### உள்கட்டமைப்பு தேவைகள்
- Cloud ஹோஸ்டிங் சேவை (Railway, Render, DigitalOcean, AWS போன்றவை)
- உங்கள் நிகழ்விற்கான டொமைன் பெயர்
- அறிவிப்புகளுக்கான மின்னஞ்சல் சேவை (விருப்பத்தேர்வு)
- விருப்பத்தேர்வு API செலவுகளுக்கான பட்ஜெட் (Sentry போன்றவை)

### நேர அர்ப்பணிப்பு

## Backend சொந்த ஹோஸ்டிங் வழிகாட்டி

### சூழல் கட்டமைப்பு

இந்த அமைவுகளுடன் `.env` கோப்பை உருவாக்கவும்:

```bash
# Application Settings
PB_APP_NAME=Ministry Mapper
PB_APP_URL=https://your-frontend-domain.com

# Initial Admin Account (change password after first login!)
PB_ADMIN_EMAIL=admin@your-domain.com
PB_ADMIN_PASSWORD=change_this_secure_password

# Email Configuration (SMTP)
PB_SMTP_HOST=smtp.gmail.com
PB_SMTP_PASSWORD=your_app_password
PB_SMTP_USERNAME=your_email@gmail.com
PB_SMTP_PORT=587
PB_SMTP_SENDER_ADDRESS=noreply@your-domain.com
PB_SMTP_SENDER_NAME=Ministry Mapper

# Security Settings
PB_ALLOW_ORIGINS=https://your-frontend-domain.com
PB_HIDE_CONTROLS=true
PB_ENABLE_RATE_LIMITING=true

# Authentication (optional)
PB_OTP_ENABLED=false
PB_MFA_ENABLED=false

# Error Tracking (optional but recommended)
SENTRY_DSN=your_sentry_dsn
SENTRY_ENV=production
SOURCE_COMMIT=

# Service Integration (optional)
MAILERSEND_API_KEY=
MAILERSEND_FROM_EMAIL=
LAUNCHDARKLY_SDK_KEY=
LAUNCHDARKLY_CONTEXT_KEY=
```

### Docker பயன்படுத்தல்

1. **இணைப்பியை குளோன் செய்யவும்**
```bash
git clone https://github.com/rimorin/ministry-mapper-be.git
cd ministry-mapper-be
```

2. **Docker படம் உருவாக்கவும்**
```bash
docker build -t ministry-mapper-backend .
```

3. **Container இயக்கவும்**
```bash
docker run -d \
  -p 8080:8080 \
  -v /path/to/pb_data:/app/pb_data \
  --env-file .env \
  --name ministry-mapper \
  ministry-mapper-backend
```

**முக்கிய குறிப்புகள்:**
- Container உள்நாட்டில் போர்ட் 8080 இல் இயங்குகிறது
- தரவை பாதுகாக்க `/app/pb_data` க்கு நிலையான volume ஐ இணைக்கவும்
- உங்கள் தரவுத்தளம் `pb_data` கோப்புறையில் உள்ளது

4. **நிர்வாக பலகத்தை அணுகவும்**
`https://your-backend-domain.com/_/` க்கு செல்லவும் மற்றும் உங்கள் நிர்வாக சான்றுகளுடன் உள்நுழையவும்.

#### சொந்தமாக ஹோஸ்ட் செய்யப்பட்ட வலை சேவையகம்

**Nginx கட்டமைப்பு:**

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    root /var/www/ministry-mapper/build;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # Caching
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
    
    # Security headers
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
}
```

**Apache கட்டமைப்பு:**

```apache
<VirtualHost *:443>
    ServerName your-domain.com
    DocumentRoot /var/www/ministry-mapper/build
    
    SSLEngine on
    SSLCertificateFile /path/to/cert.pem
    SSLCertificateKeyFile /path/to/key.pem

    <Directory /var/www/ministry-mapper/build>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted

        RewriteEngine On
        RewriteBase /
        RewriteRule ^index\.html$ - [L]
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule . /index.html [L]
    </Directory>
</VirtualHost>
```

## சொந்தமாக ஹோஸ்ட் செய்யப்பட்ட நிகழ்வுகளை சிக்கல் தீர்த்தல்

### Backend சிக்கல்கள்

**போர்ட் ஏற்கனவே பயன்பாட்டில் உள்ளது:**
```bash
lsof -i :8080
kill -9 <process_id>
```

**தரவுத்தளம் பூட்டப்பட்டது:**
- தரவுத்தளத்தை அணுகும் அனைத்து நிகழ்வுகளையும் நிறுத்தவும்
- ஒரே ஒரு backend செயல்முறை மட்டுமே இயங்குவதை உறுதிப்படுத்தவும்
- தொங்கும் செயல்முறைகளை சரிபார்க்கவும்

**மின்னஞ்சல் அனுப்பப்படவில்லை:**
- SMTP சான்றுகளை சரிபார்க்கவும்
- Firewall SMTP போர்ட்களை தடுக்கவில்லை என சரிபார்க்கவும்
- Gmail-க்கு, App Passwords பயன்படுத்தவும்
- `pb_data/logs/` இல் மின்னஞ்சல் பதிவுகளை மதிப்பாய்வு செய்யவும்

**நிர்வாக பலகத்தை அணுக முடியவில்லை:**
- URL `https://your-domain.com/_/` ஆக இருக்க வேண்டும் (அடிக்கோடை கவனிக்கவும்)
- `PB_HIDE_CONTROLS` அமைப்பை சரிபார்க்கவும்
- Backend இயங்குகிறதா என சரிபார்க்கவும்
- உலாவி cache ஐ அழிக்கவும்

### Frontend சிக்கல்கள்

**Backend-ஐ இணைக்க முடியவில்லை:**
- `VITE_POCKETBASE_URL` சரியானதா என சரிபார்க்கவும்
- Backend-இல் CORS அமைவுகளை சரிபார்க்கவும்
- Backend அணுகக்கூடியதா என உறுதிப்படுத்தவும்
- உலாவி console-ல் பிழைகளை சரிபார்க்கவும்

**உருவாக்கல் தோல்விகள்:**
- Node.js பதிப்பு 22+ உறுதிப்படுத்தவும்
- `node_modules` நீக்கி மீண்டும் நிறுவவும்
- அனைத்து சூழல் மாறிகளும் அமைக்கப்பட்டுள்ளனவா சரிபார்க்கவும்
- உருவாக்கல் பிழை செய்திகளை மதிப்பாய்வு செய்யவும்

### செயல்திறன் சிக்கல்கள்

**மெதுவான பதில் நேரங்கள்:**
- சேவையக வளங்களை சரிபார்க்கவும் (CPU, RAM, வட்டு)
- தரவுத்தள அளவு மற்றும் queries மதிப்பாய்வு செய்யவும்
- தரவுத்தள மேம்படுத்தல்களை இயக்கவும்
- நெட்வொர்க் தாமதத்தை சரிபார்க்கவும்
- மெதுவான queries-க்கு PocketBase பதிவுகளை மதிப்பாய்வு செய்யவும்

## தனியுரிமை மற்றும் சட்ட இணக்கம்

### சொந்தமாக ஹோஸ்ட் செய்பவராக உங்கள் பொறுப்புகள்

சொந்தமாக ஹோஸ்ட் செய்யும்போது, நீங்கள் தரவு கட்டுப்படுத்தியாக இருக்கிறீர்கள் மற்றும் பின்வருவனவற்றை செய்ய வேண்டும்:

- **தனியுரிமை சட்டங்களை கடைபிடிக்கவும்**: GDPR (EU), CCPA (California) போன்றவை
- **தனியுரிமை கொள்கை உருவாக்கவும்**: தரவு சேகரிப்பு மற்றும் பயன்பாட்டை விளக்கவும்
- **சேவை விதிமுறைகள் உருவாக்கவும்**: பயனர் பொறுப்புகளை வரையறுக்கவும்
- **தரவு பாதுகாப்பை செயல்படுத்தவும்**: குறியாக்கம், அணுகல் கட்டுப்பாடுகள், காப்புப்பிரதிகள்
- **தரவு கோரிக்கைகளை கையாளவும்**: பயனர் அணுகல், நீக்கம், பெயர்வு உரிமைகள்
- **தரவு மீறல்களை தெரிவிக்கவும்**: சட்டப்பூர்வ காலக்கெடுவிற்குள்
- **பதிவுகளை பராமரிக்கவும்**: தரவு செயலாக்க செயல்பாடுகள்
- **DPO நியமிக்கவும்**: விதிமுறைகளால் தேவைப்பட்டால்

### தேவையான சட்ட ஆவணங்கள்

நீங்கள் வழங்க வேண்டும்:

1. **தனியுரிமை கொள்கை** - சட்டத்தால் தேவை, விளக்க வேண்டும்:
   - நீங்கள் எந்த தரவை சேகரிக்கிறீர்கள்
   - நீங்கள் அதை எவ்வாறு பயன்படுத்துகிறீர்கள்
   - எவ்வளவு காலம் வைத்திருக்கிறீர்கள்
   - பயனர் உரிமைகள்
   - உங்களை எவ்வாறு தொடர்பு கொள்வது

2. **சேவை விதிமுறைகள்** - வரையறுக்கிறது:
   - ஏற்கத்தக்க பயன்பாடு
   - பயனர் பொறுப்புகள்
   - பொறுப்பு வரம்புகள்
   - நிறுத்தல் கொள்கைகள்

3. **Cookie கொள்கை** - நீங்கள் cookies/கண்காணிப்பு பயன்படுத்தினால்

### தரவு பாதுகாப்பு நடவடிக்கைகள்

- எல்லா இடத்திலும் HTTPS பயன்படுத்தவும்
- ஓய்வில் உள்ள தரவை குறியாக்கம் செய்யவும்
- அணுகல் கட்டுப்பாடுகளை செயல்படுத்தவும்
- வழக்கமான பாதுகாப்பு தணிக்கைகள்
- காப்புப்பிரதி குறியாக்கம்
- பாதுகாப்பான கடவுச்சொல் கொள்கைகள்
- பல்-காரணி அங்கீகரணம்
- செயல்பாடு பதிவு
- வழக்கமான பாதுகாப்பு புதுப்பிப்புகள்
