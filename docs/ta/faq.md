### நான் என்ன தனியுரிமை சட்டங்களை கருத்தில் கொள்ள வேண்டும்?

**முக்கியம்**: Ministry Mapper குடியிருப்பு முகவரிகளை கண்காணிக்கிறது, இது தரவு தனியுரிமை சட்டங்களுக்கு உட்பட்டிருக்கலாம். இந்த சட்டங்கள் நாடுகள் மற்றும் பகுதிகளிடையே கணிசமாக வேறுபடுகின்றன:

- **ஐரோப்பா**: GDPR (பொது தரவு பாதுகாப்பு ஒழுங்காற்று)
- **கலிபோர்னியா**: CCPA (கலிபோர்னியா நுகர்வோர் தனியுரிமை சட்டம்)
- **பிரேசில்**: LGPD (Lei Geral de Proteção de Dados)
- **பல மற்றவை**: பல்வேறு உள்ளூர் ஒழுங்குமுறைகள்

**Ministry Mapper ஐ பயன்படுத்துவதற்கு முன்பு உங்கள் உள்ளூர் ஒழுங்குமுறைகளை முழுமையாக மதிப்பாய்வு செய்து இணக்கத்தை உறுதிசெய்யவும்.** உங்கள் பகுதிக்கு குறிப்பிட்ட சரியான சட்ட ஆலோசனைக்கு ஒரு சட்ட நிபுணரை அணுகவும்.

**நேர சேவை**: [ministry-mapper.com](https://ministry-mapper.com) இல் நேர சேவை இணக்க உதவியை வழங்கலாம், ஆனால் உங்கள் உள்ளூர் சட்டங்களை பின்பற்றுவதற்கு நீங்கள் பொறுப்பாளி.

**சுய-நேர்விடல்**: அனைத்து தனியுரிமை சட்ட இணக்கத்திற்கும் நீங்கள் முழு பொறுப்பாளி.

### என்ன சுற்றுச்சூழல் மாறிகள் தேவைப்படுகின்றன?

**சுய-நேர்விடலுக்கு மட்டும்** - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இந்த பிரிவு பொருந்தாது. நேர சேவை அனைத்து கட்டமைப்பையும் தானியங்கியாக கையாளுகிறது.

**முன்-முனை (.env கோப்பு):**

```
VITE_SYSTEM_ENVIRONMENT=local
VITE_VERSION=$npm_package_version
VITE_POCKETBASE_URL=your_pocketbase_backend_url
VITE_OPENROUTE_API_KEY=your_openrouteservice_key
VITE_LOCATIONIQ_API_KEY=your_locationiq_key
VITE_SENTRY_DSN=your_sentry_dsn (optional)
VITE_PRIVACY_URL=your_privacy_policy_url
VITE_TERMS_URL=your_terms_url
VITE_ABOUT_URL=your_about_page_url
```

**உற்பத்திக்கு:** மேலே உள்ளது போன்ற அதே மாறிகள், ஆனால் `VITE_SYSTEM_ENVIRONMENT=production` உடன்

முழு விவரங்களுக்கு [Frontend Setup Guide](frontend-setup.md) ஐ பாருங்கள்.

### முன்-முனையை எவ்வாறு வரிசைப்படுத்துவது?

**சுய-நேர்விடலுக்கு மட்டும்** - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இதை தவிர்க்கவும்.

1. **பயன்பாட்டை உருவாக்கவும்:**

   ```bash
   npm run build
   ```

2. மேலே குறிப்பிட்டபடி **சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும்**

3. உங்கள் நேர்விடல் வழங்குனருக்கு **`build/` கோப்புறையை வரிசைப்படுத்தவும்**:

   - Vercel: உங்கள் GitHub repo ஐ இணைத்து சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும்
   - Netlify: build கோப்புறையை இழுத்து விடவும் அல்லது Git வழியாக இணைக்கவும்
   - AWS S3: build கோப்புறையை பதிவேற்றி நிலையான வலைத்தள நேர்விடலை கட்டமைக்கவும்

4. **PocketBase பின்-முனை வரிசைப்படுத்தப்பட்டுள்ளது** மற்றும் `VITE_POCKETBASE_URL` இல் குறிப்பிட்ட URL இல் அணுகக்கூடியதாக இருப்பதை **உறுதிசெய்யவும்**

விரிவான வழிமுறைகளுக்கு [Frontend Setup Guide](frontend-setup.md) மற்றும் [Self-Hosting Guide](self-hosting.md) ஐ பாருங்கள்.

### PocketBase பின்-முனையை எவ்வாறு அமைப்பது?

**சுய-நேர்விடலுக்கு மட்டும்** - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இதை தவிர்க்கவும்.

பின்-முனை தனி களஞ்சியத்தில் நிர்வகிக்கப்படுகிறது: [ministry-mapper-be](https://github.com/rimorin/ministry-mapper-be)

**விரைவு கண்ணோட்டம்:**

1. பின்-முனை களஞ்சியத்தை குளோன் செய்யவும்
2. சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும் ([Backend Setup Guide](backend-setup.md) ஐ பாருங்கள்)
3. Docker அல்லது Railway/Render பயன்படுத்தி வரிசைப்படுத்தவும்
4. திட்டத்துடன் தரவுத்தளத்தை துவக்கவும்
5. மின்னஞ்சல் அறிவிப்புகளுக்கு SMTP ஐ கட்டமைக்கவும் (விருப்பமானது)

**விரிவான வழிமுறைகள்:** [Backend Setup Guide](backend-setup.md) மற்றும் [Self-Hosting Guide](self-hosting.md) ஐ பாருங்கள்.

**முக்கியம்**: முன்-முனையை வரிசைப்படுத்துவதற்கு முன்பு பின்-முனை வரிசைப்படுத்தப்பட்டு அணுகக்கூடியதாக இருக்க வேண்டும்.

### Sentry கண்காணிப்பை எவ்வாறு அமைப்பது?

**சுய-நேர்விடலுக்கு மட்டும்** - நேர சேவையை பயன்படுத்துகிறீர்கள் என்றால் இதை தவிர்க்கவும்.

1. [Sentry](https://sentry.io/) கணக்கை உருவாக்கவும்
2. ஒரு React திட்டத்தை உருவாக்கவும்
3. அமைப்புகளுக்கு சென்று DSN key ஐ பெறவும்
4. பின்வரும் சுற்றுச்சூழல் மாறிகளை கட்டமைக்கவும்:
   - `VITE_SENTRY_DSN`: உங்கள் Sentry திட்ட DSN
   - `VITE_SYSTEM_ENVIRONMENT`: உற்பத்திக்கு "production" என அமைக்கவும் (டிரேசிங் மாதிரி விகிதத்தை பாதிக்கிறது)
   - `VITE_VERSION`: Sentry இல் வெளியீட்டு கண்காணிப்பிற்கு பயன்படுத்தப்படுகிறது

குறிப்பு: Sentry விருப்பமானது ஆனால் உற்பத்தி வரிசைப்படுத்தல்களுக்கு பரிந்துரைக்கப்படுகிறது.

## மேம்பாடு

### பயன்பாட்டை உள்ளூரில் எவ்வாறு இயக்குவது?

**டெவலப்பர்களுக்கு மட்டும்** - இந்த பிரிவு Ministry Mapper க்கு பங்களிக்கும் டெவலப்பர்களுக்கானது.

1. **களஞ்சியத்தை குளோன் செய்யவும்:**

   ```bash
   git clone https://github.com/rimorin/ministry-mapper-v2.git
   cd ministry-mapper-v2
   ```

2. **சார்புகளை நிறுவுங்கள்:**

   ```bash
   npm install
   ```

3. தேவையான சுற்றுச்சூழல் மாறிகளுடன் **.env கோப்பை கட்டமைக்கவும்** (`.env.example` ஐ பாருங்கள்)

4. **மேம்பாட்டு சேவையகத்தை தொடங்குங்கள்:**
   ```bash
   npm start
   ```

பயன்பாடு `http://localhost:3000` இல் இயங்கும்

குறிப்பு: பயன்பாடு செயல்பட இயங்கும் PocketBase பின்-முனை தேவைப்படுகிறது. [Backend Setup Guide](backend-setup.md) ஐ பாருங்கள்.

### சோதனைகளை எவ்வாறு இயக்குவது?

```bash
npm test
```

சோதனைகள் Vitest மற்றும் Testing Library பயன்படுத்தி எழுதப்பட்டுள்ளன. சோதனை கோப்புகள் `.test.ts` நீட்டிப்புடன் மூல கோப்புகளுடன் அமைந்துள்ளன.

### குறியீட்டை எவ்வாறு வடிவமைப்பது?

**வடிவமைப்பை சரிபார்க்கவும்:**

```bash
npm run prettier
```

**வடிவமைப்பு திருத்தங்களை பயன்படுத்துங்கள்:**

```bash
npm run prettier:fix
```

திட்டம் குறியீடு வடிவமைப்பிற்கு Prettier பயன்படுத்துகிறது மற்றும் Husky வழியாக pre-commit hooks கட்டமைக்கப்பட்டுள்ளது.

### தொழில்நுட்ப அடுக்கு என்ன?

- **முன்-முனை கட்டமைப்பு**: TypeScript உடன் React 19
- **உருவாக்க கருவி**: Vite
- **UI நூலகம்**: React Bootstrap (Bootstrap 5)
- **வரைப்படங்கள்**: OpenStreetMap உடன் Leaflet
- **நிலை நிர்வாகம்**: React Context + தனிப்பயன் hooks
- **திசைப்பயணம்**: Wouter
- **ஸ்டைலிங்**: SCSS
- **பின்-முனை**: PocketBase (தனி களஞ்சியம்)
- **கண்காணிப்பு**: Sentry
- **சோதனை**: Vitest + Testing Library

விரிவான கட்டமைப்பு தகவலுக்கு CLAUDE.md ஐ பாருங்கள்.

## சிக்கல் நிவர்த்தி

### பயன்பாடு "Cannot connect to backend" காட்டுகிறது

**நேர சேவை பயனர்களுக்கு**: இந்த பிழையை நீங்கள் பார்த்தால், சேவை சிக்கல்களை எதிர்கொள்ளலாம். நிலை பக்கத்தை சரிபார்க்கவும் அல்லது ஆதரவை தொடர்புகொள்ளவும்.

**சுய-நேர்விடல் பயனர்களுக்கு**:

**சாத்தியமான காரணங்கள்:**

1. PocketBase பின்-முனை இயங்கவில்லை அல்லது அணுகக்கூடியதாக இல்லை
2. `VITE_POCKETBASE_URL` சுற்றுச்சூழல் மாறி தவறானது
3. CORS சிக்கல்கள் (பின்-முனை முன்-முனை டொமைனை அனுமதிக்கவில்லை)

**தீர்வுகள்:**

- பின்-முனை இயங்குகிறது மற்றும் அணுகக்கூடியதாக இருப்பதை சரிபார்க்கவும்
- உங்கள் சுற்றுச்சூழல் மாறிகளில் URL ஐ சரிபார்க்கவும்
- உங்கள் முன்-முனை டொமைனை அனுமதிக்க PocketBase இல் CORS அமைப்புகளை கட்டமைக்கவும்

### மீண்டும் உருவாக்கிய பிறகு மாற்றங்கள் பிரதிபலிக்கவில்லை

**சுய-நேர்விடலுக்கு மட்டும்**:

**சாத்தியமான காரணங்கள்:**

2. CDN தற்காலிக சேமிப்பு (பயன்படுத்துகிறீர்கள் என்றால்)
3. Build artifacts சரியாக வரிசைப்படுத்தப்படவில்லை

**தீர்வுகள்:**

- உலாவி தற்காலிக சேமிப்பை அழிக்கவும் அல்லது incognito பயன்முறையை பயன்படுத்தவும்
- பொருந்தும் பட்சத்தில் CDN தற்காலிக சேமிப்பை செல்லாது செய்யவும்
- உங்கள் நேர்விடல் சேவையில் build கோப்புறை முழுமையாக மாற்றப்பட்டது என்பதை சரிபார்க்கவும்
- பிழைகளுக்கு உலாவி console ஐ சரிபார்க்கவும்

### பயனர் அங்கீகாரம் வேலை செய்யவில்லை

**நேர சேவை பயனர்களுக்கு**: அங்கீகார சிக்கல்கள் இருந்தால் ஆதரவை தொடர்புகொள்ளவும்.

**சுய-நேர்விடல் பயனர்களுக்கு**:

**சாத்தியமான காரணங்கள்:**

1. PocketBase பின்-முனை பயனர் நிர்வாகம் கட்டமைக்கப்படவில்லை
2. நெட்வொர்க் இணைப்பு சிக்கல்கள்
3. தவறான PocketBase URL
4. CORS கட்டமைப்பு சிக்கல்கள்

**தீர்வுகள்:**

- PocketBase பின்-முனை சரியாக கட்டமைக்கப்பட்டுள்ளது என்பதை சரிபார்க்கவும்
- பிழை செய்திகளுக்கு உலாவி console ஐ சரிபார்க்கவும்
- பின்-முனை API endpoints ஐ நேரடியாக சோதிக்கவும்
- PocketBase இல் CORS அமைப்புகளை சரிபார்க்கவும்

### பயன்பாடு மெதுவாக அல்லது பதிலளிக்காமல் உள்ளது

**சாத்தியமான காரணங்கள்:**

4. உலாவி செயல்திறன் சிக்கல்கள்

**தீர்வுகள்:**

- சுய-நேர்விடலுக்கு: அருகிலுள்ள நேர்விடல் பகுதியை பயன்படுத்துங்கள்
- சுய-நேர்விடலுக்கு: தேவைப்பட்டால் சேவையக வளங்களை மேம்படுத்துங்கள்
- உலாவி தற்காலிக சேமிப்பு மற்றும் தரவை அழிக்கவும்
- செயல்திறன் எச்சரிக்கைகளுக்கு உலாவி console ஐ சரிபார்க்கவும்

6. தெளிவான விளக்கத்துடன் pull request சமர்பிக்கவும்

**பங்களிப்பு வழிகாட்டுதல்கள்:**

- TypeScript மற்றும் React சிறந்த நடைமுறைகளை பின்பற்றவும்
- குறியீட்டுத் தளத்தில் ஏற்கனவே உள்ள முறைகளை பயன்படுத்தவும்
- தேவைப்பட்டால் ஆவணங்களை புதுப்பிக்கவும்
- மாற்றங்களை கவனமாகவும் குறைந்தபட்சமாகவும் வைத்திருங்கள்
- சமர்பிப்பதற்கு முன்பு முழுமையாக சோதிக்கவும்

கட்டமைப்பு விவரங்கள் மற்றும் குறியீட்டு மரபுகளுக்கு CLAUDE.md ஐ பாருங்கள்.

### பயன்பாட்டை புதிய மொழியில் மொழிபெயர்க்கலாமா?

ஆம்! பயன்பாடு மொழிபெயர்ப்புகளுக்கு சர்வதேசமயமாக்கல் (i18n) பயன்படுத்துகிறது:

1. ஏற்கனவே உள்ள மொழிபெயர்ப்புகளுக்கு [முன்-முனை களஞ்சியத்தில்](https://github.com/rimorin/ministry-mapper-v2) `src/i18n/` கோப்பகத்தை சரிபார்க்கவும்
2. ஏற்கனவே உள்ள கட்டமைப்பை பின்பற்றி புதிய மொழி கோப்பை உருவாக்கவும்
3. உங்கள் மொழிபெயர்ப்புகளை சேர்க்கவும்
4. மொழி தேர்ந்தெடுப்பு கூறை புதுப்பிக்கவும்
5. pull request சமர்பிக்கவும்

### புதிய பதிப்பிற்கு எவ்வாறு மேம்படுத்துவது?

**நேர சேவை பயனர்களுக்கு**: புதுப்பிப்புகள் தானியங்கி. நீங்கள் எதுவும் செய்ய தேவையில்லை!

**சுய-நேர்விடல் பயனர்களுக்கு**:

**முன்-முனை:**

```bash
git pull origin main
npm install
npm run build
# உங்கள் நேர்விடல் சேவைக்கு புதிய build ஐ வரிசைப்படுத்துங்கள்
```

**பின்-முனை:**
பின்-முனை மேம்படுத்தல் வழிமுறைகளுக்கு [Ministry Mapper BE repository](https://github.com/rimorin/ministry-mapper-be) ஐ பார்க்கவும்.

**முக்கியம்:**

- முதலில் உங்கள் தரவை எப்போதும் காப்பு எடுத்துக்கொள்ளவும்
