# Ministry Mapper 用户指南

## 简介

欢迎使用 Ministry Mapper，这是一个现代化的 Web 应用程序，旨在帮助会众高效管理外勤服务区域。本指南将带您了解开始使用和充分利用该应用程序所需的一切。

**什么是 Ministry Mapper？**

Ministry Mapper 是一个数字化区域管理系统，取代了传统的纸质方法。它允许会众：

- 以数字方式组织和分配区域
- 实时跟踪外勤服务访问
- 协调多位传道员的活动
- 从任何有互联网的设备访问区域

**主要优势：**

- ✓ 环保 - 消除纸张浪费
- ✓ 通过云同步实时更新
- ✓ 适用于任何设备（桌面、平板、手机）
- ✓ 集成交互式地图便于导航
- ✓ 安全的基于角色的访问控制

1. 填写：姓名、电子邮件、密码、确认密码

### 登录 Ministry Mapper

![Google OAuth 登录](assets/screenshots/google_oauth_sign_in.png)

*图 6：用于更快更安全身份验证的 Google OAuth 登录选项*

一旦您的账户经过验证并获得会众访问权限：

**标准登录：**
1. 导航到您会众的 Ministry Mapper URL
2. 输入您的电子邮件地址和密码
3. 点击 **"登录"**

**Google OAuth 登录（更快）：**
1. 点击 **"使用 Google 登录"** 按钮
2. 选择您的 Google 账户
3. 自动登录（无需密码）

**第 3 步：完成 OTP 验证（如已启用）**

如果您的会众启用了一次性密码安全：

1. 输入凭据后，您将看到 OTP 验证屏幕
2. 检查您的电子邮件获取验证代码
3. 输入电子邮件中的 6 位数代码
4. 点击 **"验证"** 或 **"提交"**
5. 代码在 5-10 分钟后过期

**第 4 步：访问您的仪表板**

- **传道员**：您不会看到仪表板 - 使用发送给您的分配链接
- **只读、负责人、管理员**：您将看到适合您角色的仪表板

> **💡 提示**：在受信任的设备上保持登录状态以方便使用，但在共享计算机上始终注销。

### 了解用户角色

Ministry Mapper 使用四级访问控制系统。您的角色决定了您可以访问的功能和可以执行的操作。

#### 角色层级

```
管理员（完全访问）
    ↓
负责人（管理分配）
    ↓
只读（仅查看）
    ↓
传道员（仅链接访问）
```

---

#### 👤 传道员

**访问方式**：通过管理员或负责人发送的分配链接

**传道员可以做什么：**

- ✓ 通过共享链接访问区域
- ✓ 查看带有交互式地图的区域地图
- ✓ 访问后更新地址状态
  - 标记为：已完成、不在家、请勿打扰、无效
- ✓ 为地址添加访问备注
- ✓ 跟踪"不在家"尝试次数
- ✓ 查看地址详情和顺序

**传道员不能做什么：**

- ✗ 无仪表板访问权限
- ✗ 无法查看所有会众区域
- ✗ 无法创建或管理区域
- ✗ 无法访问分配链接在设定时间后过期（默认：24 小时，可由会众管理员配置）

**最适合**：进行外勤服务的普通传道员

---

#### 👓 只读

**访问方式**：具有只读权限的仪表板登录

**只读用户可以做什么：**

- ✓ 查看会众中的所有区域
- ✓ 查看完整的地址信息
- ✓ 查看区域进度和统计数据
- ✓ 访问区域地图
- ✓ 查看会众消息

**只读用户不能做什么：**

- ✗ 无法修改任何区域或地址数据
- ✗ 无法创建或删除区域
- ✗ 无法管理分配
- ✗ 无法更改会众设置

**最适合**：需要可见性但无编辑能力的监督者

---

#### 🎯 负责人

**访问方式**：具有负责人权限的仪表板登录

**负责人能力包括：**

- ✓ **只读可以做的所有事情**，加上：
- ✓ 创建和管理区域分配
- ✓ 为传道员生成分配链接
- ✓ 查看所有分配历史
- ✓ 访问和发布会众消息
- ✓ 管理会众选项（住户状态类型）
- ✓ 查看区域完成状态

**负责人不能做什么：**

- ✗ 无法创建或删除区域
- ✗ 无法编辑地址详情（地址、单位、楼层）
- ✗ 无法管理用户角色或权限
- ✗ 无法修改核心会众设置

**最适合**：外勤服务协调员和小组监督

---

#### 👑 管理员（区域仆人）

**访问方式**：具有完全管理权限的仪表板登录

**管理员拥有完全控制权：**

- ✓ **负责人可以做的所有事情**，加上：
- ✓ 创建、编辑和删除区域
- ✓ 添加、修改和删除地址
- ✓ 管理建筑物和单位
- ✓ 配置会众设置
  - 设置最大"不在家"尝试次数
  - 配置链接过期时间
  - 设置住户类型选项
- ✓ 邀请和管理用户
- ✓ 分配用户角色和权限
- ✓ 重置区域和地址
- ✓ 管理地理定位坐标

**最适合**：区域仆人和管理会众区域系统的人员

---

> **💡 注意**：如果您需要更改角色或不确定当前角色，请联系您的会众管理员。

### 使用地址

#### 了解地址信息

![地址状态详情](assets/screenshots/address_status_details.png)

*图 9：显示所有地址详情和当前状态的地址卡*

Ministry Mapper 中的每个地址或单位都显示全面的信息：

**基本信息：**

- **地址/单位号**：位置标识符（例如，"#05-123"）
- **楼层**：建筑楼层（对于多层物业）
- **类型**：基于会众选项的住户分类
  - 示例：中文、英文、泰米尔语、西班牙语
  - 如果配置，可以选择多种类型
- **顺序**：访问顺序号

**状态信息：**

- **状态**：当前访问状态（见下面的状态类型）
- **不在家次数**：未应答访问尝试次数
- **请勿打扰日期**：标记 DNC 的时间（如适用）

**活动跟踪：**

- **备注**：关于访问或住户的重要信息
- **最后更新**：最近更新的日期和时间
- **更新者**：进行最后更改的用户名

---

#### 地址状态类型

Ministry Mapper 使用五种标准状态类型：

| 状态          | 颜色         | 描述            | 用途                                      |
| --------------- | ------------- | ---------------------- | ------------------------------------------ |
| **未完成**    | 白色/默认 | 尚未访问        | 所有地址的初始状态           |
| **已完成**        | 绿色         | 成功联系 | 住户在家并已联系         |
| **不在家**    | 黄色/橙色 | 无人应答        | 跟踪最多尝试次数（可配置）       |
| **请勿打扰** | 红色           | 要求不访问    | 住户要求不再联系   |
| **无效**     | 灰色          | 无法访问           | 地址不存在或无法访问 |

> **💡 提示**：一旦"不在家"达到最大尝试次数（由管理员设置），该地址在进度计算中自动标记为已完成。

#### 更新地址状态

![地址不在家状态](assets/screenshots/address_not_home_status.png)

*图 10：显示所有字段和选项的更新状态模态框*

**分步流程：**

**1. 找到地址**

- 滚动浏览地址列表
- 或点击地图上的标记
- 多层建筑的地址按楼层分组

**2. 打开更新模态框**

- 点击地址/单位卡
- 更新模态框将打开，显示当前信息

**3. 更新状态**（必需）

选择适当的状态：

**📗 已完成** - 成功联系时

- 当有人应答时选择此项
- 进行了对话或放置了文献
- "不在家"计数递增重置

**🏠 不在家** - 无人应答时

- 自动递增"不在家"计数
- 系统跟踪尝试次数
- 达到最大尝试次数后视为完成

**🚫 请勿打扰** - 当要求不访问时

![地址请勿打扰状态](assets/screenshots/address_do_not_call_status.png)

*图 11：带有日期选择的请勿打扰状态更新*

- 选择此状态
- 可选择设置 DNC 日期（默认为今天）
- 如果适当，添加解释原因的备注
- **重要**：始终尊重住户的意愿

**❌ 无效** - 当地址无法访问时

- 地址不存在
- 正在施工
- 永久关闭
- 无法访问

**4. 更新住户类型**（如可用）

如果您的会众配置了住户类型：

- 选择单个或多个类型
- 示例：语言偏好、特殊情况
- 如需要清除现有类型

**5. 添加或更新备注**（可选但推荐）

**最佳实践：**

- ✓ 关注物业详情，而非个人
- ✓ 记录访问说明和时间安排
- ✓ 简洁且尊重
- ✗ 永远不要包含关于住户的个人信息
- ✗ 永远不要包含敏感信息

**好的示例：**

- "门禁物业 - 先打电话给警卫室"
- "最佳时间：周末下午 2 点后"
- "侧门可通过车道进入"

**不好的示例：**

- 居民的个人详情
- 医疗或财务信息
- 个人的姓名或描述

**6. 调整不在家次数**（如需要）

系统自动跟踪不在家尝试，但如有必要，您可以手动调整。

**7. 设置请勿打扰日期**（仅限 DNC 状态）

标记为请勿打扰时：

- 系统默认为今天的日期
- 根据特定 DNC 请求进行调整
- 日期有助于跟踪何时可能重新访问

**8. 更新地理定位**（仅限管理员）

对于单层区域：

- 点击"更新地理定位"按钮
- 使用地图设置精确位置
- 有助于导航和地图准确性

**9. 保存更改**

- 审查所有更新
- 点击 **"保存"** 按钮
- 更改立即同步给所有用户
- 成功消息确认更新

**10. 删除物业**（仅限管理员 - 单层）

对于需要删除的私人物业：

- 点击底部的"删除物业"按钮
- 确认删除
- **警告**：此操作无法撤销

### 区域分配（负责人和管理员）

Ministry Mapper 使用基于链接的分配系统。负责人和管理员创建可共享的链接，传道员使用这些链接访问区域。

#### 为什么使用基于链接的分配？

- ✓ **简单分发**：通过电子邮件、消息或短信发送链接
- ✓ **自动过期**：链接在设定时间后过期（默认：24 小时）
- ✓ **无需账户**：传道员直接通过链接工作
- ✓ **安全性**：过期的链接无法访问

![会众设置](assets/screenshots/congregation_settings.png)

#### 访问设置

1. 点击 **"设置"** 按钮或 ⚙️ 图标
2. 查看会众配置面板

#### 关键设置

**1. 最大"不在家"尝试次数**

- 默认：1
- 范围：1-4 次
- 达到后，地址在进度计算中被视为完成
- 影响区域何时显示为完成

**示例：** 如果设置为 3：

- 第一次"不在家"：计数 = 1
- 第二次"不在家"：计数 = 2
- 第三次"不在家"：计数 = 3，标记为完成

**2. 分配链接过期（小时）**

- 默认：24 小时
- 范围：1-168 小时（1 周）
- 分配链接保持活跃的时间
- 适用于新创建的链接

**3. 会众来源/位置**

- 设置您会众的位置
- 用于地图居中和路线指引
- 可以是城市名称或坐标
- 有助于路线规划

**4. OTP（一次性密码）**

- 启用/禁用登录的电子邮件 OTP
- 添加额外的安全层
- 用户登录时通过电子邮件接收代码
- 建议用于敏感的会众数据

![住户选项设置](assets/screenshots/household_options_settings.png)

*图 20：显示住户类型配置的会众选项管理*

为您的区域配置自定义住户分类类型。

**什么是会众选项？**

- 用于分类住户的自定义类别
- 示例：语言组（中文、英文、泰米尔语）
- 可以代表您会众使用的任何分类系统
- 如果配置，可以为单个住户分配多种类型

**管理选项：**

1. **查看选项**

   - 在设置中，找到"会众选项"部分
   - 查看所有配置类型的列表

2. **添加新选项**

   - 点击"添加选项"或"+"
   - 填写：
     - **代码**：短标识符（例如，"CHI"、"ENG"）
     - **描述**：全名（例如，"中文"、"英文"）
     - **是否可统计**：如果应计入区域进度则勾选
     - **是否默认**：如果应为默认选择则勾选
     - **顺序**：显示顺序号
   - 点击"保存"

3. **编辑选项**

   - 点击现有选项
   - 修改字段
   - 保存更改

   - 确认删除

**选项标志：**

**是否可统计：**
- 控制具有此类型的地址是否计入区域进度
- **勾选**：包含在进度计算中（例如，中文、英文、泰米尔语）
- **未勾选**：从进度中排除（例如，商业、施工中）

**是否默认：**
- 创建新地址时自动选择此类型
- 只有一个选项应该是默认的
- 用于您最常见的住户类型

**顺序：**
- 控制下拉菜单中的显示顺序
- 较低的数字先出现（1, 2, 3...）
- **通过设置中的拖放管理** - 只需拖动选项重新排序
- 影响地址住户下拉菜单中的顺序
- 提示：从最常见到最不常见排序

**多选配置：**

- 如果住户可以有多种类型则启用
- 示例：住户同时说中文和英文
- 禁用时，每个住户只能有一种类型

**访问：** 地址菜单 → "更改顺序"

**工作原理：**
1. 每张卡片显示一个地址及其当前序列号
2. 拖放重新排序 - 数字自动更新
3. 点击 **"保存"** 应用或 **"取消"** 放弃

**最佳实践：**
- 通过将附近地址分组来减少回溯
- 在多层建筑中将楼层分组在一起
- 创建从一端到另一端的逻辑流程
- 排序后查看地图视图

**4. 重命名**
   - 更改地址/单位的名称或标识符
   - 更新建筑名称或单位号
   - 使数据与现实世界的变化保持同步
   - 用于纠正初始录入错误

**5. 添加单位号**
   - 向现有地址添加新单位号
   - 扩展多层建筑的额外单位
   - 当新公寓添加到建筑时有用
   - 保持建筑结构组织

   - 向上扩展建筑的额外楼层
   - 用于已扩建或最初低估的建筑

   - 仅删除"已完成"和"不在家"状态
   - 不删除"请勿打扰"或"无效"状态
   - 保留备注和其他地址信息
   - 当重新开始先前完成的地址的工作时有用
   - 不影响区域中的其他地址

**9. 删除**
   - 从区域中永久删除地址、单位或楼层
   - 无法撤销 - 谨慎使用
   - 有助于删除重复条目或不存在的地址
   - 删除前仔细确认

**常见使用场景：**
- 纠正错误：重命名和更改位置

| **密码错误** | "凭据无效"错误 | 验证电子邮件 • 检查大写锁定 • 使用"忘记密码" → 通过电子邮件重置 |
| **账户未验证** | "电子邮件未验证"消息 | 检查收件箱/垃圾邮件中的验证电子邮件 • 点击链接 • 如果过期请求新的 |
| **无会众访问** | 登录但未显示区域 | 联系管理员邀请您并分配角色 • 授予访问权限后注销/登录 |
| **OTP 问题** | 未收到/无效的 OTP 代码 | 检查垃圾邮件 • 代码在 5-10 分钟后过期 • 请求新代码 • 验证电子邮件地址 |

## 隐私和安全

### 保护信息

Ministry Mapper 处理敏感的地址和个人信息。请遵守这些准则：

**账户安全：**

- ✓ **永远不要分享登录凭据**
- ✓ **使用强大、唯一的密码**（至少 6 个字符，包含数字和大写字母）
- ✓ **在共享设备上总是注销**
- ✓ **如果可用启用 OTP** 以增加安全性
- ✓ **立即报告可疑活动**

**数据隐私：**

- ✓ **只记录必要信息**
- ✓ **在所有描述中保持尊重和事实**
- ✓ **不要包含敏感个人数据**（医疗、财务等）
- ✓ **遵循住户的隐私请求**
- ✓ **遵守隐私法**（GDPR、CCPA、当地法规）

**法律合规：**

- ⚠️ **GDPR（欧洲）**：个人数据保护要求
- ⚠️ **CCPA（加利福尼亚）**：消费者隐私权
- ⚠️ **LGPD（巴西）**：数据保护法规
- ⚠️ **当地法律**：检查您所在地区的要求

### 存储的信息

**用户数据：**

- 账户详情（姓名、电子邮件、验证状态）
- 会众角色分配
- 创建/访问的分配链接
- 活动时间戳

**区域数据：**

- 地址和单位信息
- 状态更新和历史
- 备注和访问信息
- 地理定位坐标
- 进度跟踪

**系统数据：**

- 登录会话
- 实时订阅
- 消息历史
- 配置设置

### 数据存储和安全

| 组件 | 详情 |
|-----------|---------|
| **后端** | 由管理员管理的 PocketBase 数据库 • 由会众决定的托管位置 • 基于角色的访问控制 |
| **实时同步** | PocketBase 订阅 • 加密的 HTTPS 连接 • 自动重连 • 会话管理 |
| **客户端** | Service worker 仅缓存静态资源 • 不在本地存储敏感数据 • 新版本自动更新 |
| **管理员职责** | 实施备份程序 • 进行安全审计 • 保持后端更新 • 监控访问日志 |
