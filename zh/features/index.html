    功能概述¶

    简介¶

    Ministry Mapper 是一个全面的数字化区域管理平台，专为宗教团体设计。本页面详细介绍了所有功能和特性。

    🗺️ 区域管理¶

    组织地理区域¶

    创建和管理区域： - 每个会众可创建无限数量的区域 - 分配唯一的代码和描述 - 通过拖放排序按优先级组织区域 - 自动跟踪完成百分比 - 重置区域状态以重新开始跟踪 - 轻松在区域之间移动地图

    区域统计： - 汇总完成情况跟踪 - 可视化进度指示器 - 实时更新统计数据 - 按区域细分 - 历史记录跟踪

    使用场景： 将您会众的服务区域划分为可管理的区域，以便有组织地进行外勤服务工作。

    🏢 地图和地址管理¶

    互动位置跟踪¶

    地图类型：

    1. 公共地址（多层建筑） - 支持多层建筑 - 动态添加/删除楼层 - 每层包含多个单位 - 自动将地址代码应用于所有楼层 - 按楼层组织

    2. 私人地址（单层住宅） - 独立房屋或独立物业 - 无需楼层细分的简单结构 - 快速地址录入

    地图功能： - 附加 GPS 坐标以实现精确定位 - 添加详细描述 - 用于有序访问的序列号 - 与 Leaflet/OpenStreetMap 的可视化地图集成 - 获取任何地址的路线指引 - 地理定位支持

    地址操作： - 添加/删除地址代码 - 重新排序地址以优化路线 - 批量操作提高效率 - 跨楼层复制地址 - 重命名和重新组织

    随时添加地址 (v1.33+) - 传道员可在传道过程中直接添加缺失的地址，无需管理员介入 - 在地址列表末尾会出现一个 "+" 卡片，作为快速添加入口 - 适合仍在完善地区记录的会众

    📊 单位/住户跟踪¶

    全面的访问状态管理¶

    状态类型：

    状态 描述 对进度的影响
    未完成 初始状态，尚未访问 计入未完成
    已完成 成功完成 计入已完成
    不在家 无人在家，可重试 达到最大尝试次数后计入
    请勿打扰 (DNC) 住户要求不访问 不计入统计
    无效 单位不存在 不计入统计

    不在家跟踪： - 每个会众可配置最大尝试次数 - 自动计数器递增 - 达到最大尝试次数后视为"已完成" - 防止无限期重访

    完成逻辑： 

    完成条件：
  - 状态 = 已完成，或
  - 状态 = 不在家 且 尝试次数 >= 最大尝试次数

进度 % = 已完成 / 可统计单位总数 × 100

    住户选项： - 可自定义地址分类 - 可统计与不可统计类型 - 新地址的默认选项 - 每个地址可有多个选项 - 影响进度计算

    访问备注： - 为每个地址添加详细备注 - 自动时间戳跟踪 - 跟踪更新者 - 备注更改时发送电子邮件通知 - 历史备注查看

    🔗 分配系统（传道员链接）¶

    安全、限时访问¶

    链接功能： - 限时： 可配置过期时间（默认 24 小时） - 基于令牌： 安全的 20 字符访问令牌 - 无需账户： 传道员仅通过链接访问 - 两种类型： - 普通分配用于常规区域工作 - 个人分配用于个人传道

    智能分配算法：

    请求新分配时，系统：

    1. 主要因素： 选择活跃分配最少的地图
    2. 防止单个地图过载

    3. 平衡工作量

    4. 次要因素： 优先选择地理位置较近的地图

    5. 使用 GPS 坐标
    6. 50 米阈值

    7. 减少出行时间

    8. 第三因素： 优先选择完成进度较低的地图

    9. 确保区域完成均衡
    10. 防止某些区域被忽视

    分配工作流程： 

    1. 负责人生成分配链接
2. 链接分享给传道员（二维码、邮件、短信）
3. 传道员访问 /map/:assignmentId
4. 系统验证令牌和过期时间
5. 传道员实时更新地址状态
6. 链接自动过期
7. 管理员收到实时更新

    分配管理： - 查看每个地图的活跃分配 - 查看谁拥有哪个区域 - 跟踪分配历史 - 手动删除分配 - 自动清理过期链接（每 5 分钟）

    👥 用户管理和访问控制¶

    基于角色的权限系统¶

    四种用户角色：

    1. 传道员（最受限制）¶

    访问方式： 仅限限时分配链接

    权限： - ✅ 仅查看分配的地图 - ✅ 更新地址/单位状态 - ✅ 添加访问备注 - ✅ 发送反馈消息 - ❌ 无法查看其他区域 - ❌ 无法创建区域 - ❌ 无法管理用户

    使用场景： 进行区域外勤服务的传道员

    2. 只读用户¶

    访问方式： 具有只读角色的用户账户

    权限： - ✅ 查看所有会众数据 - ✅ 查看区域和地图 - ✅ 查看地址信息 - ✅ 查看消息 - ❌ 无法修改任何内容 - ❌ 无法创建分配 - ❌ 无法管理用户

    使用场景： 观察员、审计员、服务委员会成员

    3. 负责人（中级）¶

    访问方式： 具有负责人角色的用户账户

    权限： - ✅ 所有只读权限 - ✅ 创建和管理区域 - ✅ 创建和管理地图 - ✅ 添加/更新地址 - ✅ 生成传道员分配链接 - ✅ 查看所有会众数据 - ✅ 管理住户选项 - ✅ 重置区域 - ❌ 无法管理用户角色 - ❌ 无法配置会众设置 - ❌ 无法删除用户

    使用场景： 外勤服务监督、区域协调员

    4. 管理员（完全访问）¶

    访问方式： 具有管理员角色的用户账户

    权限： - ✅ 所有负责人权限 - ✅ 创建和管理用户 - ✅ 分配用户角色 - ✅ 配置会众设置 - ✅ 管理住户选项 - ✅ 删除区域和地图 - ✅ 删除用户账户 - ✅ 查看所有分配和消息 - ✅ 配置最大不在家尝试次数 - ✅ 设置链接过期时间 - ✅ 访问系统日志

    使用场景： 区域仆人、系统管理员

    用户账户管理¶

    功能： - 基于电子邮件的身份验证 - 需要电子邮件验证 - 密码重置功能 - OAuth2 Google 身份验证 - 多因素身份验证 (MFA) 支持 - 一次性密码 (OTP) 支持 - 账户禁用/启用 - 最后登录跟踪 - 多会众支持（每个会众不同角色）

    🛠️ 会众设置¶

    可自定义配置¶

    核心设置：

    1. 最大不在家尝试次数 - 设置在视为完成之前的"不在家"访问次数 - 典型值：2-3 次 - 影响进度计算 - 会众特定

    2. 分配链接过期时间 - 配置链接保持有效的时长 - 默认：24 小时 - 每个会众可自定义 - 自动清理过期链接

    3. 住户选项 - 创建自定义地址分类 - 示例："住宅"、"商业"、"高层"、"语言：西班牙语" - 将选项标记为"可统计"（影响完成百分比） - 设置新地址的默认选项 - 拖放重新排序 - 查看哪些地址具有每个选项

    4. 区域设置 - 时区配置 - 国家/地区选择 - 影响日期计算和报告

    💬 通信和消息¶

    应用内通信系统¶

    消息类型：

    1. 传道员消息 - 从传道员到负责人/管理员 - 附加到特定地图 - 特定区域反馈 - 实时发送

    2. 负责人消息 - 管理通信 - 特定地图说明 - 协调消息

    3. 管理员消息 - 可置顶以提高可见性 - 已读状态跟踪 - 重要公告 - 向传道员广播

    4. 置顶消息 - 保持可见直到取消置顶 - 每用户已读跟踪 - 自动发送电子邮件给相关用户 - 优先显示

    电子邮件通知：

    自动发送电子邮件摘要： - 每 30 分钟： 向管理员发送未读消息 - 每 30 分钟： 向传道员发送置顶的管理员消息 - 每 1 小时： 向管理员发送备注更新 - 每月： 带有 Excel 附件的会众报告

    电子邮件模板： - 专业的 HTML 模板 - 移动端响应式设计 - 包含相关链接 - 可自定义内容

    🔄 实时协作¶

    实时数据同步¶

    实时功能：

    SSE 订阅： - 通过服务器发送事件 (SSE) 实时更新 - 断开连接时自动重连 - 低延迟（通常 <100ms）

    并发编辑： - 多用户可同时工作 - 无冲突或数据丢失 - 后写入优先策略 - 基于时间戳的解决方案

    自动刷新： - 数据更改时立即更新 - 无需手动刷新页面 - 更新的视觉指示器 - 乐观 UI 更新

    可见性检测： - 当标签页变为可见时自动刷新 - 当标签页隐藏时节省带宽 - 防止数据过时

    协作场景： - 管理员在负责人查看列表时创建区域 → 立即显示 - 传道员标记单位完成 → 管理员实时看到进度更新 - 多个负责人管理不同区域 → 无冲突 - 工作进行时实时统计更新

    📱 渐进式 Web 应用 (PWA)¶

    原生应用体验¶

    PWA 功能：

    可安装： - 添加到主屏幕（移动端） - 作为桌面应用安装 - 出现在应用启动器中 - 全屏模式 - 自定义应用图标

    桌面安装： 

    访问 URL → 浏览器菜单 → "安装应用"

    移动端安装： 

    访问 URL → 分享 → "添加到主屏幕"

    离线支持： - 通过 service worker 缓存静态资源 - 离线时继续查看缓存数据 - 连接恢复时自动同步 - 后台更新

    性能优势： - 更快的加载时间（缓存资源） - 减少数据使用 - 在慢速连接上正常工作 - 流畅动画

    原生功能： - 推送通知（计划中） - 后台同步 - 原生文件处理 - 分享 API 集成

    🌍 国际化¶

    多语言支持¶

    支持的语言： 1. 英语 (en) - 主要语言 2. 西班牙语 (es) - Español 3. 印度尼西亚语 (id) - Bahasa Indonesia 4. 日语 (ja) - 日本語 5. 韩语 (ko) - 한국어 6. 马来语 (ms) - Bahasa Melayu 7. 中文 (zh) - 中文 8. 泰米尔语 (ta) - தமிழ்

    i18n 功能： - 导航中的语言选择器 - 持久化偏好设置（localStorage） - 浏览器语言检测 - 完整的 UI 翻译 - 动态切换（无需重新加载） - 易于添加新语言

    本地化版本说明： - 应用内更新日志以用户当前选择的语言显示 - 支持全部 8 种语言 - 检测到新版本时自动弹出提示，让传道员无需离开应用即可了解最新内容

    添加新语言： 1. 在 src/i18n/locales/[lang].json 中创建翻译文件 2. 从 en.json 复制结构 3. 翻译所有字符串 4. 在 src/i18n/index.ts 中注册

    🌙 深色模式¶

    系统感知主题¶

    主题选项： - 浅色模式： 传统浅色主题 - 深色模式： 护眼深色主题 - 系统： 跟随操作系统偏好

    优势： - 减少眼睛疲劳 - 更好的电池续航（OLED 屏幕） - 无障碍合规 - 现代用户体验

    实现： - CSS 自定义属性 - 平滑过渡 - 持久化偏好设置 - 无主题闪烁

    📈 报告和分析¶

    全面的统计数据¶

    月度会众报告： - 计划： 每月 1 日午夜 - 格式： Excel 工作簿 (.xlsx) - 发送： 发送电子邮件给所有管理员

    报告内容：

    1. 会众概述 - 会众名称和详情 - 报告期间 - 总体统计

    2. 区域细分 - 每个区域的完成百分比 - 每个区域的地图数量 - 地址分布 - 时间进度

    3. 地址统计 - 按类型的总地址数 - 状态分布（已完成、未完成等） - 可统计与不可统计细分 - DNC 和无效计数

    4. 传道员分配 - 活跃分配数量 - 分配历史 - 区域覆盖

    5. 进度跟踪 - 月度对比 - 趋势和洞察 - 需要关注的区域

    Excel 功能： - 专业格式 - 图表和图形 - 可用于数据透视表的数据 - 打印友好布局

    实时仪表板： - 管理面板上的实时统计 - 区域完成跟踪 - 地址状态分布 - 活跃分配监控 - 最近活动信息流

    AI报告摘要： - 月度会众报告可包含 AI 生成的关键趋势摘要，来源于地区备注和消息 - 由 OpenAI gpt-5.4-mini 提供支持 - 由 enable-report-ai-summary 功能标志控制——按部署选择是否启用 - 需要在部署环境中配置 OPENAI_API_KEY

    🔐 安全功能¶

    企业级安全¶

    身份验证方式：

    1. 电子邮件/密码 - Bcrypt 密码哈希（成本：10） - 最少 6 个字符 - 需要电子邮件验证 - 通过电子邮件重置密码

    2. OAuth2 (Google) - PKCE 流程确保安全 - 自动创建用户 - 无需密码管理 - 安全的令牌处理

    3. 一次性密码 (OTP) - 可选的 4 位数代码 - 180 秒有效期 - 额外安全层 - 电子邮件发送

    4. 多因素身份验证 (MFA) - TOTP 支持 - 为管理员增强安全性 - 1800 秒会话 - 备用代码

    访问控制：

    基于角色的访问控制 (RBAC)： - 细粒度权限 - 会众级别隔离 - 多租户架构 - 默认安全

    基于链接的访问： - 临时匿名访问 - 基于令牌的身份验证 - 自动过期 - 可撤销访问

    安全最佳实践： - HTTPS 强制 - CORS 配置 - 速率限制 - SQL 注入防护（参数化查询） - XSS 防护 - CSRF 防护 - 输入验证 - 输出清理

    审计跟踪： - 用户操作日志 - 更改跟踪 - 谁在何时更新了什么 - IP 地址记录 - 身份验证尝试

    用户生命周期管理：

    自动处理不活跃和未配置用户账户，符合 NIST AC-2 和 AC-2(3) 安全控制规范：

    • 未配置用户： 从未完成设置的账户在自动禁用并最终删除之前会收到多阶段警告邮件
    • 不活跃用户： 在可配置时间段内无登录活动的账户会收到警告，然后被禁用
    • 每日运行后台任务评估并执行生命周期策略，无需人工干预
    • 保持会众用户列表准确，减少休眠账户带来的攻击面

    错误跟踪： - Sentry 集成 - 实时错误监控 - 调试堆栈跟踪 - 版本跟踪 - 性能监控

    📧 电子邮件系统¶

    自动电子邮件通知¶

    电子邮件服务： MailerSend API v1

    电子邮件类型：

    1. 消息通知 - 计划：每 30 分钟 - 收件人：管理员 - 内容：未读的传道员消息 - 模板：专业 HTML

    2. 管理员说明 - 计划：每 30 分钟 - 收件人：传道员（非管理员用户） - 内容：置顶的管理员消息 - 模板：说明格式

    3. 备注更新 - 计划：每 1 小时 - 收件人：管理员 - 内容：地址备注更改 - 模板：备注摘要格式

    4. 月度报告 - 计划：每月 1 日 - 收件人：管理员 - 内容：带有可选 AI 生成区域摘要的统计报告 - 附件：Excel 工作簿 - AI 摘要：当通过 enable-report-ai-summary 功能标志启用并配置了 OPENAI_API_KEY 时，每个区域表都包含 AI 生成的关键趋势和值得注意观察的概述

    5. 系统电子邮件 - 电子邮件验证 - 密码重置 - 账户警报 - 身份验证通知

    模板功能： - 移动端响应式 HTML - 专业设计 - 内联 CSS 以实现兼容性 - 纯文本备用 - 退订链接 - 跟踪像素（可选）

    🔄 后台任务¶

    自动化任务处理¶

    任务调度器： 基于 Cron，带有 LaunchDarkly 功能标志

    计划任务：

    任务 计划 目的
    分配清理 每 5 分钟 删除过期分配
    区域汇总 每 10 分钟 更新进度统计
    消息处理 每 30 分钟 发送消息通知
    说明 每 30 分钟 发送管理员消息
    备注更新 每 1 小时 通知备注更改
    月度报告 每月 1 日 生成 Excel 报告（带可选 AI 摘要）
    未配置用户 每日 01:00 UTC 强制执行用户生命周期（警告 → 禁用 → 删除）
    不活跃用户 每日 01:30 UTC 警告并禁用不活跃账户
    新增地址通知 每日 03:00 SGT 向管理员发送应用内新增地址的每日摘要邮件

    功能标志控制： - 无需部署即可启用/禁用任务 - 逐步推出能力 - A/B 测试 - 紧急关闭开关 - 每个会众控制
