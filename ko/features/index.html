    기능 개요¶

    소개¶

    Ministry Mapper는 종교 회중을 위해 설계된 종합 디지털 구역 관리 플랫폼입니다. 이 페이지에서는 모든 기능과 역량에 대한 자세한 개요를 제공합니다.

    구역 관리¶

    지리적 영역 구성¶

    구역 생성 및 관리: - 회중당 무제한 구역 생성 - 고유 코드 및 설명 할당 - 드래그 앤 드롭 순서 지정으로 우선순위별 구역 정리 - 완료 비율 자동 추적 - 추적 재시작을 위한 구역 상태 초기화 - 구역 간 지도 쉽게 이동

    구역 통계: - 종합 완료 추적 - 시각적 진행 표시기 - 통계 실시간 업데이트 - 구역별 세부 분석 - 이력 추적

    사용 사례: 회중의 봉사 구역을 관리 가능한 구역으로 나누어 체계적인 야외 봉사 활동을 수행합니다.

    지도 및 주소 관리¶

    대화형 위치 추적¶

    지도 유형:

    1. 공공 주소 (다층 건물) - 다층 건물 지원 - 층 동적 추가/제거 - 각 층에 여러 세대 포함 - 모든 층에 자동 주소 코드 적용 - 층별 구성

    2. 개인 주소 (단층 주택) - 개별 주택 또는 독립 건물 - 층 구분 없는 단순 구조 - 빠른 주소 입력

    지도 기능: - 정확한 위치를 위한 GPS 좌표 첨부 - 상세 설명 추가 - 체계적인 방문을 위한 순서 번호 - Leaflet/OpenStreetMap을 통한 시각적 지도 통합 - 모든 주소로 길 안내 - 위치 정보 지원

    주소 작업: - 주소 코드 추가/제거 - 최적 경로를 위한 주소 재정렬 - 효율성을 위한 일괄 작업 - 층 간 주소 복사 - 이름 변경 및 재구성

    즉석 주소 추가 (v1.33+) - 전도인이 지도 작업 중 누락된 주소를 직접 추가할 수 있습니다 — 관리자 개입 불필요 - 주소 목록 끝에 "+" 카드가 빠른 추가 입력 지점으로 표시됩니다 - 구역 기록을 아직 구축 중인 회중에 이상적입니다

    세대/가구 추적¶

    종합 방문 상태 관리¶

    상태 유형:

    상태 설명 진행에 미치는 영향
    미완료 초기 상태, 아직 방문하지 않음 미완료에 포함
    완료 성공적으로 완료됨 완료로 계산
    부재 집에 아무도 없음, 재방문 가능 최대 시도 후 계산
    방문 금지 (DNC) 가구에서 방문 거부 요청 계산에서 제외
    유효하지 않음 세대가 존재하지 않음 계산에서 제외

    부재 추적: - 회중별 최대 시도 횟수 설정 가능 - 자동 카운터 증가 - 최대 시도 후 "완료"로 처리 - 무한 재방문 방지

    완료 논리: 

    완료 조건:
  - 상태 = 완료, 또는
  - 상태 = 부재 AND 시도 횟수 >= 최대_시도

진행률 % = 완료 / 계산 가능한 총 세대 × 100

    가구 옵션: - 사용자 정의 가능한 주소 분류 - 계산 가능 vs 계산 불가 유형 - 새 주소의 기본 옵션 - 주소당 여러 옵션 - 진행률 계산에 영향

    방문 메모: - 주소별 상세 메모 추가 - 자동 타임스탬프 추적 - 업데이트한 사람 추적 - 메모 변경 시 이메일 알림 - 이력 메모 조회

    배정 시스템 (전도인 링크)¶

    안전한 시간 제한 접근¶

    링크 기능: - 시간 제한: 만료 설정 가능 (기본 24시간) - 토큰 기반: 안전한 20자 접근 토큰 - 계정 불필요: 전도인은 링크만으로 접근 - 두 가지 유형: - 정규 구역 봉사를 위한 일반 배정 - 개인 봉사를 위한 개인 배정

    지능형 배정 알고리즘:

    새 배정 요청 시 시스템은:

    1. 주요 요소: 활성 배정이 가장 적은 지도 선택
    2. 단일 지도 과부하 방지

    3. 작업량 균형 조정

    4. 보조 요소: 지리적으로 가까운 지도 선호

    5. GPS 좌표 사용
    6. 50미터 임계값

    7. 이동 시간 단축

    8. 3차 요소: 낮은 완료 진행률 우선

    9. 균형 잡힌 구역 완료 보장
    10. 일부 지역 방치 방지

    배정 워크플로우: 

    1. 인도자가 배정 링크 생성
2. 전도인에게 링크 공유 (QR, 이메일, 문자)
3. 전도인이 /map/:assignmentId 접근
4. 시스템이 토큰 및 만료 확인
5. 전도인이 실시간으로 주소 상태 업데이트
6. 링크 자동 만료
7. 관리자가 실시간 업데이트 수신

    배정 관리: - 지도별 활성 배정 조회 - 누가 어떤 구역을 가지고 있는지 확인 - 배정 이력 추적 - 수동 배정 삭제 - 만료된 링크 자동 정리 (5분마다)

    사용자 관리 및 접근 제어¶

    역할 기반 권한 시스템¶

    네 가지 사용자 역할:

    1. 전도인 (가장 제한적)¶

    접근 방법: 시간 제한 배정 링크만

    권한: - ✅ 배정된 지도만 조회 - ✅ 주소/세대 상태 업데이트 - ✅ 방문 메모 추가 - ✅ 피드백 메시지 전송 - ❌ 다른 구역 볼 수 없음 - ❌ 구역 생성 불가 - ❌ 사용자 관리 불가

    사용 사례: 구역 봉사를 하는 야외 봉사 전도인

    2. 읽기 전용¶

    접근 방법: 읽기 전용 역할의 사용자 계정

    권한: - ✅ 모든 회중 데이터 조회 - ✅ 구역 및 지도 조회 - ✅ 주소 정보 조회 - ✅ 메시지 조회 - ❌ 수정 불가 - ❌ 배정 생성 불가 - ❌ 사용자 관리 불가

    사용 사례: 옵저버, 감사자, 봉사 위원회 위원

    3. 인도자 (중간 수준)¶

    접근 방법: 인도자 역할의 사용자 계정

    권한: - ✅ 모든 읽기 전용 권한 - ✅ 구역 생성 및 관리 - ✅ 지도 생성 및 관리 - ✅ 주소 추가/업데이트 - ✅ 전도인 배정 링크 생성 - ✅ 모든 회중 데이터 조회 - ✅ 가구 옵션 관리 - ✅ 구역 초기화 - ❌ 사용자 역할 관리 불가 - ❌ 회중 설정 구성 불가 - ❌ 사용자 삭제 불가

    사용 사례: 야외 봉사 감독자, 구역 조정자

    4. 관리자 (전체 접근)¶

    접근 방법: 관리자 역할의 사용자 계정

    권한: - ✅ 모든 인도자 권한 - ✅ 사용자 생성 및 관리 - ✅ 사용자 역할 할당 - ✅ 회중 설정 구성 - ✅ 가구 옵션 관리 - ✅ 구역 및 지도 삭제 - ✅ 사용자 계정 삭제 - ✅ 모든 배정 및 메시지 조회 - ✅ 최대 부재 시도 횟수 구성 - ✅ 링크 만료 시간 설정 - ✅ 시스템 로그 접근

    사용 사례: 구역 종, 시스템 관리자

    사용자 계정 관리¶

    기능: - 이메일 기반 인증 - 이메일 인증 필수 - 비밀번호 재설정 기능 - OAuth2 Google 인증 - 다단계 인증 (MFA) 지원 - 일회용 비밀번호 (OTP) 지원 - 계정 비활성화/활성화 - 마지막 로그인 추적 - 다중 회중 지원 (회중별 다른 역할)

    회중 설정¶

    사용자 정의 가능한 구성¶

    핵심 설정:

    1. 최대 부재 시도 횟수 - 완료로 간주하기 전 "부재" 방문 횟수 설정 - 일반적: 2-3회 시도 - 진행률 계산에 영향 - 회중별 설정

    2. 배정 링크 만료 - 링크 유효 기간 구성 - 기본값: 24시간 - 회중별 사용자 정의 가능 - 만료된 링크 자동 정리

    3. 가구 옵션 - 사용자 정의 주소 분류 생성 - 예: "주거", "사업체", "고층", "언어: 스페인어" - "계산 가능" 옵션 표시 (완료율에 영향) - 새 주소의 기본 옵션 설정 - 드래그 앤 드롭으로 순서 변경 - 각 옵션이 적용된 주소 조회

    4. 지역 설정 - 시간대 구성 - 국가/지역 선택 - 날짜 계산 및 보고서에 영향

    커뮤니케이션 및 메시징¶

    앱 내 커뮤니케이션 시스템¶

    메시지 유형:

    1. 전도인 메시지 - 전도인에서 인도자/관리자로 - 특정 지도에 첨부 - 구역별 피드백 - 실시간 전달

    2. 인도자 메시지 - 관리 커뮤니케이션 - 지도별 지침 - 조정 메시지

    3. 관리자 메시지 - 가시성을 위해 고정 가능 - 읽음 상태 추적 - 중요 공지 - 전도인에게 방송

    4. 고정 메시지 - 고정 해제까지 표시 유지 - 사용자별 읽음 추적 - 관련 사용자에게 자동 이메일 - 우선 표시

    이메일 알림:

    자동 이메일 요약 전송: - 30분마다: 관리자에게 읽지 않은 메시지 - 30분마다: 전도인에게 고정된 관리자 메시지 - 1시간마다: 관리자에게 메모 업데이트 - 매월: Excel 첨부파일과 함께 회중 보고서

    이메일 템플릿: - 전문적인 HTML 템플릿 - 모바일 반응형 디자인 - 관련 링크 포함 - 사용자 정의 가능한 내용

    실시간 협업¶

    실시간 데이터 동기화¶

    실시간 기능:

    SSE 구독: - Server-Sent Events (SSE)를 통한 실시간 업데이트 - 연결 끊김 시 자동 재연결 - 낮은 지연 시간 (일반적으로 100ms 미만)

    동시 편집: - 여러 사용자가 동시에 작업 가능 - 충돌이나 데이터 손실 없음 - 마지막 쓰기 우선 전략 - 타임스탬프 기반 해결

    자동 새로고침: - 변경 시 데이터 즉시 업데이트 - 수동 페이지 새로고침 불필요 - 업데이트에 대한 시각적 표시 - 낙관적 UI 업데이트

    가시성 감지: - 탭이 보이게 되면 자동으로 새로고침 - 탭이 숨겨지면 대역폭 절약 - 오래된 데이터 방지

    협업 시나리오: - 관리자가 구역 생성 중 인도자가 목록 조회 → 즉시 표시 - 전도인이 세대 완료 표시 → 관리자가 진행 업데이트 실시간 확인 - 여러 인도자가 다른 구역 관리 → 충돌 없음 - 작업 진행에 따른 실시간 통계 업데이트

    보안 기능¶

    엔터프라이즈급 보안¶

    인증 방법:

    1. 이메일/비밀번호 - Bcrypt 비밀번호 해싱 (비용: 10) - 최소 6자 - 이메일 인증 필수 - 이메일을 통한 비밀번호 재설정

    2. OAuth2 (Google) - 보안을 위한 PKCE 흐름 - 자동 사용자 생성 - 비밀번호 관리 불필요 - 안전한 토큰 처리

    3. 일회용 비밀번호 (OTP) - 선택적 4자리 코드 - 180초 유효 - 추가 보안 계층 - 이메일 전달

    4. 다단계 인증 (MFA) - TOTP 지원 - 관리자를 위한 강화된 보안 - 1800초 세션 - 백업 코드

    접근 제어:

    역할 기반 접근 제어 (RBAC): - 세밀한 권한 - 회중 수준 격리 - 다중 테넌트 아키텍처 - 기본적으로 안전

    링크 기반 접근: - 임시 익명 접근 - 토큰 기반 인증 - 자동 만료 - 취소 가능한 접근

    보안 모범 사례: - HTTPS 적용 - CORS 구성 - 속도 제한 - SQL 인젝션 방지 (매개변수화된 쿼리) - XSS 보호 - CSRF 보호 - 입력 검증 - 출력 살균

    감사 추적: - 사용자 작업 로깅 - 변경 추적 - 누가 무엇을 언제 업데이트했는지 - IP 주소 로깅 - 인증 시도

    사용자 수명 주기 관리:

    NIST AC-2 및 AC-2(3) 보안 통제에 따른 비활성 및 미프로비저닝 사용자 계정의 자동 처리:

    • 미프로비저닝 사용자: 설정을 완료하지 않은 계정은 자동으로 비활성화되고 최종 삭제되기 전에 다단계 경고 이메일을 받습니다
    • 비활성 사용자: 설정 가능한 기간 동안 로그인 활동이 없는 계정은 경고를 받은 후 비활성화됩니다
    • 백그라운드 작업이 매일 실행되어 수명 주기 정책을 평가하고 적용합니다 — 수동 개입 불필요
    • 회중 사용자 목록을 정확하게 유지하고 휴면 계정으로 인한 공격 표면을 최소화합니다

    오류 추적: - Sentry 통합 - 실시간 오류 모니터링 - 디버깅을 위한 스택 추적 - 릴리스 추적 - 성능 모니터링

    SSE (Server-Sent Events): - 실시간 구독 - 자동 재연결 - 낮은 지연 시간 - 이벤트 기반 업데이트

    통합 옵션: - PocketBase SDK (JavaScript/TypeScript) - 직접 HTTP 요청 - SSE 클라이언트 - OAuth2 통합 - Webhook 지원 (계획됨)

    데이터 관리¶

    견고한 데이터 레이어¶

    데이터베이스: PocketBase를 통한 SQLite

    기능: - ACID 트랜잭션 - 외래 키 제약 - 캐스케이드 삭제 - JSON 필드 지원 - 전문 검색 - 자동 백업

    데이터 가져오기/내보내기: - Excel 내보내기 (보고서) - CSV 내보내기 (계획됨) - API를 통한 JSON 내보내기 - 백업/복원

    데이터 무결성: - 참조 무결성 - 검증 규칙 - 고유 제약 - 검사 제약 - 자동 생성 ID
